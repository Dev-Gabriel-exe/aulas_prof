JWT — JSON Web Token

JWT é o padrão de autenticação stateless mais usado em APIs REST. O token carrega as informações do usuário embutidas e assinadas — o servidor não precisa consultar um banco de dados para validar cada requisição.

Header

Algoritmo de assinatura e tipo do token

{"alg":"HS256","typ":"JWT"}

.

Payload

Claims: sub (usuário), exp (expiração), roles (permissões)

{"sub":"ana","exp":1751234567}

.

Signature

HMAC-SHA256 do header+payload com a chave secreta

SflKxwRJSMeKKF2...

Por que stateless é importante em ML: APIs de inferência escalam horizontalmente (múltiplas réplicas no k8s). Autenticação stateful (sessão em banco) exigiria que toda requisição fosse para a mesma réplica ou que todas compartilhassem um banco de sessões. Com JWT, qualquer réplica valida o token localmente usando apenas a chave secreta — sem comunicação extra.
Implementando JWT na FastAPI
Python — auth.py
import os
from datetime import datetime, timedelta, timezone
from typing import Optional
from jose import JWTError, jwt
from passlib.context import CryptContext
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer

# Em produção: variável de ambiente, não hardcoded
SECRET_KEY = os.getenv("JWT_SECRET_KEY", "chave-secreta-troque-em-producao")
ALGORITHM  = "HS256"
TOKEN_EXPIRES_MINUTES = 60

pwd_context   = CryptContext(schemes=["bcrypt"], deprecated="auto")
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

# Banco de usuários simulado (use BD real em produção)
USUARIOS = {
    "ana": {
        "senha_hash": pwd_context.hash("senha123"),
        "role": "data_scientist"
    },
    "bot_cliente": {
        "senha_hash": pwd_context.hash("api_key_xyz"),
        "role": "api_consumer"
    }
}

def autenticar_usuario(username: str, password: str) -> Optional[dict]:
    user = USUARIOS.get(username)
    if not user:
        return None
    if not pwd_context.verify(password, user["senha_hash"]):
        return None
    return {"username": username, "role": user["role"]}

def criar_token(data: dict) -> str:
    payload = data.copy()
    payload["exp"] = datetime.now(timezone.utc) + timedelta(minutes=TOKEN_EXPIRES_MINUTES)
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

def get_usuario_atual(token: str = Depends(oauth2_scheme)) -> dict:
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username = payload.get("sub")
        role     = payload.get("role")
        if not username:
            raise HTTPException(status.HTTP_401_UNAUTHORIZED, "Token inválido")
        return {"username": username, "role": role}
    except JWTError:
        raise HTTPException(status.HTTP_401_UNAUTHORIZED, "Token expirado ou inválido")
Python — endpoints de login e predict protegido
from fastapi import FastAPI
from fastapi.security import OAuth2PasswordRequestForm
from auth import autenticar_usuario, criar_token, get_usuario_atual

app = FastAPI()

# Endpoint de login — retorna o JWT
@app.post("/token")
def login(form: OAuth2PasswordRequestForm = Depends()):
    user = autenticar_usuario(form.username, form.password)
    if not user:
        raise HTTPException(401, "Usuário ou senha incorretos")
    token = criar_token({"sub": user["username"], "role": user["role"]})
    return {"access_token": token, "token_type": "bearer"}

# Endpoint protegido — exige token válido
@app.post("/predict")
def predict(data: InputData, usuario=Depends(get_usuario_atual)):
    print(f"Predição solicitada por: {usuario['username']} ({usuario['role']})")
    # ... lógica de inferência
    return {"prediction": 1}

Rate Limiting por Usuário e Log de Auditoria
Rate limiting com SlowAPI

Rate limiting controla quantas requisições um cliente pode fazer em um intervalo de tempo. Para APIs de ML, é a principal defesa contra model extraction e abuso de recursos. O SlowAPI é uma biblioteca que adiciona rate limiting ao FastAPI com decoradores simples.

Python — rate limiting com SlowAPI
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address
from slowapi.errors import RateLimitExceeded
from fastapi import FastAPI, Request

# Limiter por IP (simples) ou por usuário autenticado
limiter = Limiter(key_func=get_remote_address)
app = FastAPI()
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)

# 10 predições por minuto por IP
@app.post("/predict")
@limiter.limit("10/minute")
def predict(request: Request, data: InputData):
    # request é obrigatório quando se usa o limiter
    return {"prediction": 1}

# Limite diferente para endpoint batch (mais restritivo)
@app.post("/predict/batch")
@limiter.limit("3/minute")
def predict_batch(request: Request, data: BatchInput):
    return {"resultados": []}
Rate limiting por usuário autenticado

Limitar por IP é fácil de contornar com proxies. A abordagem mais robusta é limitar pelo usuário autenticado no JWT — assim cada conta tem sua própria cota, independente do IP.

Python — rate limiting por usuário JWT
from fastapi import Depends, Request
from auth import get_usuario_atual

def get_username_for_rate_limit(request: Request) -> str:
    """Extrai username do JWT para usar como chave do rate limit."""
    auth_header = request.headers.get("Authorization", "")
    if auth_header.startswith("Bearer "):
        try:
            from jose import jwt
            token   = auth_header.split(" ")[1]
            payload = jwt.decode(token, SECRET_KEY, algorithms=["HS256"])
            return payload.get("sub", "anonimo")
        except:
            pass
    return get_remote_address(request)  # fallback para IP

# Limiter usando username como chave
limiter_usuario = Limiter(key_func=get_username_for_rate_limit)

@app.post("/predict")
@limiter_usuario.limit("60/minute")   # 60 predições/min por usuário
def predict(request: Request, data: InputData,
            usuario=Depends(get_usuario_atual)):
    return {"prediction": 1}

Log de auditoria — quem fez o quê e quando

Auditoria é obrigatória em contextos regulatórios (LGPD, GDPR, sistemas financeiros). Para APIs de ML, os campos mínimos de cada predição registrada são: usuário, timestamp, endpoint, input hash, output e latência.

Python — middleware de auditoria
import json, time, hashlib, logging
from fastapi import Request, Response

audit_log = logging.getLogger("auditoria")
audit_log.setLevel(logging.INFO)

# Handler que grava em arquivo separado
handler = logging.FileHandler("logs/auditoria.jsonl")
audit_log.addHandler(handler)

@app.middleware("http")
async def middleware_auditoria(request: Request, call_next):
    inicio = time.perf_counter()

    # Capturar o body da requisição para hash
    body = await request.body()

    response: Response = await call_next(request)

    duracao_ms = (time.perf_counter() - inicio) * 1000

    # Extrair usuário do token (se houver)
    username = "anonimo"
    auth = request.headers.get("Authorization", "")
    if auth.startswith("Bearer "):
        try:
            payload = jwt.decode(auth.split()[1], SECRET_KEY, algorithms=["HS256"])
            username = payload.get("sub", "anonimo")
        except: pass

    # Gravar evento de auditoria (nunca o input bruto — apenas hash)
    audit_log.info(json.dumps({
        "ts"       : time.time(),
        "usuario"  : username,
        "metodo"   : request.method,
        "endpoint" : request.url.path,
        "ip"       : request.client.host,
        "status"   : response.status_code,
        "ms"       : round(duracao_ms, 2),
        "input_hash": hashlib.sha256(body).hexdigest()[:16]
    }))

    return response
Nunca logue o input bruto da predição. Os dados de entrada podem conter informações pessoais (CPF, renda, histórico de saúde). Logue apenas o hash do input — suficiente para rastreabilidade sem expor dados sensíveis. Em sistemas que exigem full audit trail, armazene os dados criptografados separadamente com acesso restrito.

API de Inferência Segura — Todas as Camadas Integradas
Camadas de segurança da API completa

A segurança de uma API de ML é construída em camadas — cada uma com responsabilidade específica:

  • Camada 1 — Autenticação JWT: só clientes com token válido chegam aos endpoints de predição. Tokens expiram em 60 minutos.
  • Camada 2 — Rate limiting por usuário: mesmo um usuário autenticado não pode fazer consultas ilimitadas. Limites diferentes por perfil (humano vs sistema).
  • Camada 3 — Validação de input (Pydantic): rejeita automaticamente dados malformados antes de chegarem ao modelo. Limites de tamanho de lista previnem DoS.
  • Camada 4 — Auditoria: todo acesso é registrado com hash do input, usuário e resultado. Rastreabilidade para compliance.
  • Camada 5 — Monitoramento (Prometheus): métricas de tentativas de rate limit excedido, erros 401/403 e padrões anômalos alertam sobre possíveis ataques.

JWT

Header.Payload.Signature — stateless e escalável. Token expira, é assinado com chave secreta e carrega username + role. 401 automático se inválido

Rate Limiting

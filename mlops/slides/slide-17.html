Governança de ML e SHAP — Explicações Globais
Governança de ML — o contexto regulatório

Governança de ML é o conjunto de práticas, políticas e ferramentas para garantir que modelos sejam desenvolvidos e operados de forma responsável, transparente e auditável. Deixou de ser opcional: regulamentações já exigem explicações de decisões automatizadas em diversos setores.

ContextoObrigaçãoFerramenta
Crédito / FinançasExplicar por que crédito foi negado (LGPD Art. 20, Fair Credit)SHAP local por predição
SaúdeJustificar diagnóstico ou triagem para o médico responsávelLIME local + relatório
RH / ContrataçãoDemonstrar ausência de viés de gênero/raça nas decisõesSHAP global + análise de equidade
SegurosExplicar cálculo de risco personalizadoSHAP global + local
Setor PúblicoAuditabilidade de decisões que afetam cidadãosDocumentação + MLflow
O que é SHAP — Shapley Additive Explanations

SHAP é baseado na teoria dos jogos cooperativos. A ideia central: cada feature é um "jogador" que contribui para o resultado final (a predição). O valor Shapley de cada feature é a sua contribuição marginal média considerando todas as possíveis combinações com as outras features.

Propriedades que fazem o SHAP ser o padrão ouro:

  • Consistência: se uma feature contribui mais em dois modelos, seu valor SHAP será maior no modelo em que ela é mais importante.
  • Localidade + globalidade: SHAP explica predições individuais (local) E pode ser agregado para importância global do modelo.
  • Soma zero: a soma dos valores SHAP de todas as features mais o valor base (predição média) é exatamente igual à predição do modelo para aquela instância.
  • Model-agnostic (com ressalvas): funciona com qualquer modelo via KernelSHAP, mas é muito mais rápido com TreeSHAP para modelos baseados em árvores.

SHAP na prática — explicações globais
Python — shap_global.py
import shap
import pickle
import pandas as pd
import matplotlib.pyplot as plt

# Carregar modelo e dados
with open("models/model.pkl", "rb") as f:
    model = pickle.load(f)

X_treino = pd.read_csv("data/raw/dataset.csv").drop("target", axis=1)
feature_names = X_treino.columns.tolist()

# TreeExplainer é otimizado para Random Forest, GBM, XGBoost
explainer = shap.TreeExplainer(model)

# Calcular valores SHAP para todo o conjunto de treino
shap_values = explainer.shap_values(X_treino)
# Para classificação binária: shap_values[1] = valores para classe 1

# ── Gráfico 1: importância global (média dos |SHAP|) ──────────────
shap.summary_plot(
    shap_values[1], X_treino,
    plot_type="bar",
    feature_names=feature_names,
    show=False
)
plt.tight_layout()
plt.savefig("reports/shap_importancia_global.png", dpi=150)
plt.close()

# ── Gráfico 2: beeswarm — direção e magnitude dos valores SHAP ─────
shap.summary_plot(
    shap_values[1], X_treino,
    feature_names=feature_names,
    show=False
)
plt.tight_layout()
plt.savefig("reports/shap_beeswarm.png", dpi=150)
plt.close()

print("Relatórios SHAP salvos em reports/")

SHAP Local — Explicar Predições Individuais — e LIME
SHAP local — por que ESTA predição foi assim?

Enquanto o SHAP global responde "quais features são mais importantes para o modelo em geral?", o SHAP local responde "por que o modelo deu ESTA predição para ESTE cliente específico?". É o SHAP local que satisfaz o Art. 20 da LGPD.

Python — SHAP local para uma instância
import shap, pickle
import numpy as np
import pandas as pd

with open("models/model.pkl", "rb") as f:
    model = pickle.load(f)

X_treino = pd.read_csv("data/raw/dataset.csv").drop("target", axis=1)
feature_names = X_treino.columns.tolist()
explainer     = shap.TreeExplainer(model)

# Instância específica a explicar
instancia = pd.DataFrame([{
    "historico_credito": 3,
    "renda_mensal": 2500,
    "dividas_abertas": 5,
    "tempo_emprego": 6,
    "idade": 28
}])

# SHAP values para a instância
shap_vals_instancia = explainer.shap_values(instancia)[1][0]

# Montar explicação legível
explicacao = sorted(
    zip(feature_names, shap_vals_instancia),
    key=lambda x: abs(x[1]),
    reverse=True
)

print(f"Predição: {model.predict(instancia)[0]}")
print(f"Valor base: {explainer.expected_value[1]:.4f}")
print("\nContribuição de cada feature:")
for feat, val in explicacao:
    direcao = "▲ aumenta risco" if val > 0 else "▼ reduz risco"
    print(f"  {feat:25s}: {val:+.4f}  {direcao}")

# Gráfico waterfall para a instância
shap_exp = shap.Explanation(
    values=shap_vals_instancia,
    base_values=explainer.expected_value[1],
    data=instancia.values[0],
    feature_names=feature_names
)
shap.plots.waterfall(shap_exp, show=False)
import matplotlib.pyplot as plt
plt.savefig("reports/shap_waterfall_cliente.png", dpi=150, bbox_inches="tight")
plt.close()
Como interpretar o waterfall: começa no valor base (predição média do modelo). Cada barra adiciona ou subtrai a contribuição da feature para chegar à predição final desta instância. Features em vermelho aumentam o risco, em azul reduzem. A soma de todas as barras mais o valor base = predição final.
LIME — Local Interpretable Model-agnostic Explanations

O LIME usa uma abordagem diferente do SHAP: para explicar uma predição específica, ele perturba a instância (gera variações próximas), coleta as predições do modelo original para essas variações e treina um modelo linear simples sobre elas. Os coeficientes desse modelo linear são a explicação.

Vantagens do LIME:

  • Funciona com qualquer modelo (black-box total)
  • Explicações fáceis de entender (coeficientes lineares)
  • Suporta texto, imagem e tabular com a mesma API

Limitações do LIME:

  • Instável: explicações podem variar entre execuções
  • Sem garantias teóricas como o SHAP
  • Mais lento que o TreeSHAP para modelos de árvore
Python — LIME para dados tabulares
from lime.lime_tabular import LimeTabularExplainer
import numpy as np, pickle, pandas as pd

with open("models/model.pkl", "rb") as f:
    model = pickle.load(f)

X_treino = pd.read_csv("data/raw/dataset.csv").drop("target", axis=1)

# Criar o explainer LIME com os dados de treino como referência
explainer_lime = LimeTabularExplainer(
    training_data=X_treino.values,
    feature_names=X_treino.columns.tolist(),
    class_names=["nao_churn", "churn"],
    mode="classification"
)

# Instância a explicar
instancia = np.array([3, 2500, 5, 6, 28])

# Gerar explicação local
exp = explainer_lime.explain_instance(
    data_row=instancia,
    predict_fn=model.predict_proba,
    num_features=5     # top 5 features mais influentes
)

# Exibir explicação
for feat, peso in exp.as_list():
    direcao = "↑ churn" if peso > 0 else "↓ churn"
    print(f"{feat:30s}: {peso:+.4f}  {direcao}")

# Salvar como HTML interativo
exp.save_to_file("reports/lime_explicacao.html")
SHAP vs LIME — quando usar cada um
CritérioSHAPLIME
Base teóricaTeoria dos jogos (Shapley values) — sólidaAproximação linear local — heurística
EstabilidadeAlta — mesma instância → mesmo resultadoBaixa — amostras aleatórias podem variar
Velocidade (árvores)Muito rápido com TreeSHAPMais lento (requer muitas predições)
Modelos suportadosQualquer (KernelSHAP) ou árvores (TreeSHAP)Qualquer modelo via predict_fn
Uso de texto/imagemLimitadoSuporte nativo excelente
Melhor paraModelos tabulares, especialmente árvores; relatórios de complianceNLP e visão computacional; prototipagem rápida
Regra prática: para compliance regulatório com modelos tabulares sklearn/XGBoost, use SHAP com TreeExplainer — é mais rápido, mais estável e tem melhores garantias teóricas. Reserve o LIME para texto e imagem, ou quando precisar explicar modelos completamente opacos sem acesso às árvores internas.

Endpoint /explain na API e Relatório de Auditoria
Adicionando /explain à API FastAPI

O endpoint /explain recebe os mesmos dados do /predict e retorna a predição junto com a explicação estruturada — quais features mais contribuíram e em qual direção.

Python — endpoint /explain
import shap, pickle
import numpy as np
from fastapi import FastAPI
from pydantic import BaseModel
from typing import List, Dict

app = FastAPI()

# Carregar modelo e explainer na inicialização
with open("models/model.pkl", "rb") as f:
    MODEL = pickle.load(f)

EXPLAINER    = shap.TreeExplainer(MODEL)
FEATURE_NAMES = ["historico_credito", "renda_mensal",
                 "dividas_abertas", "tempo_emprego", "idade"]

class ExplainInput(BaseModel):
    features: List[float]
    top_n: int = 5           # quantas features retornar

class FeatureContribuicao(BaseModel):
    feature: str
    valor_feature: float
    contribuicao_shap: float
    direcao: str              # "aumenta_risco" ou "reduz_risco"

class ExplainOutput(BaseModel):
    prediction: int
    probability: float
    valor_base: float
    explicacao: List[FeatureContribuicao]

@app.post("/explain", response_model=ExplainOutput)
def explain(data: ExplainInput):
    X = np.array(data.features).reshape(1, -1)

    # Predição
    pred  = int(MODEL.predict(X)[0])
    proba = float(MODEL.predict_proba(X)[0][pred])

    # SHAP values
    shap_vals = EXPLAINER.shap_values(X)[1][0]
    base_val  = float(EXPLAINER.expected_value[1])

    # Ordenar por contribuição absoluta
    contribuicoes = sorted(
        zip(FEATURE_NAMES, data.features, shap_vals),
        key=lambda x: abs(x[2]),
        reverse=True
    )[:data.top_n]

    explicacao = [
        FeatureContribuicao(
            feature=f,
            valor_feature=v,
            contribuicao_shap=round(s, 4),
            direcao="aumenta_risco" if s > 0 else "reduz_risco"
        )
        for f, v, s in contribuicoes
    ]

    return ExplainOutput(
        prediction=pred,
        probability=round(proba, 4),
        valor_base=round(base_val, 4),
        explicacao=explicacao
    )

Relatório de auditoria de equidade

Um dos usos mais importantes do SHAP global é verificar se o modelo usa features proxy de raça, gênero ou outras categorias protegidas. Se a feature "CEP" tem SHAP alto, pode ser proxy de raça. O relatório de equidade deve ser gerado periodicamente e arquivado.

Python — relatório de equidade com SHAP
import pandas as pd, numpy as np, shap, json
from datetime import datetime

def gerar_relatorio_equidade(model, X_test, feature_names, caminho_saida):
    explainer  = shap.TreeExplainer(model)
    shap_vals  = explainer.shap_values(X_test)[1]
    imp_media  = np.abs(shap_vals).mean(axis=0)
    imp_df     = pd.DataFrame({
        "feature"  : feature_names,
        "shap_mean": imp_media
    }).sort_values("shap_mean", ascending=False)

    # Features que podem ser proxies problemáticas
    FEATURES_ATENCAO = ["cep", "bairro", "escolaridade", "nome"]
    alertas = [f for f in feature_names if f.lower() in FEATURES_ATENCAO]

    relatorio = {
        "data_geracao": datetime.now().isoformat(),
        "total_amostras": len(X_test),
        "importancia_features": imp_df.to_dict(orient="records"),
        "alertas_proxy": alertas,
        "aprovado_equidade": len(alertas) == 0
    }

    with open(caminho_saida, "w") as f:
        json.dump(relatorio, f, indent=2, ensure_ascii=False)

    print(f"Relatório salvo: {caminho_saida}")
    if alertas:
        print(f"⚠️  Features potencialmente problemáticas: {alertas}")
    else:
        print("✅ Nenhuma feature proxy identificada")
    return relatorio

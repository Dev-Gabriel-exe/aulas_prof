O que é Drift — Tipos, Causas e Métricas Estatísticas
O modelo fantasma — o problema que o monitoramento resolve

Você colocou o modelo em produção. As métricas de treino eram excelentes. Todos estavam satisfeitos. Seis meses depois, silenciosamente, o modelo passou a errar muito mais — mas como ninguém monitorava, ninguém sabia.

Esse é o modelo fantasma: tecnicamente vivo, servindo predições, mas produzindo resultados cada vez piores enquanto o mundo ao redor muda.

Por que acontece: modelos aprendem padrões dos dados de treino. Quando os dados do mundo real mudam — comportamento de usuários, sazonalidade, mudanças econômicas, novos produtos — o modelo continua usando o padrão antigo. A distância entre o que o modelo espera e o que chegam se chama drift.
Os três tipos de drift

Data Drift

A distribuição dos dados de entrada (features) muda em relação aos dados de treino. O modelo não foi "avisado" dessas mudanças.

Ex.: sistema de crédito treinado antes de uma crise — o perfil de renda dos clientes muda drasticamente.

Concept Drift

A relação entre features e target muda. Os dados de entrada podem parecer normais, mas o que eles significam mudou.

Ex.: modelo de detecção de spam — atacantes aprendem a contornar os padrões e o comportamento de spam muda.

Prediction Drift

A distribuição das predições do modelo muda. Sintoma de data drift ou concept drift — mais fácil de detectar quando não há labels reais.

Ex.: recomendação de produtos — as predições passam a concentrar em poucos itens, sem variedade.

Qual monitorar primeiro? Sempre comece com data drift e prediction drift — são detectáveis sem precisar de labels reais (que muitas vezes só chegam dias ou semanas depois). Concept drift é mais difícil de detectar e geralmente requer comparação de métricas do modelo ao longo do tempo com dados rotulados.
Métricas estatísticas para detectar drift

A detecção de drift é essencialmente um problema de comparação de distribuições. Os testes estatísticos abaixo quantificam o quanto a distribuição atual diverge da distribuição de referência (treino).

Kolmogorov-Smirnov (KS) Compara distribuições contínuas. Mede a maior diferença entre as CDFs das duas amostras. Numéricas
Population Stability Index (PSI) Mede quanto a distribuição de uma variável mudou. PSI < 0.1 = estável; 0.1–0.2 = monitorar; > 0.2 = drift significativo. Numéricas
Chi-Quadrado (χ²) Testa se a distribuição de frequências de uma variável categórica mudou significativamente. Categóricas
Jensen-Shannon Divergence Versão simétrica da divergência de Kullback-Leibler. Valor entre 0 (igual) e 1 (totalmente diferente). Ambas
Wasserstein Distance Mede o "custo de transporte" para transformar uma distribuição na outra. Intuitiva e robusta a outliers. Numéricas
Python — calculando KS e PSI manualmente
from scipy import stats
import numpy as np

def calcular_psi(referencia, producao, n_bins=10):
    """PSI: < 0.1 estável | 0.1-0.2 monitorar | > 0.2 drift"""
    bins = np.percentile(referencia, np.linspace(0, 100, n_bins + 1))
    ref_freq = np.histogram(referencia, bins=bins)[0] / len(referencia)
    prod_freq = np.histogram(producao, bins=bins)[0] / len(producao)
    # Evitar log(0)
    ref_freq  = np.where(ref_freq  == 0, 0.0001, ref_freq)
    prod_freq = np.where(prod_freq == 0, 0.0001, prod_freq)
    psi = np.sum((prod_freq - ref_freq) * np.log(prod_freq / ref_freq))
    return psi

# Simular dados com drift
treino    = np.random.normal(0,   1, 1000)   # referência
producao  = np.random.normal(0.8, 1, 500)    # com drift

ks_stat, ks_pval = stats.ks_2samp(treino, producao)
psi               = calcular_psi(treino, producao)

print(f"KS statistic : {ks_stat:.4f} (p-value: {ks_pval:.4f})")
print(f"PSI          : {psi:.4f} → {'DRIFT' if psi > 0.2 else 'OK'}")

Evidently AI — Relatórios, Testes e Alertas Automáticos
O que é Evidently AI

Evidently AI é uma biblioteca open-source para monitoramento de modelos de ML. Ela automatiza a comparação entre um dataset de referência (treino) e um dataset atual (produção), gerando relatórios visuais em HTML e resultados de testes estruturados em JSON.

Dois modos principais:

  • Reports (Relatórios): análise exploratória — gera HTML com distribuições, métricas de drift e qualidade dos dados. Ideal para auditoria manual periódica.
  • Test Suites (Suítes de Testes): testes automatizáveis com critério de passou/falhou. Ideal para CI/CD — se o drift ultrapassar o limiar, o pipeline falha e dispara alerta.
Gerando relatórios com Evidently
Python — relatório de drift de dados
import pandas as pd
import numpy as np
from evidently.report import Report
from evidently.metric_preset import DataDriftPreset, DataQualityPreset

# Simular dados de treino (referência)
np.random.seed(42)
n = 1000
treino = pd.DataFrame({
    "feature_1": np.random.normal(0, 1, n),
    "feature_2": np.random.normal(5, 2, n),
    "categoria": np.random.choice(["A", "B", "C"], n),
    "target"   : np.random.randint(0, 2, n),
})

# Simular dados de produção COM drift
producao = pd.DataFrame({
    "feature_1": np.random.normal(1.5, 1.5, n // 2),  # drift na média e desvio
    "feature_2": np.random.normal(5,   2,   n // 2),  # sem drift
    "categoria": np.random.choice(["A", "B", "C", "D"], n // 2),  # nova categoria D
    "target"   : np.random.randint(0, 2, n // 2),
})

# Criar e executar o relatório
relatorio = Report(metrics=[
    DataDriftPreset(),    # drift por feature + resumo geral
    DataQualityPreset(),  # nulos, duplicatas, outliers
])

relatorio.run(reference_data=treino, current_data=producao)

# Salvar como HTML interativo
relatorio.save_html("reports/drift_report.html")
print("Relatório salvo em reports/drift_report.html")

# Também disponível como dicionário Python
resultado = relatorio.as_dict()
n_drift = resultado["metrics"][0]["result"]["number_of_drifted_columns"]
print(f"Features com drift detectado: {n_drift}")
Test Suites — monitoramento automatizável com critérios

Enquanto os Reports são para análise humana, as Test Suites são para automação — cada teste tem um critério de aprovação e retorna True/False. Isso permite integrar no pipeline de CI/CD ou rodar como job agendado.

Python — test suite de monitoramento
from evidently.test_suite import TestSuite
from evidently.tests import (
    TestNumberOfDriftedColumns,
    TestShareOfDriftedColumns,
    TestColumnDrift,
    TestNumberOfMissingValues,
)

suite = TestSuite(tests=[
    # Máximo de 1 feature com drift
    TestNumberOfDriftedColumns(lt=2),

    # No máximo 30% das features com drift
    TestShareOfDriftedColumns(lt=0.3),

    # feature_1 especificamente não pode ter drift
    TestColumnDrift(column_name="feature_1", stattest="ks"),

    # Nenhum valor nulo nos dados de produção
    TestNumberOfMissingValues(lt=10),
])

suite.run(reference_data=treino, current_data=producao)

# Verificar se passou em todos os testes
resultado = suite.as_dict()
passou = all(
    test["status"] == "SUCCESS"
    for test in resultado["tests"]
)

if not passou:
    print("⚠️  ALERTA: drift detectado — considerar retreinamento")
    suite.save_html("reports/drift_alerta.html")
else:
    print("✅ Dados dentro dos limites esperados")
Integração com CI/CD: esse script pode ser executado como um job agendado (cron diário, por exemplo). Se passou == False, o script pode: (1) salvar o relatório, (2) enviar email/Slack de alerta, (3) abrir uma issue no GitHub automaticamente, e (4) acionar o retreinamento do modelo via pipeline DVC ou MLflow.
Monitorando predições sem labels reais

Em muitos cenários reais, os labels verdadeiros chegam muito tempo depois das predições (ex.: inadimplência de crédito demora meses para ser confirmada). O monitoramento de prediction drift detecta anomalias sem depender dos labels.

Python — monitorar distribuição das predições
import pickle
import pandas as pd
from evidently.report import Report
from evidently.metric_preset import DataDriftPreset

with open("models/model.pkl", "rb") as f:
    model = pickle.load(f)

# Dados de treino — predições de referência
X_treino = treino.drop("target", axis=1).select_dtypes("number")
pred_treino = pd.DataFrame({
    "prediction": model.predict(X_treino),
    "proba_1": model.predict_proba(X_treino)[:, 1]
})

# Dados de produção — predições atuais
X_prod = producao.drop("target", axis=1).select_dtypes("number")
pred_prod = pd.DataFrame({
    "prediction": model.predict(X_prod),
    "proba_1": model.predict_proba(X_prod)[:, 1]
})

# Comparar distribuição das predições
rel_pred = Report(metrics=[DataDriftPreset()])
rel_pred.run(reference_data=pred_treino, current_data=pred_prod)
rel_pred.save_html("reports/prediction_drift.html")

Pipeline de Monitoramento Automatizado e Atividade Prática
Pipeline de monitoramento end-to-end

Em produção, o monitoramento não é executado manualmente. É um job agendado que roda periodicamente, coleta os dados de produção do período, compara com a referência e toma ações automáticas.

Python — monitor.py — script de monitoramento completo
"""
Pipeline de monitoramento de drift.
Execute periodicamente (ex: cron diário).
"""
import os, json, logging
from datetime import datetime
import pandas as pd
from evidently.test_suite import TestSuite
from evidently.tests import (
    TestNumberOfDriftedColumns,
    TestShareOfDriftedColumns,
    TestNumberOfMissingValues,
)

logging.basicConfig(level=logging.INFO)
log = logging.getLogger("monitor")

# ── Configuração ──────────────────────────────────────────────────
LIMIAR_DRIFT_COLUNAS   = 2     # máximo de features com drift
LIMIAR_DRIFT_SHARE     = 0.3  # máximo de 30% das features
LIMIAR_VALORES_NULOS   = 50   # máximo de valores nulos
PASTA_RELATORIOS       = "reports/"
os.makedirs(PASTA_RELATORIOS, exist_ok=True)

def carregar_dados_producao():
    """Aqui você conectaria ao banco/log real de produção."""
    return pd.read_csv("data/producao_hoje.csv")

def carregar_referencia():
    return pd.read_csv("data/raw/dataset.csv")

def executar_monitoramento():
    log.info("Iniciando monitoramento de drift...")

    referencia = carregar_referencia()
    producao   = carregar_dados_producao()

    suite = TestSuite(tests=[
        TestNumberOfDriftedColumns(lt=LIMIAR_DRIFT_COLUNAS),
        TestShareOfDriftedColumns(lt=LIMIAR_DRIFT_SHARE),
        TestNumberOfMissingValues(lt=LIMIAR_VALORES_NULOS),
    ])
    suite.run(reference_data=referencia, current_data=producao)

    resultado = suite.as_dict()
    passou    = all(t["status"] == "SUCCESS" for t in resultado["tests"])
    ts        = datetime.now().strftime("%Y%m%d_%H%M")

    # Salvar relatório HTML sempre
    html_path = f"{PASTA_RELATORIOS}drift_{ts}.html"
    suite.save_html(html_path)

    # Salvar resultado JSON para histórico
    json_path = f"{PASTA_RELATORIOS}drift_{ts}.json"
    with open(json_path, "w") as f:
        json.dump({"ts": ts, "passou": passou, "tests": resultado["tests"]}, f, indent=2)

    if passou:
        log.info("✅ Todos os testes passaram. Modelo estável.")
    else:
        log.warning("⚠️  Drift detectado! Relatório: %s", html_path)
        disparar_alerta(html_path, resultado["tests"])

    return passou

def disparar_alerta(html_path, testes):
    # Na prática: enviar email, mensagem Slack, abrir issue no GitHub
    falhas = [t["name"] for t in testes if t["status"] != "SUCCESS"]
    log.error("Testes com falha: %s", falhas)
    log.error("Relatório completo em: %s", html_path)
    # Sugestão: acionar retreinamento via DVC / MLflow Pipeline

if __name__ == "__main__":
    executar_monitoramento()

Use o dataset da Semana 3 como referência (treino). Você vai gerar dois datasets de produção simulados: um sem drift e um com drift intencional — e verificar que o Evidently detecta apenas o segundo.
Passo 01 — Instalar Evidently e criar datasets simulados

Instale o Evidently. Carregue o dataset de treino como referência. Crie dois datasets de produção: prod_ok.csv (mesma distribuição do treino) e prod_drift.csv (com feature_1 tendo média deslocada em +2 desvios padrão). Salve os três como CSV.

Básico
Python — criar_dados.py
import numpy as np
import pandas as pd
from sklearn.datasets import make_classification

np.random.seed(42)
X, y = make_classification(n_samples=1000, n_features=10, random_state=42)
cols = [f"f{i}" for i in range(10)]
treino = pd.DataFrame(X, columns=cols)
treino["target"] = y
treino.to_csv("data/raw/dataset.csv", index=False)

# Sem drift — mesma distribuição
X2, y2 = make_classification(n_samples=500, n_features=10, random_state=99)
prod_ok = pd.DataFrame(X2, columns=cols)
prod_ok["target"] = y2
prod_ok.to_csv("data/prod_ok.csv", index=False)

# Com drift — feature f0 deslocada
prod_drift = prod_ok.copy()
prod_drift["f0"] = prod_drift["f0"] + 2.0
prod_drift.to_csv("data/prod_drift.csv", index=False)
Passo 02 — Gerar relatórios HTML com Evidently

Crie um Report com DataDriftPreset() para cada dataset de produção (ok e drift). Salve os dois relatórios HTML em reports/. Abra os dois no navegador e compare os resultados — observe quais features aparecem como "drifted" no relatório com drift e por quê.

Básico
Python
import os, pandas as pd
from evidently.report import Report
from evidently.metric_preset import DataDriftPreset

os.makedirs("reports", exist_ok=True)
ref = pd.read_csv("data/raw/dataset.csv")

for nome, path in [("ok", "data/prod_ok.csv"), ("drift", "data/prod_drift.csv")]:
    prod = pd.read_csv(path)
    r = Report(metrics=[DataDriftPreset()])
    r.run(reference_data=ref, current_data=prod)
    r.save_html(f"reports/relatorio_{nome}.html")
    res = r.as_dict()["metrics"][0]["result"]
    print(f"{nome}: {res['number_of_drifted_columns']} features com drift")
Passo 03 — Test Suite com critérios de alerta

Crie uma Test Suite com critérios: máximo 1 feature com drift e máximo 20% das features com drift. Rode contra os dois datasets. O resultado de prod_ok deve passar em todos os testes. O de prod_drift deve falhar em pelo menos um. Imprima qual teste falhou.

Médio
Python
from evidently.test_suite import TestSuite
from evidently.tests import TestNumberOfDriftedColumns, TestShareOfDriftedColumns

for nome, path in [("ok", "data/prod_ok.csv"), ("drift", "data/prod_drift.csv")]:
    prod = pd.read_csv(path)
    suite = TestSuite(tests=[
        TestNumberOfDriftedColumns(lt=2),
        TestShareOfDriftedColumns(lt=0.2),
    ])
    suite.run(reference_data=ref, current_data=prod)
    res = suite.as_dict()
    passou = all(t["status"] == "SUCCESS" for t in res["tests"])
    icon = "✅" if passou else "⚠️"
    print(f"{icon} {nome}: passou={passou}")
    if not passou:
        falhas = [t["name"] for t in res["tests"] if t["status"] != "SUCCESS"]
        print(f"  Falhas: {falhas}")

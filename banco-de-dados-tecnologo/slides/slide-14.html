B-tree e Hash — como os índices funcionam por dentro
O problema sem índice — Sequential Scan

Imagine uma tabela pedidos com 5 milhões de linhas. A query WHERE cliente_id = 42 precisa encontrar todos os pedidos desse cliente. Sem índice, o PostgreSQL não tem escolha: lê cada linha da tabela, uma a uma, da primeira à última — isso se chama Sequential Scan.

Sequential Scan (sem índice):

  • Lê todas as 5M linhas independente do filtro
  • Complexidade O(n) — dobrar a tabela dobra o tempo
  • Eficiente quando retorna a maioria das linhas (>10–15%)
  • Sem custo de manutenção — não há estrutura extra

Index Scan (com B-tree):

  • Navega pela árvore em O(log n) — 5M linhas = ~23 comparações
  • Salta direto para os registros que satisfazem o filtro
  • Custo de manutenção: cada INSERT/UPDATE/DELETE atualiza o índice
  • Ocupa espaço em disco — tipicamente 10–30% do tamanho da tabela
Índice B-tree — a estrutura padrão do PostgreSQL

B-tree (Balanced Tree) é o tipo de índice padrão do PostgreSQL. É uma árvore onde todos os nós folha estão na mesma profundidade, garantindo O(log n) para qualquer busca. Suporta comparações de igualdade (=), intervalo (<, >, BETWEEN), ordenação e LIKE com prefixo fixo (LIKE 'abc%').

SQL — criação de índices B-tree
-- Índice simples em uma coluna
CREATE INDEX idx_pedidos_cliente_id
    ON pedidos (cliente_id);

-- Índice com nome descritivo (boa prática)
CREATE INDEX idx_pedidos_criado_em
    ON pedidos (criado_em);

-- Índice UNIQUE — garante unicidade E cria índice
CREATE UNIQUE INDEX idx_clientes_cpf
    ON clientes (cpf);

-- Ver índices de uma tabela
SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'pedidos';

-- Remover índice
DROP INDEX idx_pedidos_cliente_id;

-- Criar índice sem bloquear tabela (produção)
CREATE INDEX CONCURRENTLY idx_pedidos_status
    ON pedidos (status);
CREATE INDEX CONCURRENTLY cria o índice sem bloquear leituras e escritas na tabela — essencial em produção. O índice normal (CREATE INDEX) bloqueia escritas durante a construção, o que pode causar indisponibilidade em tabelas grandes.
Hash index — busca exata em O(1)

Índices Hash armazenam um mapa de hash(valor) → localização no disco. A busca é O(1) — independentemente do tamanho da tabela, sempre são feitas exatamente 1–2 operações. A limitação fatal: só funciona com igualdade exata (=). Não suporta <, >, BETWEEN, ORDER BY ou LIKE.

Índices compostos — a ordem das colunas importa

Um índice composto cobre múltiplas colunas em uma só estrutura. A regra fundamental: o índice é útil se a query filtrar pelo prefixo do índice — ou seja, pelas primeiras colunas na ordem em que foram declaradas.

SQL — índice composto e regra do prefixo
-- Índice composto: (status, criado_em)
CREATE INDEX idx_pedidos_status_data
    ON pedidos (status, criado_em);

-- ✓ USA o índice — filtra pela 1ª coluna (status)
SELECT * FROM pedidos WHERE status = 'entregue';

-- ✓ USA o índice — filtra por ambas as colunas
SELECT * FROM pedidos
WHERE status = 'entregue' AND criado_em > '2025-01-01';

-- ✗ NÃO usa o índice — pula a 1ª coluna do prefixo
SELECT * FROM pedidos WHERE criado_em > '2025-01-01';
-- Para isso, precisaria de um índice separado em (criado_em)

-- Índice covering (INCLUDE) — evita heap fetch
CREATE INDEX idx_pedidos_covering
    ON pedidos (cliente_id)
    INCLUDE (total, status);
-- Query que usa só cliente_id, total e status
-- nunca precisa acessar a tabela principal (index-only scan)

Regra prática para compostos: coloque primeiro a coluna com maior seletividade (menos valores distintos para filtrar mais) e que aparece mais em filtros de igualdade. Colunas de range (>, BETWEEN) ficam por último no índice.
EXPLAIN ANALYZE — lendo o plano de execução

EXPLAIN mostra o plano que o PostgreSQL pretende executar. EXPLAIN ANALYZE executa a query de verdade e mostra o plano com tempos reais. É a ferramenta essencial para entender por que uma query é lenta.

SQL — usando EXPLAIN ANALYZE
-- EXPLAIN: só mostra o plano (não executa)
EXPLAIN
SELECT * FROM pedidos WHERE cliente_id = 42;

-- EXPLAIN ANALYZE: executa E mostra tempos reais
EXPLAIN ANALYZE
SELECT * FROM pedidos WHERE cliente_id = 42;

-- EXPLAIN (ANALYZE, BUFFERS): mostra cache hits/misses também
EXPLAIN (ANALYZE, BUFFERS)
SELECT p.id, p.total, c.nome
FROM pedidos p
JOIN clientes c ON c.id = p.cliente_id
WHERE p.status = 'entregue'
  AND p.criado_em > '2025-01-01';

Quando índices prejudicam a performance

Índices não são gratuitos. Cada índice precisa ser atualizado a cada INSERT, UPDATE e DELETE. Em tabelas com alta taxa de escrita, índices demais podem tornar as escritas mais lentas do que as leituras ganharam.

Evite indexar: colunas com poucos valores distintos e sem seletividade (ex: coluna ativo BOOLEAN — apenas true/false, índice seria inútil pois retorna 50% das linhas). Colunas raramente usadas em filtros. Tabelas pequenas (menos de ~1.000 linhas) — Seq Scan é mais rápido que Index Scan para tabelas pequenas.
SQL — inspecionar índices usados e não usados
-- Índices existentes e quanto espaço ocupam
SELECT
    indexname,
    pg_size_pretty(pg_relation_size(indexrelid)) AS tamanho,
    idx_scan     AS vezes_usado,
    idx_tup_read AS linhas_lidas_via_indice
FROM pg_stat_user_indexes
WHERE relname = 'pedidos'
ORDER BY idx_scan DESC;

-- Índices nunca usados (candidatos a remoção)
SELECT schemaname, tablename, indexname
FROM pg_stat_user_indexes
WHERE idx_scan = 0
  AND indexname NOT LIKE '%pkey%'   -- não remover PKs
ORDER BY tablename;

-- Atualizar estatísticas do planner (usar quando rows estimado ≠ real)
ANALYZE pedidos;
VACUUM ANALYZE pedidos;  -- limpa linhas mortas E atualiza estatísticas

Benchmark em Python — medir impacto real dos índices
Python — benchmark completo com e sem índice
import time, os
import psycopg2

conn = psycopg2.connect(os.getenv("DATABASE_URL"))
conn.autocommit = True
cur = conn.cursor()

def medir(descricao, sql, params=None, repeticoes=5):
    """Executa a query N vezes e retorna a média em ms."""
    tempos = []
    for _ in range(repeticoes):
        t0 = time.perf_counter()
        cur.execute(sql, params)
        cur.fetchall()
        tempos.append((time.perf_counter() - t0) * 1000)
    media = sum(tempos) / len(tempos)
    print(f"  {descricao:<45} {media:>8.1f} ms")
    return media

# ── Queries de teste ──────────────────────────────────────────────
Q_IGUALDADE = "SELECT * FROM pedidos_bench WHERE cliente_id = %s"
Q_RANGE     = "SELECT * FROM pedidos_bench WHERE criado_em > %s"
Q_COMPOSTO  = "SELECT * FROM pedidos_bench WHERE status = %s AND criado_em > %s"
Q_COVERING  = "SELECT cliente_id, total FROM pedidos_bench WHERE cliente_id = %s"

print("="*65)
print("FASE 1 — SEM ÍNDICE (apenas PK)")
print("="*65)
t1_sem = medir("WHERE cliente_id = 42",        Q_IGUALDADE, (42,))
t2_sem = medir("WHERE criado_em > '2025-01-01'", Q_RANGE,     ("2025-01-01",))
t3_sem = medir("WHERE status='entregue' AND data",Q_COMPOSTO,  ("entregue","2025-01-01"))

# ── Criar índices ─────────────────────────────────────────────────
print("\nCriando índices...")
cur.execute("CREATE INDEX IF NOT EXISTS idx_bench_cliente ON pedidos_bench(cliente_id)")
cur.execute("CREATE INDEX IF NOT EXISTS idx_bench_data ON pedidos_bench(criado_em)")
cur.execute("CREATE INDEX IF NOT EXISTS idx_bench_composto ON pedidos_bench(status, criado_em)")
cur.execute("CREATE INDEX IF NOT EXISTS idx_bench_covering ON pedidos_bench(cliente_id) INCLUDE (total)")
print("Índices criados.\n")

print("="*65)
print("FASE 2 — COM ÍNDICES")
print("="*65)
t1_com = medir("WHERE cliente_id = 42",        Q_IGUALDADE, (42,))
t2_com = medir("WHERE criado_em > '2025-01-01'", Q_RANGE,     ("2025-01-01",))
t3_com = medir("WHERE status='entregue' AND data",Q_COMPOSTO,  ("entregue","2025-01-01"))
t4_cov = medir("Index Only Scan (covering)",      Q_COVERING,  (42,))

# ── Resumo ────────────────────────────────────────────────────────
print("\n"+"="*65)
print("GANHO DE PERFORMANCE")
print("="*65)
for desc, sem, com in [
    ("cliente_id = 42",    t1_sem, t1_com),
    ("criado_em range",    t2_sem, t2_com),
    ("status + data",      t3_sem, t3_com),
]:
    ganho = sem / com if com > 0 else 0
    print(f"  {desc:<30} {ganho:>6.0f}× mais rápido")

cur.close()
conn.close()
Lendo o EXPLAIN em Python
Python — capturar e imprimir o plano de execução
import psycopg2, os, json

conn = psycopg2.connect(os.getenv("DATABASE_URL"))
cur = conn.cursor()

def explain(sql, params=None, analisar=True):
    """Retorna o plano de execução formatado."""
    modo = "EXPLAIN (ANALYZE, FORMAT JSON)" if analisar else "EXPLAIN (FORMAT JSON)"
    cur.execute(f"{modo} {sql}", params)
    plano = cur.fetchone()[0][0]  # JSON do PostgreSQL
    return plano

# Comparar planos antes e depois do índice
queries = [
    ("Sem índice (Seq Scan esperado)",
     "SELECT * FROM pedidos_bench WHERE cliente_id = %s", (42,)),
    ("Com índice B-tree",
     "SELECT * FROM pedidos_bench WHERE cliente_id = %s", (42,)),
]

for desc, sql, params in queries:
    plano = explain(sql, params)
    no_raiz = plano["Plan"]
    node_type    = no_raiz.get("Node Type", "?")
    actual_time  = no_raiz.get("Actual Total Time", 0)
    actual_rows  = no_raiz.get("Actual Rows", 0)
    total_cost   = no_raiz.get("Total Cost", 0)

    print(f"\n── {desc} ──")
    print(f"  Nó raiz:    {node_type}")
    print(f"  Custo est.: {total_cost:.2f}")
    print(f"  Tempo real: {actual_time:.3f} ms")
    print(f"  Linhas ret: {actual_rows}")

cur.close()
conn.close()

Exercício 03

Médio — Index Only Scan e covering index
Python
import psycopg2, time, os

conn = psycopg2.connect(os.getenv("DATABASE_URL"))
conn.autocommit = True
cur = conn.cursor()

SQL = "SELECT cliente_id, total FROM pedidos_bench WHERE cliente_id = %s"
PARAMS = (42,)

def media_ms(n=10):
    ts = []
    for _ in range(n):
        t0 = time.perf_counter()
        cur.execute(SQL, PARAMS)
        cur.fetchall()
        ts.append((time.perf_counter()-t0)*1000)
    return sum(ts)/len(ts)

def node_type():
    cur.execute(f"EXPLAIN (ANALYZE, FORMAT JSON) {SQL}", PARAMS)
    plano = cur.fetchone()[0][0]
    return plano["Plan"].get("Node Type")

# Remover índices anteriores em cliente_id para partir do zero
cur.execute("DROP INDEX IF EXISTS idx_bench_cliente")
cur.execute("DROP INDEX IF EXISTS idx_bench_covering")

t_sem = media_ms()
tipo_sem = node_type()
print(f"Sem índice:              {t_sem:.2f} ms | {tipo_sem}")

# Índice simples (Index Scan — ainda vai ao heap)
cur.execute("CREATE INDEX idx_bench_cliente ON pedidos_bench(cliente_id)")
t_simples = media_ms()
tipo_simples = node_type()
print(f"Com B-tree simples:      {t_simples:.2f} ms | {tipo_simples}")

# Covering index (Index Only Scan — não vai ao heap)
cur.execute("DROP INDEX idx_bench_cliente")
cur.execute("CREATE INDEX idx_bench_covering ON pedidos_bench(cliente_id) INCLUDE (total)")
t_cov = media_ms()
tipo_cov = node_type()
print(f"Com covering index:      {t_cov:.2f} ms | {tipo_cov}")
print(f"Ganho covering vs sem:   {t_sem/t_cov:.1f}×")

cur.close(); conn.close()

Desafio — Analisador automático de plano de execução
Python
import psycopg2, os

conn = psycopg2.connect(os.getenv("DATABASE_URL"))
conn.autocommit = True
cur = conn.cursor()

def extrair_nos(no, profundidade=0):
    """Extrai recursivamente todos os nós do plano."""
    nos = [(profundidade, no)]
    for filho in no.get("Plans", []):
        nos.extend(extrair_nos(filho, profundidade + 1))
    return nos

def analisar_query(sql, params=None):
    """Analisa uma query e imprime resumo do plano de execução."""
    cur.execute(f"EXPLAIN (ANALYZE, FORMAT JSON) {sql}", params)
    plano = cur.fetchone()[0][0]
    nos = extrair_nos(plano["Plan"])

    print("\n" + "─"*60)
    print(f"QUERY: {sql[:70]}..." if len(sql) > 70 else f"QUERY: {sql}")
    print("─"*60)
    alertas = []

    for prof, no in nos:
        indent  = "  " * prof
        tipo    = no.get("Node Type", "?")
        custo   = no.get("Total Cost", 0)
        t_real  = no.get("Actual Total Time", 0)
        rows_e  = no.get("Plan Rows", 0)
        rows_r  = no.get("Actual Rows", 0)
        tabela  = no.get("Relation Name", "")

        sinal = "⚠️ " if tipo == "Seq Scan" else "   "
        print(f"{indent}{sinal}{tipo:<25} custo={custo:>10.1f} | t={t_real:>8.2f}ms | rows est={rows_e} real={rows_r}")

        # Detectar Seq Scan suspeito
        if tipo == "Seq Scan" and rows_e > 100_000:
            alertas.append(f"Seq Scan em '{tabela}' com {rows_e:,} linhas estimadas — considere um índice")

    if alertas:
        print("\n⚠️  ALERTAS:")
        for a in alertas:
            print(f"   → {a}")
    else:
        print("\n✓ Nenhum Seq Scan suspeito detectado")

    print(f"Execution Time: {plano.get('Execution Time',0):.2f} ms\n")

# Testar com 3 queries
analisar_query("SELECT * FROM pedidos_bench WHERE cliente_id = %s", (42,))
analisar_query("SELECT * FROM pedidos_bench WHERE total > %s", (9000,))
analisar_query(
    "SELECT * FROM pedidos_bench WHERE status = %s AND criado_em > %s",
    ("entregue", "2025-01-01")
)

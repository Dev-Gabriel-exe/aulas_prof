Qualidade de dados — as 6 dimensões e como medi-las
O problema do dado ruim

Imagine treinar um modelo de recomendação de produtos usando uma coluna preco que tem 30% dos valores nulos, datas de compra lançadas no futuro e IDs de cliente duplicados com nomes diferentes. O modelo vai aprender padrões que não existem no mundo real.

Governança de dados é o conjunto de políticas, processos e ferramentas que garantem que os dados sejam confiáveis, acessíveis, compreensíveis e protegidos ao longo de todo o seu ciclo de vida.

Regra dos 80/20 na prática: em projetos de dados reais, cerca de 80% do tempo é gasto em limpeza, validação e entendimento dos dados — e apenas 20% em modelagem. Governança busca inverter essa proporção ao longo do tempo.
As 6 dimensões de qualidade de dados
Completude

Todos os campos obrigatórios estão preenchidos? % de nulos por coluna.

Acurácia

O valor reflete a realidade? CPF válido, e-mail com @, preço positivo.

Unicidade

Sem duplicatas indesejadas. Um cliente não deve aparecer duas vezes com IDs diferentes.

Temporalidade

Dados estão atualizados? Data de nascimento não pode ser futura.

Consistência

Mesmo valor em fontes diferentes. Endereço no CRM igual ao do sistema de entregas?

Conformidade

Segue o formato esperado? CEP no padrão 00000-000, UF com 2 letras, CNPJ com 14 dígitos.

Diagnóstico rápido de qualidade com pandas
Python — auditoria de qualidade em DataFrame
import pandas as pd
from sqlalchemy import create_engine
import os

engine = create_engine(os.getenv("DATABASE_URL"))
df = pd.read_sql("SELECT * FROM clientes", engine)

# ── 1. Completude ────────────────────────────────────────────────
nulos = df.isnull().sum()
pct_nulos = (nulos / len(df) * 100).round(2)
print("=== Completude (% nulos por coluna) ===")
print(pd.DataFrame({"nulos": nulos, "%": pct_nulos})[pct_nulos > 0])

# ── 2. Unicidade ─────────────────────────────────────────────────
duplicatas = df.duplicated(subset=["cpf"]).sum()
print(f"\n=== Unicidade === \nCPFs duplicados: {duplicatas}")

# ── 3. Acurácia — preço positivo, email com @ ────────────────────
if "email" in df.columns:
    emails_invalidos = df[~df["email"].str.contains("@", na=False)]
    print(f"\n=== Acurácia ===\nE-mails inválidos: {len(emails_invalidos)}")

# ── 4. Temporalidade — datas no futuro ───────────────────────────
if "data_nascimento" in df.columns:
    df["data_nascimento"] = pd.to_datetime(df["data_nascimento"], errors="coerce")
    futuros = df[df["data_nascimento"] > pd.Timestamp("today")]
    print(f"Datas de nascimento no futuro: {len(futuros)}")

# ── 5. Conformidade — CEP no padrão ──────────────────────────────
if "cep" in df.columns:
    cep_invalido = df[~df["cep"].astype(str).str.match(r"^\d{5}-\d{3}$")]
    print(f"CEPs fora do padrão: {len(cep_invalido)}")

# ── Resumo ───────────────────────────────────────────────────────
print(f"\nTotal de registros: {len(df)}")
print(f"Colunas: {list(df.columns)}")
print(f"Tipos:\n{df.dtypes}")
Great Expectations — validação declarativa e automatizável

Great Expectations (GX) permite escrever expectativas sobre os dados como código — e testá-las automaticamente a cada execução do pipeline. Se uma expectativa falha, o pipeline para antes de propagar dados ruins.

Python — Great Expectations básico com pandas
import great_expectations as gx
import pandas as pd

# Dataset de exemplo
df = pd.DataFrame({
    "cliente_id": [1, 2, 3, 4, 5],
    "email":      ["a@b.com", "c@d.com", "invalido", "e@f.com", "g@h.com"],
    "total_gasto":[4239.9, 1200.0, -50.0, 399.9, 4150.0],
    "status":     ["ativo", "inativo", "ativo", "suspenso", "ATIVO"],
})

# Criar contexto e dataset validável
context = gx.get_context()
ds = context.sources.add_pandas("minha_fonte")
da = ds.add_dataframe_asset("clientes")
batch = da.build_batch_request(dataframe=df)

# Criar suite de expectativas
suite = context.add_expectation_suite("clientes_suite")
validator = context.get_validator(batch_request=batch, expectation_suite=suite)

# ── Definir expectativas ─────────────────────────────────────────
# Coluna cliente_id: sem nulos e valores únicos
validator.expect_column_values_to_not_be_null("cliente_id")
validator.expect_column_values_to_be_unique("cliente_id")

# total_gasto: sempre positivo
validator.expect_column_values_to_be_between("total_gasto", min_value=0)

# status: só valores válidos do conjunto
validator.expect_column_values_to_be_in_set(
    "status", ["ativo", "inativo", "suspenso"]
)

# email: não pode ser nulo
validator.expect_column_values_to_not_be_null("email")

# Salvar e executar a validação
validator.save_expectation_suite()
results = validator.validate()

print(f"Sucesso: {results['success']}")
for r in results["results"]:
    status = "✓" if r["success"] else "✗"
    exp = r["expectation_config"]["expectation_type"]
    col = r["expectation_config"]["kwargs"].get("column", "")
    print(f"  {status} {exp} [{col}]")

Linhagem de dados — rastreando a jornada do dado

Linhagem (ou data lineage) é a capacidade de responder: "De onde veio este dado e como ele foi transformado?"

Em um pipeline BD → Parquet → Delta Lake → modelo de ML, se o modelo começa a errar, precisamos saber se o problema está nos dados brutos, em alguma transformação intermediária ou no próprio modelo.

Exemplo de mapa de linhagem (texto)
# Linhagem da feature "ticket_medio"

PostgreSQL (tabela pedidos)
  └─ raw/pedidos/         # extração bruta, sem alteração
       └─ silver/pedidos/ # limpeza: nulos→0, status.lower(), anomalias removidas
            └─ gold/features_cliente/
                 ├─ ticket_medio = mean(total) por cliente_id
                 └─ usado em: modelo_churn_v2.pkl (treino: 2025-01-15)
Python — registrar linhagem manualmente em JSON
import json, hashlib
from datetime import datetime

def registrar_linhagem(entrada, saida, transformacoes, arquivo="lineage_log.json"):
    """Registra uma etapa de linhagem no arquivo de log."""
    registro = {
        "timestamp": datetime.now().isoformat(),
        "entrada": entrada,
        "saida": saida,
        "transformacoes": transformacoes,
    }
    try:
        with open(arquivo) as f:
            log = json.load(f)
    except FileNotFoundError:
        log = []
    log.append(registro)
    with open(arquivo, "w") as f:
        json.dump(log, f, indent=2, ensure_ascii=False)
    print(f"Linhagem registrada: {entrada} → {saida}")

# Uso no pipeline
registrar_linhagem(
    entrada="postgresql://pedidos",
    saida="lake/raw/pedidos/",
    transformacoes=["nenhuma — cópia fiel"]
)

registrar_linhagem(
    entrada="lake/raw/pedidos/",
    saida="lake/silver/pedidos/",
    transformacoes=[
        "total nulo → 0",
        "status.lower().strip()",
        "removidos registros com total <= 0",
        "criadas colunas ano e mes",
    ]
)

registrar_linhagem(
    entrada="lake/silver/pedidos/",
    saida="lake/gold/features_cliente",
    transformacoes=[
        "groupby cliente_id",
        "agg: count, mean, sum, max, lambda cancelado",
        "criada coluna dias_inativo",
        "escrita como Delta Lake",
    ]
)
Catálogo de dados — metadados e descoberta

Um catálogo de dados é o "Google" interno dos seus dados. Engenheiros e cientistas buscam datasets disponíveis, entendem o que cada coluna significa e sabem quem é o responsável — sem precisar perguntar para ninguém.

O que um catálogo contém:

  • Nome e descrição de cada dataset e coluna
  • Owner — equipe ou pessoa responsável
  • Classificação de sensibilidade — público, interno, confidencial, dados pessoais (LGPD)
  • Frequência de atualização — horária, diária, sob demanda
  • Linhagem — de onde vem e para onde vai

Ferramentas de mercado:

  • OpenMetadata — open-source, completo, self-hosted
  • DataHub (LinkedIn) — open-source, muito usado em Big Data
  • Atlan / Alation — SaaS empresarial
  • dbt docs — catálogo embutido para transformações dbt
  • README por pasta — solução mínima válida para projetos pequenos
Python — catálogo mínimo em YAML / JSON

LGPD aplicada a bancos de dados

A Lei Geral de Proteção de Dados (Lei nº 13.709/2018) regula o tratamento de dados pessoais — qualquer informação que identifique ou torne identificável uma pessoa natural. Pipelines de dados não estão isentos.

Dados pessoais

Nome, CPF, e-mail, telefone, endereço, IP, localização, comportamento de compra.

Dados sensíveis

Saúde, biometria, raça, religião, opinião política, vida sexual. Proteção reforçada — base legal mais restrita.

Titular de dados

A pessoa a quem os dados se referem tem direitos: acesso, correção, exclusão, portabilidade, revogação de consentimento.

Obrigação LGPDImpacto no banco/pipelineTécnica
MinimizaçãoNão coletar mais do que o necessárioSELECT só das colunas necessárias; não replicar dados pessoais para todas as zonas
FinalidadeDados usados só para o propósito declaradoControle de acesso por papel (RBAC); catálogo com finalidade documentada
SegurançaProteger contra acesso não autorizadoCriptografia em repouso e em trânsito; VPC; secrets manager
ExclusãoDireito ao esquecimentoManter mapeamento titular → registros; script de exclusão em cascata
AnonimizaçãoDado anônimo não é dado pessoalHash irreversível (SHA-256 + salt), generalização, supressão
Atenção: copiar dados pessoais brutos para a zona gold ou para datasets de treino de ML sem anonimização é uma violação de LGPD. A zona raw pode ter dados pessoais — a silver e a gold idealmente não.

Python — mascaramento e anonimização de dados pessoais
import pandas as pd, hashlib, os
from faker import Faker

fake = Faker("pt_BR")

# Dataset fictício com dados pessoais
df_raw = pd.DataFrame({
    "cliente_id":   [1, 2, 3, 4],
    "nome":         ["Ana Silva", "Bruno Costa", "Carla Melo", "Diego Nunes"],
    "cpf":          ["111.222.333-44", "555.666.777-88", "999.000.111-22", "333.444.555-66"],
    "email":        ["ana@x.com", "bruno@y.com", "carla@z.com", "diego@w.com"],
    "idade":        [28, 45, 33, 61],
    "total_gasto":  [4239.9, 1200.0, 399.9, 4150.0],
})

SALT = os.getenv("HASH_SALT", "meu_salt_secreto")

def hash_campo(valor):
    """Hash SHA-256 irreversível com salt."""
    return hashlib.sha256(f"{SALT}{valor}".encode()).hexdigest()[:12]

def faixa_etaria(idade):
    """Generaliza idade em faixa."""
    faixas = [(18,24),(< span class="num">25,34),(35,44),(45,59),(60,99)]
    for lo, hi in faixas:
        if lo <= idade <= hi:
            return f"{lo}-{hi}"
    return "desconhecida"

df_silver = df_raw.copy()

# Supressão — remover nome e CPF da zona silver
df_silver = df_silver.drop(columns=["nome", "cpf"])

# Pseudonimização por hash — manter email como chave de join, sem expor
df_silver["email_hash"] = df_silver["email"].apply(hash_campo)
df_silver = df_silver.drop(columns=["email"])

# Generalização — substituir idade por faixa etária
df_silver["faixa_etaria"] = df_silver["idade"].apply(faixa_etaria)
df_silver = df_silver.drop(columns=["idade"])

print("=== Dataset raw (DADOS PESSOAIS - zona restrita) ===")
print(df_raw.to_string(index=False))
print("\n=== Dataset silver (ANONIMIZADO - pode ser usado em ML) ===")
print(df_silver.to_string(index=False))
Pipeline completo com checagem de qualidade embutida
Python — pipeline BD → raw → silver (com validação GX + linhagem)
import pandas as pd, json, hashlib, os
from datetime import datetime
from sqlalchemy import create_engine

engine = create_engine(os.getenv("DATABASE_URL"))
LOG = []

def log(entrada, saida, transformacoes):
    LOG.append({
        "ts": datetime.now().isoformat(),
        "de": entrada, "para": saida,
        "ops": transformacoes
    })

def validar_qualidade(df, etapa):
    """Checagem mínima de qualidade — levanta erro se falhar."""
    erros = []
    if df["id"].isnull().any():
        erros.append("id com nulos")
    if df["id"].duplicated().any():
        erros.append("id duplicado")
    if ("total" in df.columns) and (df["total"] < 0).any():
        erros.append("total negativo")
    if erros:
        raise ValueError(f"[{etapa}] Qualidade reprovada: {erros}")
    print(f"[{etapa}] ✓ Qualidade OK — {len(df)} registros")

SALT = os.getenv("HASH_SALT", "salt_dev")

# ── ETAPA 1: raw ──────────────────────────────────────────────────
df_raw = pd.read_sql("SELECT id, cliente_id, email, total, status, criado_em FROM pedidos", engine)
df_raw.to_parquet("lake/raw/pedidos/", partition_cols=["status"], index=False)
log("postgresql/pedidos", "lake/raw/pedidos", ["extração sem transformação"])
validar_qualidade(df_raw, "raw")

# ── ETAPA 2: silver — limpar + anonimizar ─────────────────────────
df_s = df_raw.copy()
df_s["criado_em"] = pd.to_datetime(df_s["criado_em"])
df_s["status"]    = df_s["status"].str.lower().str.strip()
df_s["total"]     = df_s["total"].fillna(0)
df_s["ano"]       = df_s["criado_em"].dt.year
df_s["mes"]       = df_s["criado_em"].dt.month

# Anonimizar e-mail (hash) e remover se presente
if "email" in df_s.columns:
    df_s["email_hash"] = df_s["email"].apply(
        lambda v: hashlib.sha256(f"{SALT}{v}".encode()).hexdigest()[:16]
    )
    df_s = df_s.drop(columns=["email"])

df_s = df_s[df_s["total"] > 0]
df_s.to_parquet("lake/silver/pedidos/", partition_cols=["ano","mes"], index=False, compression="zstd")
log("lake/raw/pedidos", "lake/silver/pedidos", [
    "total nulo → 0", "status normalizado", "email substituído por hash SHA-256",
    "filtrado total > 0", "particionado por ano e mês"
])
validar_qualidade(df_s, "silver")

# ── Gravar linhagem ───────────────────────────────────────────────
with open("lake/lineage_log.json", "w") as f:
    json.dump(LOG, f, indent=2, ensure_ascii=False)
print("Linhagem gravada. Pipeline concluído.")

import pandas as pd, hashlib, os
from sqlalchemy import create_engine

engine = create_engine(os.getenv("DATABASE_URL"))
df = pd.read_sql("SELECT id, nome, cpf, email, idade, cidade FROM clientes", engine)

def hash_email(v):
    return hashlib.sha256(f"{SALT}{v}".encode()).hexdigest()[:16]

df_masked = df.copy()

df_masked = df_masked.drop(columns=["idade"])

print("=== ANTES ===")

Para cada um, documente: nome, descrição, owner, frequência de atualização, formato, sensibilidade LGPD e lista de colunas com tipo e flag lgpd: true/false. Salve em catalog.json e leia de volta para imprimir as colunas com lgpd: true.

Exercício 05 — Desafio

Implemente um pipeline completo de governança: (1) carregue a tabela clientes; (2) valide com Great Expectations antes de qualquer transformação — se falhar, grave um log de erros e interrompa o pipeline; (3) se passar, aplique mascaramento (supressão de CPF e nome, hash do email, faixa etária); (4) salve em silver/clientes/ como Parquet particionado por cidade; (5) registre a linhagem em JSON com as transformações aplicadas. O pipeline deve imprimir um resumo final com quantidade de registros antes e depois, colunas removidas e expectativas validadas.

Desafio — Pipeline de governança end-to-end
Python
import pandas as pd, hashlib, json, os
from datetime import datetime
from sqlalchemy import create_engine
import great_expectations as gx

engine = create_engine(os.getenv("DATABASE_URL"))
SALT = os.getenv("HASH_SALT", "salt_dev")

# ── 1. Carregar ───────────────────────────────────────────────────
df = pd.read_sql("SELECT id, nome, cpf, email, idade, cidade FROM clientes", engine)
registros_antes = len(df)
print(f"Carregados: {registros_antes} clientes")

# ── 2. Validar qualidade com Great Expectations ───────────────────
context = gx.get_context()
ds = context.sources.add_pandas("clientes_gov")
da = ds.add_dataframe_asset("clientes_raw")
batch = da.build_batch_request(dataframe=df)
suite = context.add_expectation_suite("clientes_gov_suite")
v = context.get_validator(batch_request=batch, expectation_suite=suite)

v.expect_column_values_to_not_be_null("id")
v.expect_column_values_to_be_unique("id")
v.expect_column_values_to_not_be_null("cidade")
v.save_expectation_suite()
results = v.validate()

if not results["success"]:
    erros = [r for r in results["results"] if not r["success"]]
    with open("quality_errors.json", "w") as f:
        json.dump(erros, f, indent=2)
    raise RuntimeError(f"Pipeline interrompido: {len(erros)} expectativa(s) falhou. Ver quality_errors.json")

print("✓ Qualidade validada — prosseguindo")
exp_ok = [r["expectation_config"]["expectation_type"] for r in results["results"] if r["success"]]

# ── 3. Mascaramento ───────────────────────────────────────────────
df_s = df.copy()
colunas_removidas = ["nome", "cpf"]
df_s = df_s.drop(columns=colunas_removidas)
df_s["email_hash"] = df["email"].apply(
    lambda v: hashlib.sha256(f"{SALT}{v}".encode()).hexdigest()[:16]
)
df_s = df_s.drop(columns=["email"])
colunas_removidas.append("email")

def faixa(i):
    for lo, hi in [(18,24),(25,34),(35,44),(45,59),(60,120)]:
        if lo <= i <= hi:
            return f"{lo}–{hi}"
    return "?"
df_s["faixa_etaria"] = df_s["idade"].apply(faixa)
df_s = df_s.drop(columns=["idade"])
colunas_removidas.append("idade")

# ── 4. Salvar Parquet particionado ────────────────────────────────
df_s["cidade"] = df_s["cidade"].fillna("Desconhecida")
df_s.to_parquet("silver/clientes/", partition_cols=["cidade"], index=False, compression="zstd")

# ── 5. Linhagem ───────────────────────────────────────────────────
linhagem = {
    "ts": datetime.now().isoformat(),
    "de": "postgresql/clientes",
    "para": "silver/clientes",
    "ops": [
        "supressão: nome, cpf, email",
        "email substituído por email_hash SHA-256+salt",
        "idade generalizada em faixa_etaria",
        "cidade nula → 'Desconhecida'",
        "particionado por cidade",
    ]
}
with open("silver/lineage_clientes.json", "w") as f:
    json.dump(linhagem, f, indent=2, ensure_ascii=False)

# ── Resumo final ──────────────────────────────────────────────────
print("\n══════════ RESUMO DO PIPELINE ══════════")
print(f"Registros antes:   {registros_antes}")
print(f"Registros depois:  {len(df_s)}")
print(f"Colunas removidas: {colunas_removidas}")
print(f"Expectativas OK:   {exp_ok}")
print("Linhagem gravada em silver/lineage_clientes.json")
print("════════════════════════════════════════")

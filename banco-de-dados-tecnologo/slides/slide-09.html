Revisão rápida — por que Parquet?

Parquet é um formato de arquivo colunar e binário. Em vez de guardar dados linha por linha (como CSV), guarda coluna por coluna. O ganho:

  • Leitura seletiva: uma query que usa só 3 de 50 colunas lê apenas aquelas 3
  • Compressão superior: valores repetidos na mesma coluna comprimem muito melhor
  • Schema embutido: tipos preservados — sem surpresa com "data lida como string"
  • Estatísticas de coluna: min/max/null por row group — permite pular blocos inteiros
  • Compatível com tudo: pandas, Spark, DuckDB, Athena, BigQuery
  • Não é human-readable — precisa de ferramenta para inspecionar
Particionamento — dividir para conquistar

Particionamento organiza os arquivos em pastas baseadas em valores de colunas. Ao ler, o engine pula as partições que não atendem ao filtro — sem nem abrir o arquivo.

Estrutura particionada no sistema de arquivos
pedidos/
├── ano=2024/
│   ├── mes=10/
│   │   └── part-00000.parquet   # só pedidos de out/2024
│   ├── mes=11/
│   │   └── part-00000.parquet
│   └── mes=12/
│       └── part-00000.parquet
└── ano=2025/
    └── mes=01/
        └── part-00000.parquet
Python — escrever Parquet particionado
import pandas as pd
from sqlalchemy import create_engine
import os

engine = create_engine(os.getenv("DATABASE_URL"))

df = pd.read_sql("""
    SELECT id, cliente_id, status, total, criado_em
    FROM pedidos
    ORDER BY criado_em
""", engine)

# Criar colunas de partição
df["criado_em"] = pd.to_datetime(df["criado_em"])
df["ano"] = df["criado_em"].dt.year
df["mes"] = df["criado_em"].dt.month

# Escrever particionado por ano e mês
df.to_parquet(
    "silver/pedidos/",
    partition_cols=["ano", "mes"],    # cria subpastas automaticamente
    index=False,
    compression="snappy",
    engine="pyarrow"
)
print("Partições criadas:")
for root, dirs, files in os.walk("silver/pedidos"):
    for f in files:
        print(f"  {os.path.join(root,f)}")
Python — ler com predicate pushdown (só 1 partição)
import pyarrow.parquet as pq

# Ler APENAS janeiro de 2025 — não toca nas outras partições
dataset = pq.read_table(
    "silver/pedidos/",
    filters=[("ano", "=", 2025), ("mes", "=", 1)]
)
df_jan = dataset.to_pandas()
print(f"Linhas lidas: {len(df_jan)}")

# Também funciona com pandas direto
df_jan = pd.read_parquet(
    "silver/pedidos/",
    filters=[("ano", "=", 2025), ("mes", "=", 1)],
    columns=["id", "cliente_id", "total"]   # column pruning
)
Predicate pushdown + column pruning = o engine lê apenas os arquivos das partições filtradas e apenas as colunas solicitadas. Em datasets de TB, isso é a diferença entre segundos e horas.

S3 e Delta Lake — armazenamento e transações
S3 — armazenamento de objetos

Amazon S3 (e equivalentes como MinIO, GCS, Azure Blob) é a base de praticamente todo Data Lake moderno. É barato, durável, escalável e agnóstico de formato.

🥉 s3://lake/raw/
Dados brutos

Exatamente como chegaram — CSV, JSON, logs. Imutável. Serve como backup e ponto de reprocessamento.

🥈 s3://lake/silver/
Dados limpos

Parquet particionado, tipos corretos, nulos tratados. Pode ser consultado com confiança por Spark ou Athena.

🥇 s3://lake/gold/
Features prontas

Tabelas wide para ML, aggregados para BI. Consumo direto sem transformação adicional.

Python — boto3: upload e download no S3 / MinIO
import boto3, os
from io import BytesIO
import pandas as pd

# Conectar ao MinIO local (compatível com S3)
s3 = boto3.client(
    "s3",
    endpoint_url="http://localhost:9000",    # MinIO local
    aws_access_key_id="minioadmin",
    aws_secret_access_key="minioadmin",
)
# Para S3 real: remover endpoint_url e usar credenciais IAM

BUCKET = "data-lake"

# Upload de arquivo local
s3.upload_file(
    "silver/pedidos/ano=2025/mes=01/part-00000.parquet",
    BUCKET,
    "silver/pedidos/ano=2025/mes=01/part-00000.parquet"
)

# Ler Parquet direto do S3 sem baixar o arquivo
obj = s3.get_object(Bucket=BUCKET, Key="silver/pedidos/ano=2025/mes=01/part-00000.parquet")
df = pd.read_parquet(BytesIO(obj["Body"].read()))
print(df.head())

# Listar objetos com prefixo (equivale a ls de pasta)
response = s3.list_objects_v2(Bucket=BUCKET, Prefix="silver/pedidos/")
for obj in response.get("Contents", []):
    print(obj["Key"], "|", obj["Size"], "bytes")
Delta Lake — ACID sobre arquivos Parquet

Delta Lake adiciona uma camada de metadados transacionais sobre arquivos Parquet. Os dados continuam em Parquet, mas uma pasta _delta_log/ registra todas as operações.

ACID Transactions

Múltiplos writers simultâneos sem corromper dados. Commit atômico — ou toda a operação funciona ou nada.

Time Travel

Consultar versões anteriores dos dados. "Como estava a tabela ontem às 18h?" — útil para auditoria e ML reproducível.

Schema Evolution

Adicionar colunas sem reescrever toda a tabela. Registra a mudança de schema no log com timestamp.

Upsert (MERGE)

INSERT + UPDATE em uma operação atômica. Essencial para incrementais — inserir novos e atualizar existentes.

Python — Delta Lake com biblioteca deltalake
from deltalake import DeltaTable, write_deltalake
import pandas as pd

# Escrever tabela Delta pela primeira vez
df_v1 = pd.DataFrame({
    "cliente_id": [1, 2, 3],
    "total_gasto": [4239.90, 1200.00, 399.90],
    "ultima_compra": ["2025-01-15", "2025-01-10", "2025-01-18"],
})
write_deltalake("gold/features_cliente", df_v1)
print("Versão 0 escrita")

# Escrever novos dados (append — versão 1)
df_v2 = pd.DataFrame({
    "cliente_id": [4, 5],
    "total_gasto": [4150.00, 89.90],
    "ultima_compra": ["2024-10-05", "2024-11-20"],
})
write_deltalake("gold/features_cliente", df_v2, mode="append")
print("Versão 1 (append) escrita")

# Ler a tabela atual
dt = DeltaTable("gold/features_cliente")
df_atual = dt.to_pandas()
print(f"Versão atual: {dt.version()} | {len(df_atual)} linhas")

# TIME TRAVEL — ler versão anterior
dt_v0 = DeltaTable("gold/features_cliente", version=0)
df_ontem = dt_v0.to_pandas()
print(f"Versão 0: {len(df_ontem)} linhas")

# Ver histórico de todas as versões
print(dt.history())
_delta_log/ é o coração do Delta Lake — uma pasta de arquivos JSON que registra cada operação. Nunca delete essa pasta. O VACUUM remove arquivos de dados antigos, mas o log permanece.

Pipeline Lakehouse — banco → silver → gold → Delta
Pipeline end-to-end com Delta Lake
Python — pipeline completo BD → Lakehouse
import pandas as pd, os, shutil
from sqlalchemy import create_engine
from deltalake import DeltaTable, write_deltalake
import pyarrow.parquet as pq

engine = create_engine(os.getenv("DATABASE_URL"))

# ── ETAPA 1: raw — extrair do banco bruto ────────────────────────
df_raw = pd.read_sql("SELECT * FROM pedidos", engine)
df_raw.to_parquet(
    "lake/raw/pedidos/",
    partition_cols=["status"],
    index=False,
    compression="snappy"
)
print(f"raw: {len(df_raw)} linhas")

# ── ETAPA 2: silver — limpar e tipar ─────────────────────────────
df_raw["criado_em"] = pd.to_datetime(df_raw["criado_em"])
df_raw["status"]    = df_raw["status"].str.lower().str.strip()
df_raw["total"]     = df_raw["total"].fillna(0)
df_raw["ano"]       = df_raw["criado_em"].dt.year
df_raw["mes"]       = df_raw["criado_em"].dt.month
df_silver = df_raw[df_raw["total"] > 0].copy()

df_silver.to_parquet(
    "lake/silver/pedidos/",
    partition_cols=["ano", "mes"],
    index=False,
    compression="zstd"
)
print(f"silver: {len(df_silver)} linhas")

# ── ETAPA 3: gold — features de cliente em Delta Lake ────────────
df_gold = df_silver.groupby("cliente_id").agg(
    total_pedidos    =("id",       "count"),
    ticket_medio     =("total",    "mean"),
    total_gasto      =("total",    "sum"),
    ultimo_pedido    =("criado_em","max"),
    qtd_cancelados   =("status",   lambda x: (x == "cancelado").sum()),
).reset_index()

df_gold["dias_inativo"] = (pd.Timestamp("today") - df_gold["ultimo_pedido"]).dt.days
df_gold["ticket_medio"] = df_gold["ticket_medio"].round(2)
df_gold["gerado_em"]    = pd.Timestamp("now")

# Escrever como Delta Lake
write_deltalake("lake/gold/features_cliente", df_gold, mode="overwrite")

dt = DeltaTable("lake/gold/features_cliente")
print(f"gold Delta v{dt.version()}: {len(df_gold)} clientes")

# Ler de volta para validar
df_check = dt.to_pandas()
print(df_check[["cliente_id","total_gasto","dias_inativo"]])
Time Travel na prática
Python — consultar versões históricas
dt = DeltaTable("lake/gold/features_cliente")

# Ver todo o histórico
historico = dt.history()
for h in historico:
    print(f"v{h['version']} | {h['timestamp']} | {h['operation']}")

# Ler versão específica pelo número
df_v0 = DeltaTable("lake/gold/features_cliente", version=0).to_pandas()

# Ler versão por timestamp (como estava num momento específico)
df_ontem = DeltaTable(
    "lake/gold/features_cliente",
    version=dt.version_at_timestamp("2025-01-14T12:00:00")
).to_pandas()

# Comparar versões: quais clientes mudaram?
v_atual = dt.to_pandas().set_index("cliente_id")["total_gasto"]
v_antiga = df_v0.set_index("cliente_id")["total_gasto"]
diff = (v_atual - v_antiga).dropna()
print("Clientes com total_gasto alterado:\n", diff[diff != 0])

Exercício 01

Extraia a tabela clientes do banco para um DataFrame. Escreva em Parquet particionado por cidade na pasta silver/clientes/. Depois leia de volta filtrando apenas a cidade "São Paulo" usando predicate pushdown e column pruning (só colunas id, nome e email). Compare o tempo de leitura com e sem filtro.

Básico — Parquet particionado
Python
import pandas as pd, time, os
from sqlalchemy import create_engine

engine = create_engine(os.getenv("DATABASE_URL"))
df = pd.read_sql("SELECT * FROM clientes", engine)
df["cidade"] = df["cidade"].fillna("Desconhecida")

# Escrever particionado
df.to_parquet("silver/clientes/", partition_cols=["cidade"],
              index=False, compression="zstd")

# Ler SEM filtro
t0 = time.time()
df_all = pd.read_parquet("silver/clientes/")
print(f"Sem filtro: {len(df_all)} linhas em {time.time()-t0:.4f}s")

# Ler COM predicate pushdown + column pruning
t0 = time.time()
df_sp = pd.read_parquet(
    "silver/clientes/",
    filters=[("cidade", "=", "São Paulo")],
    columns=["id", "nome", "email"]
)
print(f"Com filtro: {len(df_sp)} linhas em {time.time()-t0:.4f}s")
print(df_sp)

Exercício 04

Compare o tamanho em disco de um mesmo dataset de pedidos salvo em: (a) CSV sem compressão, (b) Parquet com Snappy, (c) Parquet com Zstd. Calcule a razão de compressão relativa ao CSV e imprima a tabela comparativa.

Médio — Comparativo de formatos
Python
import pandas as pd, os
from sqlalchemy import create_engine

engine = create_engine(os.getenv("DATABASE_URL"))
df = pd.read_sql("SELECT * FROM pedidos", engine)

formatos = {
    "CSV (sem compressão)":   (lambda: df.to_csv("pedidos.csv", index=False),   "pedidos.csv"),
    "Parquet Snappy":          (lambda: df.to_parquet("pedidos_snappy.parquet", compression="snappy", index=False), "pedidos_snappy.parquet"),
    "Parquet Zstd":            (lambda: df.to_parquet("pedidos_zstd.parquet",   compression="zstd",   index=False), "pedidos_zstd.parquet"),
}

resultados = []
csv_size = None
for nome, (fn, path) in formatos.items():
    fn()
    size = os.path.getsize(path)
    if csv_size is None:
        csv_size = size
    resultados.append({"Formato": nome, "Bytes": size,
                        "KB": round(size/1024,1),
                        "Razão vs CSV": f"{csv_size/size:.1f}x menor"})

print(pd.DataFrame(resultados).to_string(index=False))

Desafio — Pipeline incremental com Delta Lake
Python
from deltalake import DeltaTable, write_deltalake
import pandas as pd, shutil

# Limpar execução anterior (só para teste)
shutil.rmtree("gold/incremental", ignore_errors=True)

# 1. Versão inicial
df_dia1 = pd.DataFrame({
    "cliente_id": [1,2,3],
    "total_gasto": [4239.90, 1200.00, 399.90],
    "dias_inativo": [5, 51, 43],
})
write_deltalake("gold/incremental", df_dia1)
print("v0 escrita — 3 clientes")

# 2. Nova execução: 2 novos + 1 atualizado via overwrite parcial
# Na prática real usaria MERGE/UPSERT; aqui simulamos com overwrite
df_dia2 = pd.DataFrame({
    "cliente_id": [1,2,3,4,5],
    "total_gasto": [5189.80, 1200.00, 399.90, 4150.00, 89.90],
    "dias_inativo": [4, 52, 44, 83, 203],
})
write_deltalake("gold/incremental", df_dia2, mode="overwrite")
print("v1 escrita — 5 clientes (cliente 1 atualizado)")

dt = DeltaTable("gold/incremental")

# 3. Comparar versões para o cliente 1
v0 = DeltaTable("gold/incremental", version=0).to_pandas()
v1 = dt.to_pandas()
c1_v0 = v0[v0.cliente_id==1]["total_gasto"].values[0]
c1_v1 = v1[v1.cliente_id==1]["total_gasto"].values[0]
print(f"Cliente 1 — antes: R${c1_v0:.2f} | depois: R${c1_v1:.2f}")

# 4. Vacuum dry run — ver o que seria removido
arquivos = dt.vacuum(retention_hours=0, dry_run=True)
print(f"Arquivos que seriam removidos pelo vacuum: {len(arquivos)}")
for f in arquivos:
    print(" ", f)

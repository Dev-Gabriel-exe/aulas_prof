Revisão do Modelo Relacional
Você já viu isso no 2° período. Vamos revisar de forma objetiva e garantir que todo mundo está na mesma base antes de avançar para SQL avançado na semana que vem.
Chave Primária (PK)

Identifica de forma única cada linha de uma tabela. Não pode ser nula. Não pode se repetir.

SQL — PostgreSQL 
CREATE TABLE clientes (
  id        SERIAL PRIMARY KEY,         -- PK: auto-incremento
  nome      VARCHAR(100) NOT NULL,
  email     VARCHAR(150) UNIQUE NOT NULL,  -- único + obrigatório
  criado_em TIMESTAMP DEFAULT NOW()
);
Chave Estrangeira (FK) e Integridade Referencial

A FK cria o vínculo entre tabelas. O banco rejeita automaticamente operações que quebrariam esse vínculo — você não consegue inserir um pedido para um cliente inexistente.

SQL — PostgreSQL 
CREATE TABLE pedidos (
  id         SERIAL PRIMARY KEY,
  cliente_id INT NOT NULL REFERENCES clientes(id) ON DELETE CASCADE,
  total      NUMERIC(10,2) NOT NULL,
  criado_em  TIMESTAMP DEFAULT NOW()
);

-- ON DELETE CASCADE  → deleta os pedidos ao deletar o cliente
-- ON DELETE RESTRICT → bloqueia a deleção se houver pedidos (padrão)
-- ON DELETE SET NULL → coloca NULL na FK ao deletar o cliente
Tipos de Relacionamento
TipoExemplo RealComo Implementar
1 : 1Pessoa ↔ CPFFK com UNIQUE na tabela filha
1 : NCliente → PedidosFK na tabela do lado "N"
N : NPedido ↔ ProdutoTabela intermediária
SQL — Tabela N:N 
CREATE TABLE itens_pedido (
  pedido_id  INT REFERENCES pedidos(id)  ON DELETE CASCADE,
  produto_id INT REFERENCES produtos(id) ON DELETE RESTRICT,
  quantidade INT NOT NULL,
  preco_unit NUMERIC(10,2) NOT NULL,
  PRIMARY KEY (pedido_id, produto_id)   -- PK composta
);
Por que isso importa para IA?

Modelos de Machine Learning aprendem padrões dos dados. Se o banco tiver dados duplicados (falta de normalização), relacionamentos quebrados (sem FK) ou inconsistências (valores nulos inesperados), o modelo vai aprender padrões errados.

Regra de ouro: modelagem correta → dados limpos → modelo confiável. Não existe atalho.

Vamos usar o loja_exemplo — 4 tabelas simples que refletem um sistema real. Rode o script abaixo para criar tudo.

SQL — Setup do banco loja_exemplo 
CREATE TABLE clientes (
  id SERIAL PRIMARY KEY, nome VARCHAR(100) NOT NULL,
  email VARCHAR(150) UNIQUE, cidade VARCHAR(80), criado_em TIMESTAMP DEFAULT NOW()
);
CREATE TABLE produtos (
  id SERIAL PRIMARY KEY, nome VARCHAR(120) NOT NULL,
  categoria VARCHAR(60), preco NUMERIC(10,2) NOT NULL, estoque INT DEFAULT 0
);
CREATE TABLE pedidos (
  id SERIAL PRIMARY KEY, cliente_id INT REFERENCES clientes(id),
  status VARCHAR(20) DEFAULT 'pendente', total NUMERIC(10,2), criado_em TIMESTAMP DEFAULT NOW()
);
CREATE TABLE itens_pedido (
  pedido_id INT REFERENCES pedidos(id), produto_id INT REFERENCES produtos(id),
  quantidade INT NOT NULL, preco_unit NUMERIC(10,2) NOT NULL,
  PRIMARY KEY (pedido_id, produto_id)
);

SELECT, WHERE e filtros
SQL
-- Filtro com BETWEEN
SELECT nome, preco FROM produtos
WHERE categoria = 'Eletrônicos'
  AND preco BETWEEN 100 AND 1000
  AND estoque > 0;

-- ILIKE: busca parcial sem distinção de maiúsculas (PostgreSQL)
SELECT * FROM clientes WHERE nome ILIKE '%silva%';

-- IN: múltiplos valores
SELECT * FROM pedidos WHERE status IN ('pendente', 'processando');

-- IS NULL
SELECT nome FROM clientes WHERE email IS NULL;

-- Paginação
SELECT nome, preco FROM produtos
ORDER BY preco DESC
LIMIT 5 OFFSET 0;   -- página 1 · 5 produtos mais caros
JOINs — o coração do modelo relacional
INNER JOIN → apenas linhas que têm correspondência nos dois lados.
LEFT JOIN → todas as linhas da tabela esquerda, com ou sem correspondência.
SQL — INNER JOIN
-- Pedidos com nome do cliente
SELECT
  c.nome        AS cliente,
  p.criado_em   AS data_pedido,
  p.status,
  p.total
FROM pedidos p
INNER JOIN clientes c ON c.id = p.cliente_id
ORDER BY p.criado_em DESC;
SQL — LEFT JOIN (inclui quem nunca comprou)
SELECT
  c.nome,
  COUNT(p.id) AS total_pedidos
FROM clientes c
LEFT JOIN pedidos p ON p.cliente_id = c.id
GROUP BY c.id, c.nome
ORDER BY total_pedidos DESC;
SQL — JOIN com 3 tabelas
-- Itens de um pedido específico com nome do produto
SELECT
  c.nome                              AS cliente,
  pr.nome                             AS produto,
  i.quantidade,
  i.preco_unit,
  (i.quantidade * i.preco_unit)       AS subtotal
FROM clientes c
INNER JOIN pedidos p      ON p.cliente_id = c.id
INNER JOIN itens_pedido i ON i.pedido_id  = p.id
INNER JOIN produtos pr    ON pr.id        = i.produto_id
WHERE p.id = 5;
GROUP BY + HAVING
Diferença crucial: WHERE filtra linhas antes de agrupar. HAVING filtra depois de agrupar, e só aceita condições sobre funções de agregação (COUNT, SUM, AVG...).
SQL — GROUP BY + HAVING
-- Total gasto por cliente — só quem fez 2+ pedidos entregues
SELECT
  c.nome,
  COUNT(p.id)      AS qtd_pedidos,
  AVG(p.total)     AS ticket_medio,
  SUM(p.total)     AS total_gasto
FROM clientes c
INNER JOIN pedidos p ON p.cliente_id = c.id
WHERE p.status = 'entregue'       -- filtra ANTES de agrupar
GROUP BY c.id, c.nome
HAVING COUNT(p.id) >= 2           -- filtra DEPOIS de agrupar
ORDER BY total_gasto DESC;

6 Exercícios — do básico ao desafio
Tente resolver cada exercício antes de ver o gabarito. O gabarito é uma possibilidade — podem existir outras consultas igualmente corretas.
Exercício 01

Liste todos os produtos da categoria 'Eletrônicos' com preço acima de R$ 200, ordenados do mais barato para o mais caro.

Básico
SQL
SELECT nome, preco
FROM produtos
WHERE categoria = 'Eletrônicos' AND preco > 200
ORDER BY preco ASC;
Exercício 02

Mostre o nome de todos os clientes que não têm email cadastrado.

Básico
SQL
SELECT nome FROM clientes WHERE email IS NULL;
Exercício 03

Liste o nome do cliente e o status de todos os pedidos. Inclua também os clientes que nunca fizeram pedido (eles devem aparecer com status NULL).

Básico
SQL
SELECT c.nome, p.status
FROM clientes c
LEFT JOIN pedidos p ON p.cliente_id = c.id;
Exercício 04

Qual o total gasto por cada cliente? Mostre nome, quantidade de pedidos e soma total. Considere apenas pedidos com status 'entregue'. Ordene do maior para o menor gasto.

Médio
SQL
SELECT
  c.nome,
  COUNT(p.id)  AS qtd_pedidos,
  SUM(p.total) AS total_gasto
FROM clientes c
INNER JOIN pedidos p ON p.cliente_id = c.id
WHERE p.status = 'entregue'
GROUP BY c.id, c.nome
ORDER BY total_gasto DESC;
Exercício 05

Liste os produtos que aparecem em pelo menos um pedido, com o nome do produto e a quantidade total vendida (soma de todos os itens). Ordene do mais vendido para o menos vendido.

Médio
SQL
SELECT
  pr.nome,
  SUM(i.quantidade) AS qtd_vendida
FROM produtos pr
INNER JOIN itens_pedido i ON i.produto_id = pr.id
GROUP BY pr.id, pr.nome
ORDER BY qtd_vendida DESC;
Exercício 06 — Desafio

Qual cliente comprou o produto mais caro? Mostre o nome do cliente, o nome do produto e o valor pago. Use 3 JOINs.

Desafio
SQL
SELECT
  c.nome  AS cliente,
  pr.nome AS produto,
  i.preco_unit AS valor_pago
FROM clientes c
INNER JOIN pedidos p      ON p.cliente_id = c.id
INNER JOIN itens_pedido i ON i.pedido_id  = p.id
INNER JOIN produtos pr    ON pr.id        = i.produto_id
ORDER BY i.preco_unit DESC
LIMIT 1;

Etapa 6 · Cardinalidades •⏱ ~2 min

Cardinalidade: quantas ocorrências participam?

1:1 — um para um · 1:N — um para muitos · N:N — muitos para muitos.

Pergunta rápida

Um curso pode possuir quantos alunos?

Curso 1:N Aluno

Um curso pode ter vários alunos matriculados, mas cada aluno pertence a apenas um curso — por isso a cardinalidade é um para muitos (1:N).

1:1

Pessoa e CPF — cada pessoa possui exatamente um CPF, e vice-versa.

1:N

Curso e Alunos — um curso reúne vários alunos.

N:N

Alunos e Disciplinas — resolvido pela entidade associativa Matrícula.

Etapa 10 · Atividade para casa •⏱ ~2 min

Atividade — modele um sistema de biblioteca

Enunciado

Uma biblioteca precisa armazenar informações sobre livros, autores, usuários e empréstimos.

  1. Identifique pelo menos quatro entidades.
  2. Defina pelo menos três atributos para cada entidade.
  3. Identifique os relacionamentos entre as entidades.
  4. Defina as cardinalidades de cada relacionamento.
  5. Desenhe um pequeno Diagrama Entidade-Relacionamento.

Critérios esperados

  • Clareza na identificação das entidades
  • Coerência dos atributos escolhidos
  • Relacionamentos compatíveis com o problema
  • Cardinalidades justificadas
  • Organização visual do modelo

Pense no empréstimo como o elemento que conecta o usuário ao livro.

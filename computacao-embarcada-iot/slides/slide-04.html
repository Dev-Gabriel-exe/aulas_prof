Sensores — e o DHT22 de temperatura e umidade
O que é um sensor — a ponte entre mundos

Um sensor é um dispositivo que converte uma grandeza física do mundo real — temperatura, pressão, luz, distância, movimento — em um sinal elétrico que o microcontrolador consegue ler. Sem sensores, um sistema embarcado é cego: executa lógica, mas não percebe o ambiente.

A distinção mais importante é entre sensores analógicos e sensores digitais:

TipoComo funcionaComo ler no ArduinoExemplo
Analógico Varia a tensão de saída proporcionalmente à grandeza medida (0–5V) analogRead(A0) → 0 a 1023 LDR (luz), NTC (temperatura), potenciômetro
Digital simples Sinal HIGH ou LOW — presente ou ausente digitalRead(pino) → HIGH/LOW PIR (movimento), reed switch, fim-de-curso
Digital com protocolo Envia dados codificados por um barramento (1-Wire, I2C, SPI) Biblioteca específica — ex: dht.readTemperature() DHT22, BMP280, MPU6050
Por que sensores digitais com protocolo são melhores? Eles já fazem a conversão internamente — entregam o valor em °C ou %RH direto, sem você precisar converter tensão em grandeza física. Além disso, são imunes a ruído elétrico no cabo, pois transmitem dados codificados.

Resistor pull-up obrigatório. O pino DATA do DHT22 usa protocolo 1-Wire — ele precisa de um resistor de 10kΩ entre DATA e VCC para funcionar corretamente. Sem ele, as leituras serão NaN (Not a Number) ou completamente erradas. A maioria dos módulos DHT22 de protoboard já tem esse resistor embutido — verifique no seu módulo.
Instalando a biblioteca e primeiro código

O DHT22 usa um protocolo serial proprietário — comunicação de 1-Wire com timing específico. Implementar esse protocolo do zero seria complexo. Por isso usamos a biblioteca DHT sensor library da Adafruit, que abstrai tudo isso.

1
Instalar a biblioteca
No Arduino IDE: Sketch → Include Library → Manage Libraries. Busque por "DHT sensor library" e instale a versão da Adafruit. Instale também a dependência "Adafruit Unified Sensor" quando solicitado.
2
Montar o circuito
VCC → 5V, DATA → pino 2 (com resistor 10kΩ entre DATA e 5V), GND → GND. Se estiver usando módulo de 3 pinos, o resistor já está na placa.
Arduino C — DHT22 básico
#include <DHT.h>

#define PINO_DHT  2
#define TIPO_DHT  DHT22   // ou DHT11 se for o modelo mais simples

DHT dht(PINO_DHT, TIPO_DHT);

void setup() {
  Serial.begin(9600);
  dht.begin();
  Serial.println("DHT22 iniciado. Aguarde 2s para primeira leitura...");
  delay(2000);  // DHT22 precisa de ~2s para estabilizar
}

void loop() {
  float temp   = dht.readTemperature();      // °C
  float umid   = dht.readHumidity();         // %RH
  float tempF  = dht.readTemperature(true); // °F (parâmetro true)

  // isnan() detecta leitura inválida (NaN = Not a Number)
  if (isnan(temp) || isnan(umid)) {
    Serial.println("ERRO: falha na leitura do DHT22!");
    Serial.println("Verifique a ligação e o resistor pull-up.");
    delay(2000);
    return;   // pula o resto do loop
  }

  // Índice de calor (sensação térmica)
  float ic = dht.computeHeatIndex(temp, umid, false);

  Serial.println("================================");
  Serial.print("Temperatura : "); Serial.print(temp, 1); Serial.println(" °C");
  Serial.print("Umidade     : "); Serial.print(umid, 1); Serial.println(" %RH");
  Serial.print("Sensação    : "); Serial.print(ic,   1); Serial.println(" °C");
  Serial.println("================================");

  delay(2000); // DHT22: mínimo 2s entre leituras
}
Sempre verifique isnan(). Se a leitura falhar (cabo solto, resistor ausente, timing incorreto), readTemperature() retorna NaN. Sem verificação, NaN se propaga por todos os cálculos e você recebe valores absurdos como -127.0°C sem saber por quê.

HC-SR04 — sensor ultrassônico de distância
Como funciona o HC-SR04

O HC-SR04 mede distância usando ultrassom: emite um pulso sonoro acima da faixa audível (40 kHz), aguarda o eco refletir no objeto alvo, e mede o tempo entre o envio e o retorno. Com a velocidade do som conhecida, calcula-se a distância.

O sensor tem 4 pinos: VCC (5V), GND, TRIG (trigger — você envia o pulso) e ECHO (o sensor retorna o tempo de voo). A comunicação é por pulso digital — sem protocolo especial, sem biblioteca obrigatória.

Pino ECHO em 5V — cuidado com ESP32! O HC-SR04 opera em 5V e o pino ECHO retorna 5V. O ESP32 suporta apenas 3.3V nos pinos GPIO. Use um divisor de tensão (resistores 1kΩ + 2kΩ) ou um módulo HC-SR04 de 3.3V se usar com ESP32.
Código completo — HC-SR04 sem biblioteca

O HC-SR04 não precisa de biblioteca. Você controla diretamente com digitalWrite() para o trigger e pulseIn() para medir o tempo do echo.

Arduino C — HC-SR04 completo
#define PINO_TRIG  9
#define PINO_ECHO  10

void setup() {
  Serial.begin(9600);
  pinMode(PINO_TRIG, OUTPUT);
  pinMode(PINO_ECHO, INPUT);
}

long medirDistancia() {
  // 1. Garante nível baixo antes do disparo
  digitalWrite(PINO_TRIG, LOW);
  delayMicroseconds(2);

  // 2. Pulso HIGH de 10µs → dispara a medição
  digitalWrite(PINO_TRIG, HIGH);
  delayMicroseconds(10);
  digitalWrite(PINO_TRIG, LOW);

  // 3. pulseIn() mede o tempo do echo em microssegundos
  //    timeout de 30ms (objetos até ~500cm)
  long duracao = pulseIn(PINO_ECHO, HIGH, 30000);

  // 4. Converte para centímetros
  //    velocidade do som ≈ 0.034 cm/µs, ida e volta ÷ 2
  long distancia = duracao * 0.034 / 2;

  return distancia;
}

void loop() {
  long dist = medirDistancia();

  if (dist == 0 || dist > 400) {
    Serial.println("Fora de alcance (>400cm ou sem eco)");
  } else {
    Serial.print("Distância: ");
    Serial.print(dist);
    Serial.println(" cm");
  }
  delay(200); // 5 leituras por segundo
}
pulseIn(pino, HIGH, timeout) aguarda o pino ir para HIGH, começa a contar, aguarda voltar para LOW e retorna o tempo em microssegundos. O terceiro parâmetro (timeout) evita que o programa trave indefinidamente se não houver eco — retorna 0 quando expira.

Calibração — por que todo sensor mente um pouco
O que é calibração e por que é necessária

Nenhum sensor é perfeitamente preciso. Cada unidade fabricada tem tolerâncias ligeiramente diferentes. A temperatura ambiente afeta o comportamento. O cabo introduz resistência. A tensão de alimentação não é exatamente 5.000V.

Calibrar é o processo de comparar as leituras do seu sensor com uma referência conhecida e aplicar uma correção matemática. Em projetos profissionais é uma etapa obrigatória antes de qualquer deploy.

Passo 1
Obter uma referência confiável
Para temperatura: termômetro de mercúrio calibrado, ou app de clima local (para referência aproximada). Para distância: régua ou trena metálica precisa.
Passo 2
Coletar múltiplas amostras
Nunca calibre com uma única leitura. Colete 10–20 amostras, calcule a média. Valores outlier (muito fora da média) devem ser descartados.
Passo 3
Calcular o offset
Offset = valor_referência − valor_sensor. Se o sensor lê 24.3°C e o termômetro marca 25.0°C, o offset é +0.7°C. Aplique: temp_corrigida = temp_sensor + 0.7.
Passo 4
Verificar em múltiplos pontos
Um sensor pode ter offset diferente em faixas diferentes de temperatura. Para maior precisão, calibre em 2–3 pontos e interpole linearmente.
Arduino C — DHT22 com calibração e média
#include <DHT.h>

#define PINO_DHT  2
#define TIPO_DHT  DHT22
#define N_AMOSTRAS 10

// Offsets de calibração — ajuste após comparar com referência
const float OFFSET_TEMP = 0.0;   // ex: +0.7 se sensor lê 0.7°C abaixo
const float OFFSET_UMID = 0.0;   // ex: -3.0 se sensor lê 3% acima

DHT dht(PINO_DHT, TIPO_DHT);

float mediaTemperatura() {
  float soma = 0; int validas = 0;
  for (int i = 0; i < N_AMOSTRAS; i++) {
    float t = dht.readTemperature();
    if (!isnan(t)) { soma += t; validas++; }
    delay(200);
  }
  return validas > 0 ? (soma / validas) + OFFSET_TEMP : NAN;
}

float mediaUmidade() {
  float soma = 0; int validas = 0;
  for (int i = 0; i < N_AMOSTRAS; i++) {
    float u = dht.readHumidity();
    if (!isnan(u)) { soma += u; validas++; }
    delay(200);
  }
  return validas > 0 ? (soma / validas) + OFFSET_UMID : NAN;
}

void setup() {
  Serial.begin(9600);
  dht.begin();
  delay(2000);
}

void loop() {
  Serial.println("Coletando 10 amostras...");
  float temp = mediaTemperatura();
  float umid = mediaUmidade();

  if (isnan(temp) || isnan(umid)) {
    Serial.println("Erro de leitura!"); return;
  }

  Serial.print("Temp (calibrada): "); Serial.print(temp, 2); Serial.println("°C");
  Serial.print("Umid (calibrada): "); Serial.print(umid, 2); Serial.println("%");
  delay(5000);
}
LAB — DHT22 + HC-SR04 integrados
Objetivo do LAB: um programa único lê temperatura, umidade e distância, e exibe tudo no Serial Monitor de forma organizada. Use millis() para não bloquear o loop.
1
Montar ambos os sensores
DHT22: DATA no pino 2, pull-up 10kΩ. HC-SR04: TRIG no pino 9, ECHO no pino 10. Ambos alimentados em 5V e GND.
2
Testar cada sensor separadamente
Antes de integrar, verifique que cada sensor funciona individualmente. Muito mais fácil encontrar problemas assim do que depurar dois sensores ao mesmo tempo.
3
Integrar no loop com millis()
DHT22 lê a cada 3s (mínimo 2s). HC-SR04 lê a cada 500ms. Use dois contadores millis() independentes — não delay(). O Serial Monitor deve mostrar as duas fontes de dados continuamente.
4
Calibrar o HC-SR04
Posicione um objeto a distâncias conhecidas (10cm, 20cm, 50cm) medidas com régua. Anote o valor lido pelo sensor e calcule o offset. A velocidade do som varia com temperatura — tente compensar usando a leitura do DHT22.
Arduino C — DHT22 + HC-SR04 integrados com millis()
#include <DHT.h>

#define PINO_DHT   2
#define TIPO_DHT   DHT22
#define PINO_TRIG  9
#define PINO_ECHO  10

DHT dht(PINO_DHT, TIPO_DHT);

unsigned long ultimaLeituraDHT  = 0;
unsigned long ultimaLeituraSonar = 0;
float tempAtual = 0, umidAtual = 0;

long lerDistancia(float tempC) {
  // Compensação de temperatura: v_som = 331.3 + 0.606 × T (m/s)
  float vSom = (331.3 + (0.606 * tempC)) / 10000.0; // cm/µs

  digitalWrite(PINO_TRIG, LOW);  delayMicroseconds(2);
  digitalWrite(PINO_TRIG, HIGH); delayMicroseconds(10);
  digitalWrite(PINO_TRIG, LOW);

  long dur = pulseIn(PINO_ECHO, HIGH, 30000);
  return (dur > 0) ? (long)(dur * vSom / 2) : -1;
}

void setup() {
  Serial.begin(9600);
  dht.begin();
  pinMode(PINO_TRIG, OUTPUT);
  pinMode(PINO_ECHO, INPUT);
  delay(2000);
}

void loop() {
  unsigned long agora = millis();

  // DHT22: a cada 3 segundos
  if (agora - ultimaLeituraDHT >= 3000) {
    ultimaLeituraDHT = agora;
    float t = dht.readTemperature();
    float u = dht.readHumidity();
    if (!isnan(t) && !isnan(u)) {
      tempAtual = t; umidAtual = u;
      Serial.print("[DHT22] Temp: ");  Serial.print(t, 1);
      Serial.print("°C | Umid: ");       Serial.print(u, 1);
      Serial.println("%");
    }
  }

  // HC-SR04: a cada 500ms
  if (agora - ultimaLeituraSonar >= 500) {
    ultimaLeituraSonar = agora;
    long dist = lerDistancia(tempAtual > 0 ? tempAtual : 25.0);
    if (dist > 0 && dist <= 400) {
      Serial.print("[HC-SR04] Distância: "); Serial.print(dist); Serial.println(" cm");
    } else {
      Serial.println("[HC-SR04] Fora de alcance");
    }
  }
}
Compensação de temperatura no HC-SR04: a velocidade do som muda com a temperatura — a 0°C é ~331 m/s, a 25°C é ~346 m/s. Usar a temperatura do DHT22 para compensar a fórmula do ultrassônico aumenta a precisão da medição de distância em condições extremas.

DHT22

Protocolo 1-Wire, pull-up 10kΩ obrigatório, isnan() para verificar erro, mínimo 2s entre leituras, float para temperatura.

HC-SR04

Calibração

Todo sensor tem erro sistemático. Offset = referência − sensor. Média de amostras reduz ruído. Velocidade do som varia com temperatura.

ESP32 — o microcontrolador IoT e a conexão Wi-Fi
Por que o ESP32 — além do Wi-Fi

O Arduino Uno é excelente para aprender, mas tem limitações claras: 2KB de RAM, sem conectividade e um único núcleo a 16MHz. O ESP32, lançado pela Espressif em 2016, resolveu todos esses pontos de uma vez — e se tornou o hardware padrão de projetos IoT do mundo todo.

Não é só o Wi-Fi que diferencia: o ESP32 tem dois núcleos (você pode rodar a lógica de rede em um núcleo e a leitura de sensores no outro), 520KB de RAM, ADC de 12 bits, DAC, touchpads, motor PWM dedicado, e suporte a BLE 4.2 além do Wi-Fi 802.11 b/g/n. Tudo isso num chip que custa menos que uma coxinha.

ATENÇÃO — tensão dos pinos: os GPIOs do ESP32 operam em 3.3V e não são tolerantes a 5V. Ligar um pino do Arduino Uno (5V) diretamente num GPIO do ESP32 pode danificar o chip permanentemente. Use divisores de tensão ou level shifters quando necessário. O DHT22 funciona em 3.3V — sem problemas.

Conectando ao Wi-Fi — WiFi.h por dentro

A biblioteca WiFi.h já vem inclusa no pacote ESP32 para Arduino. A conexão básica envolve três passos: fornecer SSID e senha, aguardar o status WL_CONNECTED e depois usar o IP obtido.

RSSI: -58 dBm
Modos Wi-Fi do ESP32:
STA (Station) — conecta a um roteador existente. É o modo que usamos hoje.
AP (Access Point) — o ESP32 vira o roteador, outros dispositivos conectam nele.
AP+STA — os dois ao mesmo tempo. Útil para configuração remota.
Arduino C — Conexão Wi-Fi com reconexão automática
#include <WiFi.h>

const char* SSID     = "NomeDaRede";    // substitua
const char* SENHA    = "SenhaDaRede";   // substitua

void conectarWifi() {
  WiFi.mode(WIFI_STA);               // modo Station (cliente)
  WiFi.begin(SSID, SENHA);

  Serial.print("Conectando a \"");
  Serial.print(SSID); Serial.print("\"");

  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }

  Serial.println("\nConectado!");
  Serial.print("IP: ");
  Serial.println(WiFi.localIP());
  Serial.print("RSSI: ");
  Serial.print(WiFi.RSSI()); Serial.println(" dBm");
}

void setup() {
  Serial.begin(115200);
  conectarWifi();
}

void loop() {
  // Reconecta automaticamente se cair a conexão
  if (WiFi.status() != WL_CONNECTED) {
    Serial.println("Wi-Fi perdido. Reconectando...");
    conectarWifi();
  }
  delay(10000);
}
Nunca suba credenciais Wi-Fi no GitHub. Use um arquivo separado config.h listado no .gitignore, ou variáveis de ambiente. Em produção, use o mecanismo de provisioning do ESP32 (SmartConfig ou Wi-Fi Manager).

Protocolo HTTP — GET, POST e ArduinoJson
HTTP por dentro — o protocolo que move a web

HTTP (HyperText Transfer Protocol) é o protocolo de comunicação da web. Toda vez que seu navegador abre um site, ele manda uma requisição HTTP para um servidor e recebe uma resposta. No ESP32, faremos exatamente isso — só que quem "abre o site" é o microcontrolador.

O HTTP funciona em modelo request-response: o cliente (ESP32) sempre inicia, o servidor sempre responde. Cada resposta tem um código de status que indica o resultado, e um body com o conteúdo.

HTTPClient GET — buscando dados de uma API

A classe HTTPClient abstrai todo o HTTP no ESP32 — abertura de conexão, envio dos headers, leitura da resposta. Para um GET simples, você fornece a URL e lê o response body como String.

Arduino C — HTTP GET para API pública
#include <WiFi.h>
#include <HTTPClient.h>

// Após conectarWifi() no setup()...

void fazerGet() {
  HTTPClient http;

  // Inicia a conexão com a URL alvo
  http.begin("http://192.168.1.100:5000/api/status");

  int httpCode = http.GET();   // envia a requisição

  if (httpCode > 0) {
    Serial.print("Código HTTP: "); Serial.println(httpCode);

    if (httpCode == HTTP_CODE_OK) {          // 200
      String payload = http.getString();   // lê o body
      Serial.println("Resposta: " + payload);
    }
  } else {
    Serial.print("Erro na requisição: ");
    Serial.println(http.errorToString(httpCode));
  }

  http.end();   // SEMPRE liberar a conexão
}
http.end() é obrigatório. Se você não chamar, a conexão fica aberta e o ESP32 esgota os sockets disponíveis após algumas requisições — causando falhas silenciosas. Sempre coloque em um bloco que garanta a execução, mesmo em caso de erro.
HTTPClient POST com JSON — ArduinoJson

Para enviar dados de sensores, precisamos montar um JSON e enviá-lo no body do POST. A biblioteca ArduinoJson (instalar via Gerenciador de Bibliotecas: "ArduinoJson by Benoit Blanchon") torna isso seguro e eficiente — sem concatenação manual de strings que vaza memória.

Arduino C — HTTP POST com JSON (ArduinoJson)
#include <WiFi.h>
#include <HTTPClient.h>
#include <ArduinoJson.h>

const char* URL_SERVIDOR = "http://192.168.1.100:5000/api/dados";

bool enviarDados(float temperatura, float umidade) {
  if (WiFi.status() != WL_CONNECTED) {
    Serial.println("Wi-Fi desconectado — abortando POST");
    return false;
  }

  // Monta o JSON com ArduinoJson
  StaticJsonDocument<128> doc;
  doc["temperatura"] = temperatura;
  doc["umidade"]     = umidade;
  doc["dispositivo"]  = "esp32-lab";

  char jsonBuffer[128];
  serializeJson(doc, jsonBuffer);
  // jsonBuffer agora contém: {"temperatura":27.3,"umidade":64.8,"dispositivo":"esp32-lab"}

  HTTPClient http;
  http.begin(URL_SERVIDOR);
  http.addHeader("Content-Type", "application/json");

  int httpCode = http.POST(jsonBuffer);

  bool sucesso = false;
  if (httpCode == HTTP_CODE_OK || httpCode == 201) {
    Serial.print("Enviado! HTTP "); Serial.println(httpCode);
    sucesso = true;
  } else {
    Serial.print("Erro HTTP: "); Serial.println(httpCode);
  }

  http.end();
  return sucesso;
}
Pergunta para a turma: A função enviarDados() retorna um bool indicando sucesso. Como você usaria isso no loop() para tentar novamente em caso de falha — sem bloquear o programa com delay()?
🙋 O ESP32 executa o POST mas o servidor nunca recebe nada. O Serial Monitor mostra "Erro HTTP: -1". Qual é a causa mais provável?
A
O JSON está malformado — o servidor rejeita o body
B
O ESP32 não conseguiu conectar ao servidor — IP/porta errado, servidor não está rodando, ou estão em redes diferentes
C
A biblioteca HTTPClient não suporta POST — use apenas GET com parâmetros na URL
D
O código HTTP -1 significa que o servidor respondeu 401 Unauthorized

Python — Servidor Flask (executar no computador)
# servidor_iot.py — executar no computador com: python servidor_iot.py
# pip install flask

from flask import Flask, request, jsonify
from datetime import datetime

app = Flask(__name__)
leituras = []   # armazena os dados recebidos em memória

@app.route('/api/dados', methods=['POST'])
def receber_dados():
    dados = request.get_json()
    if not dados:
        return jsonify({'erro': 'JSON inválido'}), 400

    dados['timestamp'] = datetime.now().strftime('%H:%M:%S')
    leituras.append(dados)

    print(f"[{dados['timestamp']}] T={dados.get('temperatura','?')}°C  "
          f"U={dados.get('umidade','?')}%  de={dados.get('dispositivo','?')}")

    return jsonify({'status': 'salvo', 'total': len(leituras)}), 200

@app.route('/api/dados', methods=['GET'])
def listar_dados():
    return jsonify(leituras)

if __name__ == '__main__':
    print("Servidor IoT rodando em http://0.0.0.0:5000")
    app.run(host='0.0.0.0', port=5000, debug=True)
Arduino C — ESP32 + DHT22 → API REST completo
#include <WiFi.h>
#include <HTTPClient.h>
#include <ArduinoJson.h>
#include <DHT.h>

// ── Configurações ── substitua antes de compilar
const char* SSID        = "NomeDaRede";
const char* SENHA       = "SenhaDaRede";
const char* URL_SERVIDOR = "http://192.168.1.100:5000/api/dados";

#define PINO_DHT   4
#define TIPO_DHT   DHT22
#define INTERVALO  10000UL   // 10 segundos em ms

DHT dht(PINO_DHT, TIPO_DHT);
unsigned long ultimoEnvio = 0;

void conectarWifi() {
  WiFi.mode(WIFI_STA);
  WiFi.begin(SSID, SENHA);
  Serial.print("Conectando ao Wi-Fi");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500); Serial.print(".");
  }
  Serial.println("\nConectado! IP: " + WiFi.localIP().toString());
}

bool enviarDados(float t, float u) {
  HTTPClient http;
  http.begin(URL_SERVIDOR);
  http.addHeader("Content-Type", "application/json");
  http.setTimeout(5000);   // 5s timeout — não trava para sempre

  StaticJsonDocument<128> doc;
  doc["temperatura"] = t;
  doc["umidade"]     = u;
  doc["dispositivo"]  = "esp32-lab";

  char buf[128];
  serializeJson(doc, buf);

  int code = http.POST(buf);
  http.end();

  if (code == HTTP_CODE_OK || code == 201) {
    Serial.printf("[OK] T=%.1f°C U=%.1f%% → HTTP %d\n", t, u, code);
    return true;
  }
  Serial.printf("[ERRO] HTTP %d\n", code);
  return false;
}

void setup() {
  Serial.begin(115200);
  dht.begin();
  conectarWifi();
  delay(2000);   // estabilizar o DHT22
}

void loop() {
  // Reconexão automática
  if (WiFi.status() != WL_CONNECTED) {
    Serial.println("Wi-Fi perdido — reconectando...");
    conectarWifi();
  }

  if (millis() - ultimoEnvio >= INTERVALO) {
    ultimoEnvio = millis();

    float t = dht.readTemperature();
    float u = dht.readHumidity();

    if (isnan(t) || isnan(u)) {
      Serial.println("[ERRO] Falha na leitura do DHT22");
      return;
    }

    enviarDados(t, u);
  }
}

Exercício 01 — Diagnóstico de Wi-Fi

Adicione ao código da conexão Wi-Fi um diagnóstico completo que imprime no Serial Monitor: (a) o SSID ao qual está conectado, (b) o endereço IP local, (c) o RSSI em dBm com uma classificação textual ("Excelente" acima de -50, "Bom" até -70, "Fraco" abaixo de -70). Use WiFi.SSID(), WiFi.localIP() e WiFi.RSSI().

Conceito + LAB
Arduino C — diagnóstico de Wi-Fi
void diagnosticoWifi() {
  Serial.println("=== Diagnóstico Wi-Fi ===");
  Serial.print("SSID: ");  Serial.println(WiFi.SSID());
  Serial.print("IP:   ");  Serial.println(WiFi.localIP());

  int rssi = WiFi.RSSI();
  String qualidade;
  if      (rssi > -50) qualidade = "Excelente";
  else if (rssi > -70) qualidade = "Bom";
  else                 qualidade = "Fraco";

  Serial.printf("RSSI: %d dBm (%s)\n", rssi, qualidade.c_str());
}
Exercício 02 — Contador de falhas de envio

Modifique a função enviarDados() para manter um contador global de falhas consecutivas. Se o POST falhar 3 vezes seguidas, o ESP32 deve se desconectar do Wi-Fi com WiFi.disconnect() e reconectar. Se o POST for bem-sucedido, o contador deve ser zerado. Isso simula uma estratégia básica de recuperação de falhas em campo.

LAB
Trecho — contador de falhas
int falhasConsecutivas = 0;

// No loop(), ao chamar enviarDados():
if (enviarDados(t, u)) {
  falhasConsecutivas = 0;   // sucesso — zera
} else {
  falhasConsecutivas++;
  Serial.printf("Falhas consecutivas: %d\n", falhasConsecutivas);

  if (falhasConsecutivas >= 3) {
    Serial.println("3 falhas — resetando Wi-Fi...");
    WiFi.disconnect();
    delay(1000);
    conectarWifi();
    falhasConsecutivas = 0;
  }
}
Exercício 03 — Resposta do servidor no Serial Monitor

No código do ESP32, após um POST bem-sucedido (HTTP 200), leia o body da resposta com http.getString() e extraia com ArduinoJson o campo "total" (número de leituras armazenadas pelo servidor). Imprima no Serial Monitor: "Total de leituras no servidor: X". Isso cria um loop de confirmação — o ESP32 sabe que os dados chegaram.

Integração
Trecho — lendo resposta do servidor
// Dentro de enviarDados(), após verificar httpCode == 200:
if (code == HTTP_CODE_OK) {
  String resposta = http.getString();

  StaticJsonDocument<64> resp;
  DeserializationError err = deserializeJson(resp, resposta);

  if (!err) {
    int total = resp["total"] | 0;   // | 0 = valor padrão se campo ausente
    Serial.printf("Total de leituras no servidor: %d\n", total);
  }
}
Exercício 04 — Desafio: endpoint GET para consulta

No servidor Flask, já existe o endpoint GET /api/dados que retorna todas as leituras. Implemente no ESP32 uma função consultarUltima() que faz um GET para esse endpoint, faz parse do JSON de resposta e imprime apenas a última leitura recebida (temperatura, umidade e timestamp). Execute ela uma vez no setup(), após conectar ao Wi-Fi, para mostrar os dados mais recentes disponíveis antes de começar a enviar.

Desafio
Arduino C — GET + parse do array de leituras
void consultarUltima() {
  HTTPClient http;
  http.begin(URL_SERVIDOR);
  http.setTimeout(5000);

  int code = http.GET();
  if (code != HTTP_CODE_OK) {
    Serial.println("GET falhou ou servidor vazio");
    http.end(); return;
  }

  String payload = http.getString();
  http.end();

  // Array de objetos — tamanho dinâmico; estimativa conservadora
  DynamicJsonDocument doc(2048);
  DeserializationError err = deserializeJson(doc, payload);

  if (err || doc.size() == 0) {
    Serial.println("Nenhum dado no servidor ainda."); return;
  }

  // Pega o último elemento do array
  JsonObject ultima = doc[doc.size() - 1];
  Serial.printf("Última leitura → T=%.1f°C U=%.1f%% @ %s\n",
    (float)ultima["temperatura"],
    (float)ultima["umidade"],
    (const char*)ultima["timestamp"]);
}

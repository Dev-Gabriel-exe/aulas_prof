MCU vs Microprocessador — e os três players do semestre
Microcontrolador vs Microprocessador — a diferença fundamental

Esta é uma confusão clássica. Os dois são chips que executam instruções, mas foram projetados para contextos completamente diferentes.

O que éCPU + RAM + Flash + periféricos no mesmo chipApenas a CPU — depende de chips externos para RAM, armazenamento e I/O
CustoR$ 5 – R$ 60R$ 60 – R$ 500+
RAM2 KB – 512 KB512 MB – 16 GB (com módulo externo)
Clock16 MHz – 240 MHz1 GHz – 2.5 GHz
Sistema operacionalBare-metal ou RTOSLinux, Android, Windows IoT
ConsumoµA – mA (funciona com bateria AA por meses)mA – A (precisa de fonte de alimentação estável)
ExemplosATmega328 ESP32 RP2040Raspberry Pi 4 BeagleBone
Regra prática: se precisa de Linux, use microprocessador. Se precisa de baixo consumo, custo baixo e controle direto de hardware em tempo real, use microcontrolador. As duas categorias coexistem — muitos projetos IoT usam um MCU para coletar dados e um MPU para processar e publicar.
Pergunta para a turma: O Raspberry Pi 4 (aquele computadorzinho do tamanho de um cartão de crédito) é um microcontrolador ou um microprocessador? E o ESP32? Justifique antes de avançar.

Pinout e Datasheet — entendendo o mapa da placa
O que é pinout?

Pinout é o mapa de todos os pinos de uma placa ou chip — cada pino tem um número, um nome e uma função. Saber ler um pinout é fundamental: é a diferença entre conectar corretamente um sensor e queimar o hardware.

Cada pino pode ter múltiplas funções dependendo da configuração no código. Um mesmo pino pode ser entrada digital, saída PWM ou parte de um barramento I2C — você define no setup().

Regra de ouro: nunca conecte um pino de saída direto no GND ou nos 5V sem um resistor de proteção. Um LED ligado diretamente sem resistor vai queimar em segundos. Um resistor de 220Ω ou 330Ω entre o pino e o LED é o mínimo.
Como ler um datasheet

O datasheet é o documento técnico oficial do fabricante de um chip ou componente. É sua fonte de verdade — mais confiável que qualquer tutorial no YouTube. Parece intimidador no começo, mas você não precisa ler tudo: só as seções relevantes.

O que procurar primeiro

  • Absolute Maximum Ratings — os limites que você NUNCA pode ultrapassar. Tensão máxima, corrente máxima, temperatura.
  • Pinout diagram — o mapa dos pinos com todas as funções alternativas.
  • Electrical Characteristics — tensão de operação, corrente típica, consumo em sleep mode.
  • Communication interfaces — quais protocolos suporta (SPI, I2C, UART) e como configurar.

Datasheets para conhecer

  • ATmega328P — MCU do Arduino Uno (microchip.com)
  • ESP32 — Espressif Systems (espressif.com/en/products/socs/esp32)
  • RP2040 — Raspberry Pi Foundation (datasheets.raspberrypi.com)
  • DHT22 — sensor que usaremos na Semana 4
Exemplo prático: no datasheet do ATmega328P, a seção "Absolute Maximum Ratings" diz que a tensão máxima em qualquer pino é 6V. O datasheet do ESP32 diz 3.6V. Se você conectar um sensor de 5V num pino do ESP32 sem divisor de tensão, ultrapassou o limite e o chip pode ser danificado permanentemente.

Hello World Embarcado — o Blink
Aula de laboratório. Abra o Arduino IDE, pegue a placa e siga o passo a passo. O objetivo é simples: fazer o LED embutido piscar. Parece pouco — mas esse é o momento em que o seu código sai do computador e entra no mundo físico pela primeira vez.
Passo a passo — do zero ao LED piscando
1
Instalar o Arduino IDE
Acesse arduino.cc/en/software e baixe a versão 2.x para o seu sistema operacional. Durante a instalação, aceite a instalação dos drivers USB — eles são necessários para a placa ser reconhecida.
2
Conectar a placa
Conecte o Arduino Uno ao computador via cabo USB. O LED de energia (verde ou vermelho, marcado como "ON") deve acender. No Windows, o driver é instalado automaticamente. No macOS/Linux, normalmente é plug-and-play.
3
Selecionar a placa e a porta
No Arduino IDE: Tools → Board → Arduino AVR Boards → Arduino Uno. Depois: Tools → Port → COMx (Windows) ou /dev/ttyUSB0 ou /dev/tty.usbmodem (macOS/Linux). Se não aparecer porta, o driver não foi instalado — resolva antes de continuar.
4
Abrir o exemplo Blink
No IDE: File → Examples → 01.Basics → Blink. O código já vem pronto — leia cada linha antes de fazer upload. Entender o que está escrito é mais importante do que o LED piscar.
5
Fazer upload
Clique no botão de upload (→) ou pressione Ctrl+U. O IDE vai compilar (transformar código em binário) e enviar para a placa via USB. Durante o upload, os LEDs TX e RX da placa piscam rapidamente. Ao terminar, o LED do pino 13 começa a piscar a cada 1 segundo.
6
Modificar e experimentar
Troque os valores de delay() e faça upload novamente. Mude 1000 para 200. Mude para 50. Observe o comportamento. Esse ciclo — escrever, compilar, observar — é o ciclo de desenvolvimento embarcado que você vai repetir centenas de vezes esse semestre.
O código Blink — linha por linha
Arduino C — Blink comentado
/*
  Blink — Hello World do mundo embarcado
  Pisca o LED conectado ao pino 13 (LED embutido na placa)
  a cada 1 segundo.
*/

// LED_BUILTIN é uma constante definida pelo Arduino IDE
// No Uno ela vale 13 — no ESP32 vale 2 — no Mega vale 13
// Usar LED_BUILTIN em vez de 13 torna o código portável

void setup() {
  // pinMode configura o pino como INPUT ou OUTPUT
  // Antes de usar um pino digital, sempre configure-o
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);  // HIGH = 5V = LED aceso
  delay(1000);                       // aguarda 1000 ms = 1 segundo
  digitalWrite(LED_BUILTIN, LOW);   // LOW = 0V = LED apagado
  delay(1000);                       // aguarda 1 segundo
  // → o loop reinicia aqui — para sempre
}
Arduino C — Blink com Serial Monitor
// Versão melhorada: imprime no Serial Monitor
// Abra com Ctrl+Shift+M — velocidade 9600 baud

void setup() {
  Serial.begin(9600);           // inicia comunicação serial
  pinMode(LED_BUILTIN, OUTPUT);
  Serial.println("Arduino iniciado!");
}

int contador = 0;  // variável global — persiste entre loops

void loop() {
  contador++;
  Serial.print("Ciclo: ");
  Serial.println(contador);

  digitalWrite(LED_BUILTIN, HIGH);
  Serial.println("LED: ACESO");
  delay(1000);

  digitalWrite(LED_BUILTIN, LOW);
  Serial.println("LED: APAGADO");
  delay(1000);
}
Serial Monitor é seu melhor amigo. Em sistemas embarcados não existe console nem print bonito. O Serial Monitor é sua janela para o que está acontecendo dentro do chip. Aprenda a usar desde agora — você vai depender dele o semestre inteiro.
Erro mais comum nessa aula: "port not found" ou "device not recognized". Causas: driver não instalado, cabo USB com defeito (muitos cabos de carregador não têm os fios de dados), ou porta errada selecionada no IDE. Resolva um por vez antes de tentar de novo.

Exercício 01 — Blink com velocidade variável

Faça o LED piscar em três velocidades diferentes em sequência: rápido (100ms), médio (500ms) e lento (2000ms) — 3 piscadas de cada antes de mudar. Use o Serial Monitor para imprimir qual velocidade está ativa no momento.

Básico
Arduino C
void setup() {
  Serial.begin(9600);
  pinMode(LED_BUILTIN, OUTPUT);
}

void piscaNVezes(int intervalo, int vezes, String nome) {
  Serial.println("Velocidade: " + nome);
  for (int i = 0; i < vezes; i++) {
    digitalWrite(LED_BUILTIN, HIGH); delay(intervalo);
    digitalWrite(LED_BUILTIN, LOW);  delay(intervalo);
  }
}

void loop() {
  piscaNVezes(100,  3, "RAPIDO");
  piscaNVezes(500,  3, "MEDIO");
  piscaNVezes(2000, 3, "LENTO");
}
Exercício 02 — SOS em Morse

Implemente o sinal SOS em código Morse com o LED: · · · — — — · · · (ponto = 200ms, traço = 600ms, pausa entre letras = 400ms, pausa entre ciclos = 2000ms). Use o Serial Monitor para indicar quando começa cada letra.

Médio
Arduino C
#define PONTO  200
#define TRACO  600
#define PAUSA_LETRA  400
#define PAUSA_CICLO 2000

void setup() {
  Serial.begin(9600);
  pinMode(LED_BUILTIN, OUTPUT);
}

void flash(int duracao) {
  digitalWrite(LED_BUILTIN, HIGH); delay(duracao);
  digitalWrite(LED_BUILTIN, LOW);  delay(100);
}

void loop() {
  // S = . . .
  Serial.println("S");
  flash(PONTO); flash(PONTO); flash(PONTO);
  delay(PAUSA_LETRA);

  // O = - - -
  Serial.println("O");
  flash(TRACO); flash(TRACO); flash(TRACO);
  delay(PAUSA_LETRA);

  // S = . . .
  Serial.println("S");
  flash(PONTO); flash(PONTO); flash(PONTO);

  Serial.println("--- SOS completo ---");
  delay(PAUSA_CICLO);
}
Exercício 03 — Contador no Serial Monitor

Sem modificar o comportamento do LED, adicione ao código Blink um contador de ciclos e um temporizador de uptime: a cada ciclo, imprima no Serial Monitor o número do ciclo e quantos segundos se passaram desde que a placa foi ligada. Use a função millis() — pesquise como ela funciona.

Médio
Arduino C
int ciclo = 0;

void setup() {
  Serial.begin(9600);
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  ciclo++;
  unsigned long uptime = millis() / 1000;  // ms → segundos

  Serial.print("Ciclo #"); Serial.print(ciclo);
  Serial.print(" | Uptime: "); Serial.print(uptime);
  Serial.println("s");

  digitalWrite(LED_BUILTIN, HIGH); delay(1000);
  digitalWrite(LED_BUILTIN, LOW);  delay(1000);
}
Exercício 04 — Reflexão: por que delay() é problemático?

O delay() trava completamente o microcontrolador — durante o delay, nenhum outro código roda. Pesquise sobre a função millis() e explique, com suas palavras: como seria possível piscar o LED a cada 1 segundo e ao mesmo tempo verificar um botão continuamente, sem usar delay()? Escreva a lógica em pseudocódigo ou código real.

Desafio conceitual
Resposta — padrão millis() sem bloqueio

O problema do delay() é que ele é bloqueante: congela tudo pelo tempo especificado. A solução é usar millis() para verificar se "já passou tempo suficiente" sem travar o loop.

Arduino C — Blink sem delay (non-blocking)
unsigned long ultimoToggle = 0;
bool estadoLED = LOW;
#define INTERVALO 1000

void setup() {
  Serial.begin(9600);
  pinMode(LED_BUILTIN, OUTPUT);
  pinMode(2, INPUT_PULLUP);  // botão no pino 2
}

void loop() {
  // Verifica botão (roda TODA iteração do loop)
  if (digitalRead(2) == LOW) {
    Serial.println("Botao pressionado!");
  }

  // Verifica se passou 1 segundo
  if (millis() - ultimoToggle >= INTERVALO) {
    ultimoToggle = millis();
    estadoLED = !estadoLED;
    digitalWrite(LED_BUILTIN, estadoLED);
  }
  // O loop continua rodando IMEDIATAMENTE
  // sem travar — roda centenas de vezes por segundo
}

Este padrão — verificar se "millis() - ultimaVez >= intervalo" — é o padrão fundamental para multitarefa em Arduino. Você vai usá-lo em todos os projetos do semestre.

O que você precisa ter saído daqui sabendo
MCU vs MPU

MCU = tudo num chip, baixo consumo, função dedicada. MPU = CPU externa, precisa de SO, mais poderoso.

Pinout

Todo pino tem nome, número e função. Sempre verifique a tensão lógica antes de conectar. D13 = LED embutido no Uno.

Blink + Serial

setup() roda uma vez. loop() roda para sempre. delay() é bloqueante. Serial.println() é seu console de debug.

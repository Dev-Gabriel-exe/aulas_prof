Servo Motor — controle de ângulo por PWM

O servo motor é um atuador que mantém uma posição angular precisa. Diferente do motor DC que gira livremente, o servo tem feedback interno e circuito de controle embutido — você envia o ângulo desejado, ele vai lá e fica.

O controle é feito por PWM a 50Hz (período de 20ms). A largura do pulso determina o ângulo:

Largura do pulsoÂnguloanalogWrite()?
~0.5 ms (pulse 544µs)0°❌ Não use
Use a biblioteca Servo
~1.5 ms (pulse 1472µs)90° (centro)
~2.5 ms (pulse 2400µs)180°

Arduino C — Servo com biblioteca
#include <Servo.h>   // biblioteca nativa — não precisa instalar

Servo meuServo;

void setup() {
  Serial.begin(9600);
  meuServo.attach(9);   // pino de sinal do servo no pino 9
  meuServo.write(90);   // posição inicial: centro (90°)
  delay(500);
}

void loop() {
  // Varredura suave: 0° → 180° → 0°
  for (int ang = 0; ang <= 180; ang += 2) {
    meuServo.write(ang);
    Serial.print("Ângulo: "); Serial.println(ang);
    delay(20);
  }
  for (int ang = 180; ang >= 0; ang -= 2) {
    meuServo.write(ang);
    delay(20);
  }
}
Pico de corrente no servo. O SG90 pode consumir até 700mA durante o movimento. O pino 5V do Arduino Uno suporta apenas ~500mA total. Para servos maiores ou múltiplos servos, use fonte externa de 5–6V com GND comum ao Arduino. Nunca alimente servo pesado direto pelo pino 5V da placa.

Motor DC com Ponte H e Relé
Motor DC e a Ponte H (L298N)

Um motor DC gira quando passa corrente. Inverta a corrente — ele gira no sentido contrário. Simples na teoria, mas o Arduino não consegue fornecer corrente suficiente para um motor direto nos pinos GPIO (máx 40mA por pino). A solução é o driver de ponte H.

O L298N é o driver mais comum: aceita 5–35V na entrada do motor, até 2A por canal, e é controlado por sinais digitais do Arduino. Ele funciona como um interruptor de alta corrente acionado por sinal fraco.

Estados da Ponte H — como controlar direção e velocidade
Frente
IN1: HIGH
IN2: LOW
ENA: PWM
→ Motor horário
Ré
IN1: LOW
IN2: HIGH
ENA: PWM
→ Anti-horário
Freio
IN1: HIGH
IN2: HIGH
ENA: qualquer
→ Freio elétrico
Parado
IN1: LOW
IN2: LOW
ENA: LOW
→ Motor livre
Arduino C — Motor DC com L298N
#define ENA   6   // PWM — controla velocidade (pino ~ )
#define IN1   4   // direção 1
#define IN2   5   // direção 2

void motorFrente(int velocidade) {   // velocidade: 0–255
  digitalWrite(IN1, HIGH);
  digitalWrite(IN2, LOW);
  analogWrite(ENA, velocidade);
}

void motorRe(int velocidade) {
  digitalWrite(IN1, LOW);
  digitalWrite(IN2, HIGH);
  analogWrite(ENA, velocidade);
}

void motorParar() {
  digitalWrite(IN1, LOW);
  digitalWrite(IN2, LOW);
  analogWrite(ENA, 0);
}

void setup() {
  Serial.begin(9600);
  pinMode(ENA, OUTPUT);
  pinMode(IN1, OUTPUT);
  pinMode(IN2, OUTPUT);
}

void loop() {
  Serial.println("Frente — velocidade 50%");
  motorFrente(127);   delay(3000);

  Serial.println("Parando...");
  motorParar();       delay(1000);

  Serial.println("Ré — velocidade 75%");
  motorRe(191);       delay(3000);

  motorParar();       delay(1000);
}
Diodo de proteção. Motores DC geram tensão reversa quando desligados (back-EMF). Módulos L298N prontos já incluem diodos de proteção. Se montar uma ponte H discreta, adicione diodos 1N4007 em todos os MOSFETs/transistores — sem eles, a tensão reversa pode queimar o driver e o Arduino.
Relé — a chave que separa mundos

O relé é o componente que permite ao Arduino (mundo de 5V, miliamperes) controlar cargas de alta potência como ventiladores, lâmpadas e bombas (mundo de 110–220V, amperes). A isolação elétrica entre os dois mundos é fundamental por segurança.

Lado de controle (5V — Arduino)
Pino VCC → 5V Arduino
Pino GND → GND Arduino
Pino IN → Pino digital Arduino
Lógica: HIGH desliga, LOW liga (maioria dos módulos é ativo-baixo)
Corrente: ~70mA — use transistor se pino não suportar
ISOLAÇÃO
Lado de carga (110–220V — ventilador, lâmpada)
COM — pino comum da chave
NO — normalmente aberto: liga quando relé ativa
NC — normalmente fechado: desliga quando relé ativa
Para ligar algo quando Arduino ativa: use COM + NO
⚠ Nunca toque nos terminais de alta tensão com o circuito ligado
Arduino C — Relé básico (lógica ativo-baixo)
#define PINO_RELE  7

void setup() {
  Serial.begin(9600);
  pinMode(PINO_RELE, OUTPUT);

  // IMPORTANTE: ao iniciar, garante relé desligado
  // Módulos ativo-baixo: HIGH = relé DESLIGADO
  digitalWrite(PINO_RELE, HIGH);
  Serial.println("Relé iniciado — DESLIGADO");
}

void ligarRele() {
  digitalWrite(PINO_RELE, LOW);   // LOW = relé LIGADO (ativo-baixo)
  Serial.println("Relé: LIGADO");
}

void desligarRele() {
  digitalWrite(PINO_RELE, HIGH);  // HIGH = relé DESLIGADO
  Serial.println("Relé: DESLIGADO");
}

void loop() {
  ligarRele();   delay(5000);   // ligado por 5s
  desligarRele(); delay(5000);  // desligado por 5s
}
Ativo-baixo vs ativo-alto. A maioria dos módulos relé de Arduino é ativo-baixo: LOW no pino de controle ativa o relé, HIGH desativa. Isso é contraintuitivo mas comum — verifique a serigrafia do módulo. Se escrever HIGH no setup() para garantir o estado inicial "desligado", você está usando lógica ativo-baixo corretamente.
🙋 Você conectou um relé ativo-baixo e escreveu digitalWrite(RELE, HIGH) no setup(). O que acontece com a carga (ventilador) ao ligar o Arduino?
A
O ventilador fica desligado — comportamento correto e seguro
B
O ventilador liga imediatamente e não desliga
C
O relé queima por receber HIGH antes da inicialização
D
Depende — HIGH pode ligar ou desligar, é imprevisível

Arduino C — Sistema de ventilação automática completo
#include <DHT.h>

#define PINO_DHT    8
#define TIPO_DHT    DHT22
#define PINO_RELE   7
#define LED_VERDE   2
#define LED_AMARELO 3
#define LED_VERMELHO 4

// Limiares de temperatura — ajuste conforme o ambiente
#define TEMP_MORNO   25.0   // °C — inicia ventilação baixa
#define TEMP_QUENTE  30.0   // °C — ventilação máxima

DHT dht(PINO_DHT, TIPO_DHT);
unsigned long ultimaLeitura = 0;
int estadoAnterior = -1;    // evita imprimir o mesmo estado repetido

void definirEstado(int estado) {
  if (estado == estadoAnterior) return;  // sem mudança
  estadoAnterior = estado;

  // Desliga todos os LEDs primeiro
  digitalWrite(LED_VERDE,    LOW);
  digitalWrite(LED_AMARELO,  LOW);
  digitalWrite(LED_VERMELHO, LOW);

  if (estado == 0) {
    digitalWrite(PINO_RELE, HIGH);     // relé DESLIGADO (ativo-baixo)
    digitalWrite(LED_VERDE, HIGH);
    Serial.println("🟢 Temp OK — ventilação DESLIGADA");
  }
  else if (estado == 1) {
    digitalWrite(PINO_RELE, LOW);      // relé LIGADO
    digitalWrite(LED_AMARELO, HIGH);
    Serial.println("🟡 Temperatura elevada — ventilação BAIXA");
  }
  else if (estado == 2) {
    digitalWrite(PINO_RELE, LOW);      // relé LIGADO (máxima)
    digitalWrite(LED_VERMELHO, HIGH);
    Serial.println("🔴 ⚠ ALERTA TÉRMICO — ventilação MÁXIMA");
  }
}

void setup() {
  Serial.begin(9600);
  dht.begin();
  pinMode(PINO_RELE,    OUTPUT);
  pinMode(LED_VERDE,    OUTPUT);
  pinMode(LED_AMARELO,  OUTPUT);
  pinMode(LED_VERMELHO, OUTPUT);
  digitalWrite(PINO_RELE, HIGH);        // inicia desligado
  Serial.println("=== Sistema de Ventilação Automática ===");
  delay(2000);
}

void loop() {
  if (millis() - ultimaLeitura >= 3000) {
    ultimaLeitura = millis();

    float temp = dht.readTemperature();
    float umid = dht.readHumidity();

    if (isnan(temp)) {
      Serial.println("ERRO: falha no DHT22 — verifique ligação");
      return;
    }

    // Imprime telemetria
    Serial.print("Temp: "); Serial.print(temp, 1);
    Serial.print("°C | Umid: "); Serial.print(umid, 1);
    Serial.println("%");

    // Decide estado com histerese (+0.5°C para evitar oscilação)
    if      (temp >= TEMP_QUENTE + 0.5) definirEstado(2);
    else if (temp >= TEMP_MORNO  + 0.5) definirEstado(1);
    else if (temp < TEMP_MORNO   - 0.5) definirEstado(0);
    // Na zona de histerese: mantém estado atual
  }
}
Histerese — evitando oscilação. Sem histerese, se a temperatura fica exatamente em 25.0°C, o relé liga e desliga dezenas de vezes por segundo. Com histerese de ±0.5°C: só liga acima de 25.5°C e só desliga abaixo de 24.5°C. Esse padrão é universal em termostatos, sistemas de ar-condicionado e qualquer controle on/off.

Exercício 01 — Servo como indicador de temperatura

Use o servo motor como um "termômetro visual": mapeie a temperatura lida pelo DHT22 (faixa 15–40°C) para o ângulo do servo (0°–180°). Quanto mais quente, mais aberto. Imprima no Serial Monitor a temperatura e o ângulo correspondente a cada leitura.

LAB
Arduino C
#include <DHT.h>
#include <Servo.h>
DHT dht(8, DHT22);
Servo sv;

void setup(){
  Serial.begin(9600);
  dht.begin();
  sv.attach(9);
  delay(2000);
}

void loop(){
  float t = dht.readTemperature();
  if(isnan(t)){ delay(2000); return; }

  int ang = map((int)(t*10), 150, 400, 0, 180);
  ang = constrain(ang, 0, 180);
  sv.write(ang);

  Serial.print("Temp: "); Serial.print(t,1);
  Serial.print("°C → Servo: ");
  Serial.print(ang); Serial.println("°");
  delay(3000);
}
Exercício 02 — Motor DC com controle de velocidade

Conecte um potenciômetro no pino A0. Mapeie a leitura (0–1023) para a velocidade do motor DC (0–255). Ao girar o potenciômetro, a velocidade do motor deve aumentar ou diminuir suavemente. Adicione um botão no pino 2 que inverta a direção do motor quando pressionado (com debounce).

LAB
Arduino C
#define ENA 6
#define IN1 4
#define IN2 5
#define BTN 2
bool direcao = true;
int  estadoAnt = HIGH;
unsigned long tBtn = 0;

void setup(){
  Serial.begin(9600);
  pinMode(ENA,OUTPUT); pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(BTN, INPUT_PULLUP);
}

void loop(){
  // Botão — debounce
  int lei = digitalRead(BTN);
  if(lei != estadoAnt) tBtn = millis();
  if(millis()-tBtn>50 && lei==LOW && estadoAnt==HIGH){
    direcao = !direcao;
    Serial.println(direcao ? "→ Frente" : "← Ré");
  }
  estadoAnt = lei;

  // Potenciômetro → velocidade
  int vel = map(analogRead(A0), 0, 1023, 0, 255);
  digitalWrite(IN1, direcao ? HIGH : LOW);
  digitalWrite(IN2, direcao ? LOW  : HIGH);
  analogWrite(ENA, vel);
}
Exercício 03 — Temporizador de relé

Implemente um sistema onde o relé liga por um tempo definido e desliga automaticamente. Ao pressionar o botão no pino 2, o relé liga por 10 segundos e depois desliga. Durante o tempo ativo, o Serial Monitor deve exibir a contagem regressiva (10, 9, 8...). Pressionar o botão novamente antes do tempo expirar reinicia o contador.

LAB
Arduino C
#define RELE 7
#define BTN  2
#define DURACAO_S 10

unsigned long tRele=0, tSegundo=0, tBtn=0;
bool releLigado=false;
int contador=0, estadoAnt=HIGH;

void ligar(){
  digitalWrite(RELE, LOW);
  releLigado=true;
  tRele=millis(); tSegundo=millis();
  contador=DURACAO_S;
  Serial.println("Relé LIGADO — contagem iniciada");
}

void desligar(){
  digitalWrite(RELE, HIGH);
  releLigado=false;
  Serial.println("Relé DESLIGADO");
}

void setup(){
  Serial.begin(9600);
  pinMode(RELE,OUTPUT); digitalWrite(RELE,HIGH);
  pinMode(BTN, INPUT_PULLUP);
}

void loop(){
  unsigned long ag=millis();
  int lei=digitalRead(BTN);
  if(lei!=estadoAnt) tBtn=ag;
  if(ag-tBtn>50 && lei==LOW && estadoAnt==HIGH) ligar();
  estadoAnt=lei;

  if(releLigado){
    if(ag-tSegundo>=1000){ tSegundo=ag; contador--;
      Serial.print("Desligando em: "); Serial.println(contador);
    }
    if(contador<=0) desligar();
  }
}
Exercício 04 — Desafio: Sistema completo de controle ambiental

Integre DHT22 + relé + servo + LEDs num único sistema:

• Relé controla o ventilador automaticamente por temperatura (como no LAB)
• Servo abre uma "janela virtual" proporcionalmente à temperatura (0°=fechada, 180°=aberta)
• 3 LEDs indicam o estado (verde/amarelo/vermelho)
• Serial Monitor exibe todas as leituras e estados a cada 3s, formatado como painel:
Temp: 27.3°C | Umid: 65% | Janela: 45° | Fan: ON | Estado: MORNO

Projeto integrador
Estrutura sugerida — implemente você mesmo
  • Pinos: DHT22=8, Relé=7, Servo=9, LED_V=2, LED_A=3, LED_R=4
  • Loop principal sem delay() — use millis() para DHT (3s) e Serial (3s)
  • Função definirEstado(int estado) atualiza relé, servo e LEDs de uma vez
  • map(temp*10, 250, 400, 0, 180) para ângulo do servo (25–40°C → 0–180°)
  • constrain() para limitar ângulo e velocidade a faixas válidas
  • Histerese de ±0.5°C nas transições de estado
  • Serial.print() formatado em linha única com separadores | para facilitar leitura

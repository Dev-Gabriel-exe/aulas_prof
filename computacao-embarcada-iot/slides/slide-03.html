Tipos de dados e portas digitais
Tipos de dados no Arduino — por que isso importa num MCU

Em um computador com 16 GB de RAM você raramente precisa se preocupar com o tipo de dado. No Arduino Uno você tem 2 KB de SRAM. Cada variável mal escolhida desperdiça memória que não tem substituto. Escolher o tipo certo não é pedantismo — é necessidade.

int
2 bytes · 16 bits
−32.768 a 32.767
Contadores, pinos, valores inteiros genéricos. Tipo padrão do Arduino.
unsigned int
2 bytes · 16 bits
0 a 65.535
Quando o valor nunca é negativo. Dobra o alcance positivo do int.
long
4 bytes · 32 bits
−2.1bi a 2.1bi
millis() retorna long. Contadores de tempo e eventos longos.
float
4 bytes · 32 bits
±3.4 × 10³⁸
Temperatura, tensão, valores com casas decimais. Operações lentas no MCU.
bool
1 byte
true / false
Estado de pino, flags de controle, botões pressionados ou não.
byte
1 byte · 8 bits
0 a 255
Valor PWM (0–255), dados de protocolo serial, economiza memória.
Armadilha clássica: int tempo = millis(); — isso vai estourar o int (máx 32.767) em apenas 32 segundos. Use sempre unsigned long para millis().
Portas digitais — três modos, três comportamentos

Um pino digital só conhece dois estados: HIGH (5V no Uno) e LOW (0V). Mas o mesmo pino pode ser configurado de três formas diferentes, e a escolha errada causa bugs difíceis de debugar.

ModoFunção pinMode()Uso típicoDetalhe importante
Saída OUTPUT Controlar LED, relé, buzzer O Arduino fornece corrente. Máx 40mA por pino — sempre use resistor com LED.
Entrada INPUT Ler sensor digital, botão com resistor externo Pino "flutuante" quando nada está conectado — leituras instáveis. Evite sem resistor externo.
Entrada com pull-up INPUT_PULLUP Ler botão sem resistor externo Ativa resistor interno de ~20–50kΩ para 5V. Lógica invertida: pino lê HIGH quando botão solto, LOW quando pressionado.
Arduino C — modos de pino
// Configuração dos modos
pinMode(13, OUTPUT);        // pino 13 como saída (LED)
pinMode(7,  INPUT);          // pino 7 como entrada (precisa de resistor externo)
pinMode(2,  INPUT_PULLUP);   // pino 2 com pull-up interno (botão sem resistor)

// Escrita digital
digitalWrite(13, HIGH);       // acende LED
digitalWrite(13, LOW);        // apaga LED

// Leitura digital
int estado = digitalRead(2);  // HIGH ou LOW

// Com INPUT_PULLUP: lógica INVERTIDA
// Botão solto  → pino lê HIGH (pull-up mantém em 5V)
// Botão apertado → pino lê LOW (botão conecta ao GND)
if (estado == LOW) {
  Serial.println("Botão pressionado!");
}
Por que INPUT_PULLUP? Sem ele, um pino INPUT desconectado "flutua" — oscila aleatoriamente entre HIGH e LOW por interferência elétrica. O pull-up interno garante um nível estável de referência. É a forma mais simples e comum de ler botões no Arduino.
🙋 Você liga o Arduino e o pino 2 configurado como INPUT lê HIGH e LOW aleatoriamente sem nada conectado. Qual é a causa?
A
O código está errado — digitalRead() foi chamado antes do pinMode()
B
O pino está queimado e precisa ser substituído
C
Pino flutuante — sem resistor de pull-up ou pull-down, o pino capta ruído elétrico
D

Analógico, PWM e o mundo contínuo
analogRead() — lendo o mundo analógico

O mundo físico é analógico: temperatura varia de 18.3°C a 18.4°C a 18.5°C — não pula de 18 para 19. O Arduino Uno tem um Conversor Analógico-Digital (ADC) de 10 bits nos pinos A0–A5 que converte tensão (0–5V) em um número inteiro (0–1023).

A matemática por baixo

  • ADC de 10 bits → 2¹⁰ = 1024 valores possíveis (0 a 1023)
  • Resolução: 5V ÷ 1024 ≈ 4,9 mV por passo
  • 0V → analogRead() = 0
  • 2,5V → analogRead() ≈ 512
  • 5V → analogRead() = 1023

Convertendo para tensão real

Arduino C
int leitura = analogRead(A0);

// Converter para tensão em volts
float tensao = leitura * (5.0 / 1023.0);

Serial.print("Leitura: ");
Serial.print(leitura);
Serial.print(" | Tensão: ");
Serial.print(tensao);
Serial.println("V");
map() — remapeando escalas: a função map(valor, deMin, deMax, paraMin, paraMax) converte de uma escala para outra. Exemplo: map(leitura, 0, 1023, 0, 255) converte a leitura do potenciômetro para a escala PWM. Você vai usar isso hoje no LAB.
PWM — simular analógico com digital

O Arduino Uno é um chip digital — seus pinos só sabem HIGH ou LOW. Mas às vezes você precisa de algo no meio: 50% de brilho num LED, 30% de velocidade num motor. A solução é o PWM (Pulse Width Modulation).

A ideia é simples: ao invés de ligar ou desligar, o pino fica ligando e desligando muito rápido — tão rápido (490–980 Hz) que o LED parece ter brilho intermediário e o motor parece ter velocidade intermediária. O que você controla é o duty cycle: a porcentagem do tempo que o pino fica em HIGH.

Arduino C — PWM básico
// analogWrite funciona APENAS nos pinos com ~ no Uno:
// 3, 5, 6, 9, 10, 11

void setup() {
  pinMode(9, OUTPUT);   // pino 9 suporta PWM (~)
}

void loop() {
  // Fade in: aumenta brilho do 0 ao máximo
  for (int i = 0; i <= 255; i++) {
    analogWrite(9, i);   // 0 = apagado, 255 = brilho máximo
    delay(10);
  }
  // Fade out: diminui brilho do máximo a 0
  for (int i = 255; i >= 0; i--) {
    analogWrite(9, i);
    delay(10);
  }
}
Arduino C — potenciômetro controla LED via PWM
// Potenciômetro no A0 controla brilho do LED no pino 9
// map() converte a escala ADC (0–1023) para PWM (0–255)

void setup() {
  Serial.begin(9600);
  pinMode(9, OUTPUT);   // sem pinMode para entrada analógica
}

void loop() {
  int pot    = analogRead(A0);              // 0–1023
  int brilho = map(pot, 0, 1023, 0, 255);  // remapeia para 0–255
  analogWrite(9, brilho);

  Serial.print("Pot: ");   Serial.print(pot);
  Serial.print(" | PWM: "); Serial.println(brilho);
  delay(50);
}

Leitura de botão, debounce e Serial Monitor avançado
O problema do bounce — quando o hardware mente

Você pressiona um botão uma vez. O Arduino detecta 47 pressionamentos. Bem-vindo ao bounce.

Quando você aperta um botão físico, os contatos metálicos internos não fecham de forma limpa — eles vibram mecanicamente por 5–20ms, abrindo e fechando dezenas de vezes antes de estabilizar. Para o Arduino rodando a 16 MHz, cada uma dessas oscilações é um evento distinto.

🔓
Botão SOLTO — INPUT_PULLUP
Resistor interno puxa o pino para 5V. digitalRead() retorna HIGH. Estado de repouso.
HIGH = 1
⚡
Bounce — transição instável (~5–20ms)
Contatos oscilam rapidamente. digitalRead() alterna HIGH/LOW dezenas de vezes. Software precisa ignorar essa janela.
? ? ?
🔒
Botão PRESSIONADO — estabilizado
Contato fecha, pino vai para GND (0V). digitalRead() retorna LOW estável. Lógica INVERTIDA com INPUT_PULLUP.
LOW = 0
Arduino C — debounce por software com millis()
#define PINO_BTN     2
#define DEBOUNCE_MS  50   // aguarda 50ms de estabilidade

int           estadoAnterior  = HIGH;
int           estadoAtual     = HIGH;
unsigned long ultimaMudanca   = 0;
int           contadorCliques = 0;

void setup() {
  Serial.begin(9600);
  pinMode(PINO_BTN, INPUT_PULLUP);
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  int leitura = digitalRead(PINO_BTN);

  // Mudança detectada — reinicia o timer
  if (leitura != estadoAnterior) {
    ultimaMudanca = millis();
  }

  // Só processa se ficou estável por DEBOUNCE_MS
  if (millis() - ultimaMudanca > DEBOUNCE_MS) {
    if (leitura != estadoAtual) {
      estadoAtual = leitura;

      if (estadoAtual == LOW) {     // LOW = pressionado (INPUT_PULLUP)
        contadorCliques++;
        digitalWrite(LED_BUILTIN, !digitalRead(LED_BUILTIN)); // toggle
        Serial.print("Clique #");
        Serial.println(contadorCliques);
      }
    }
  }
  estadoAnterior = leitura;
}
Serial Monitor — além do println()

O Serial Monitor não serve só para imprimir texto. É a sua interface de debug, de monitoramento em tempo real e — com Serial.read() — de receber comandos do computador. Dominar o Serial é dominar o debug embarcado.

Arduino C — Serial avançado: recebendo comandos
// Envie '1' ou '0' pelo Serial Monitor para ligar/desligar o LED
void setup() {
  Serial.begin(9600);
  pinMode(LED_BUILTIN, OUTPUT);
  Serial.println("Digite 1 para ligar, 0 para desligar");
}

void loop() {
  if (Serial.available() > 0) {     // tem dado esperando?
    char cmd = Serial.read();         // lê 1 byte

    if (cmd == '1') {
      digitalWrite(LED_BUILTIN, HIGH);
      Serial.println("LED LIGADO");
    }
    else if (cmd == '0') {
      digitalWrite(LED_BUILTIN, LOW);
      Serial.println("LED DESLIGADO");
    }
  }
}

Exercício 01 — Qual tipo de dado?

Para cada variável abaixo, justifique qual tipo de dado você usaria no Arduino e por quê:

(a) O resultado de millis() — valor em milissegundos desde o boot
(b) O valor lido por analogRead(A0) — entre 0 e 1023
(c) A temperatura calculada de um sensor — ex: 23.7°C
(d) O estado de um LED (ligado ou desligado)
(e) Um valor PWM para enviar a um servo (0 a 255)

Conceitual
  • (a) unsigned long — millis() retorna unsigned long. Um int estouraria em 32 segundos.
  • (b) int — 0 a 1023 cabe num int (máx 32.767). Ou unsigned int para ser preciso.
  • (c) float — temperatura tem casas decimais. float é o único tipo nativo que as suporta no Arduino.
  • (d) bool — estado binário: true/false ou HIGH/LOW. Gasta apenas 1 byte.
  • (e) byte — 0 a 255 cabe exatamente num byte (1 byte, sem desperdício). int usaria 2 bytes para a mesma informação.
Exercício 02 — Encontre o bug

O código abaixo deveria piscar o LED a cada 2 segundos usando millis(), mas não funciona corretamente após alguns minutos. Identifique o bug e corrija-o.

Arduino C — com bug
int ultimoToggle = 0;   // ← bug aqui
bool estado = false;

void setup() { pinMode(LED_BUILTIN, OUTPUT); }

void loop() {
  if (millis() - ultimoToggle >= 2000) {
    ultimoToggle = millis();
    estado = !estado;
    digitalWrite(LED_BUILTIN, estado);
  }
}
Análise

Bug: int ultimoToggle — int tem máximo de 32.767. millis() chega a 32.767 em apenas ~32 segundos. Depois disso, int faz overflow e o comportamento fica indefinido.

Correção: trocar int por unsigned long — que aguenta até ~49 dias sem overflow.

Corrigido
unsigned long ultimoToggle = 0;  // ✅ correto
Exercício 03 — Dimmer com 3 níveis

Implemente um dimmer de 3 níveis controlado por um botão: cada pressionamento avança para o próximo nível (25% → 50% → 100% → 25% → ...). Use debounce, PWM no pino 9 e imprima o nível atual no Serial Monitor a cada mudança.

LAB
Arduino C
#define PINO_BTN   2
#define PINO_LED   9
#define DEBOUNCE   50

const byte niveis[]    = {64, 127, 255};   // 25%, 50%, 100%
const char* nomeNivel[] = {"25%", "50%", "100%"};
int           nivelAtual    = 0;
int           estadoAnt     = HIGH;
int           estadoAtual_  = HIGH;
unsigned long ultimaMudanca = 0;

void setup() {
  Serial.begin(9600);
  pinMode(PINO_BTN, INPUT_PULLUP);
  pinMode(PINO_LED, OUTPUT);
  analogWrite(PINO_LED, niveis[nivelAtual]);
  Serial.println("Nível inicial: 25%");
}

void loop() {
  int leitura = digitalRead(PINO_BTN);
  if (leitura != estadoAnt) ultimaMudanca = millis();
  if (millis() - ultimaMudanca > DEBOUNCE) {
    if (leitura != estadoAtual_) {
      estadoAtual_ = leitura;
      if (estadoAtual_ == LOW) {
        nivelAtual = (nivelAtual + 1) % 3;
        analogWrite(PINO_LED, niveis[nivelAtual]);
        Serial.print("Nível: ");
        Serial.println(nomeNivel[nivelAtual]);
      }
    }
  }
  estadoAnt = leitura;
}
Exercício 04 — Desafio: Semáforo não-bloqueante

Implemente um semáforo com 3 LEDs (vermelho, amarelo, verde) nos pinos 11, 10 e 9. Ciclo: verde 5s → amarelo 2s → vermelho 5s. Restrições: sem delay() em nenhuma linha — use apenas millis(). O Serial Monitor deve imprimir a cor atual e o tempo restante a cada segundo.

Desafio
Arduino C — Semáforo com millis()
#define PIN_VERDE    9
#define PIN_AMARELO 10
#define PIN_VERMELHO 11

const unsigned long duracoes[] = {5000, 2000, 5000};
const char* nomes[]             = {"VERDE", "AMARELO", "VERMELHO"};
const int  pinos[]              = {PIN_VERDE, PIN_AMARELO, PIN_VERMELHO};

int           fase           = 0;
unsigned long inicioFase     = 0;
unsigned long ultimoSegundo  = 0;

void ativarFase(int f) {
  for (int i = 0; i < 3; i++) digitalWrite(pinos[i], i == f ? HIGH : LOW);
  Serial.print("FASE: "); Serial.println(nomes[f]);
}

void setup() {
  Serial.begin(9600);
  for (int i = 0; i < 3; i++) pinMode(pinos[i], OUTPUT);
  ativarFase(fase);
  inicioFase = millis();
}

void loop() {
  unsigned long agora = millis();

  // Avança fase quando duração expirar
  if (agora - inicioFase >= duracoes[fase]) {
    fase = (fase + 1) % 3;
    inicioFase = agora;
    ativarFase(fase);
  }

  // Imprime tempo restante a cada segundo
  if (agora - ultimoSegundo >= 1000) {
    ultimoSegundo = agora;
    unsigned long restante = (duracoes[fase] - (agora - inicioFase)) / 1000;
    Serial.print(nomes[fase]);
    Serial.print(": "); Serial.print(restante); Serial.println("s restantes");
  }
}

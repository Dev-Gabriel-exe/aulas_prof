Por que protocolos — e os três mais importantes
O problema que protocolos resolvem

Até agora você controlou tudo com digitalRead() e analogRead() — um pino, um valor, sem coordenação. Isso funciona para sensores simples. Mas e quando você precisa de um display OLED? De um módulo GPS? De uma memória flash externa? De um sensor de pressão com 10 registros de configuração?

Esses dispositivos precisam trocar múltiplos bytes de dados em sequência, de forma coordenada, sabendo quem fala e quem escuta. É exatamente para isso que existem os protocolos de comunicação serial.

Serial vs Paralelo: comunicação paralela envia múltiplos bits simultaneamente em fios separados — rápida mas cara em pinos e PCB. Serial envia um bit por vez no mesmo fio — mais lenta mas usa pouquíssimos pinos. Em sistemas embarcados com pinos escassos, serial é a escolha quase universal.

I2C por dentro — como o barramento funciona

O I2C usa apenas dois fios para toda a comunicação. SDA (Serial Data) transporta os dados. SCL (Serial Clock) sincroniza a comunicação. O master controla o clock e inicia toda comunicação. Os slaves respondem quando endereçados.

Cada dispositivo I2C tem um endereço de 7 bits gravado em hardware — geralmente configurável por pinos de endereço (A0, A1, A2). O display OLED SSD1306 usa o endereço 0x3C (ou 0x3D se o pino SA0 estiver em HIGH).

Pinos I2C no Arduino Uno: SDA = A4 · SCL = A5. No ESP32: SDA = GPIO21 · SCL = GPIO22 (padrão, mas configurável). A biblioteca Wire.h abstrai tudo — você não precisa implementar o protocolo manualmente.
Scanner I2C — encontrando endereços desconhecidos

Comprou um módulo sem documentação? Não sabe o endereço I2C? O scanner percorre todos os 127 endereços possíveis e reporta quais respondem. É o primeiro código que você roda antes de qualquer projeto I2C.

Arduino C — I2C Scanner
#include <Wire.h>

void setup() {
  Wire.begin();
  Serial.begin(9600);
  Serial.println("=== Scanner I2C ===");
  Serial.println("Varrendo endereços 0x01 a 0x7F...");

  int encontrados = 0;

  for (byte addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    byte erro = Wire.endTransmission();

    if (erro == 0) {   // 0 = dispositivo respondeu
      Serial.print("Dispositivo encontrado: 0x");
      if (addr < 16) Serial.print("0");   // zero-pad
      Serial.println(addr, HEX);
      encontrados++;
    }
  }

  if (encontrados == 0) {
    Serial.println("Nenhum dispositivo encontrado!");
    Serial.println("Verifique as conexões e os pull-ups.");
  } else {
    Serial.print(encontrados);
    Serial.println(" dispositivo(s) encontrado(s).");
  }
}

void loop() {}  // scanner roda uma vez no setup()

SPI, UART e o Display OLED SSD1306
SPI — velocidade em troca de pinos

O SPI usa 4 sinais: MOSI (Master Out Slave In), MISO (Master In Slave Out), SCK (clock) e CS/SS (Chip Select — um pino por dispositivo). É síncrono e full-duplex: master e slave trocam dados simultaneamente a cada pulso de clock.

A grande desvantagem do SPI em relação ao I2C: cada dispositivo adicional precisa de um pino CS dedicado. Com 4 dispositivos SPI, você usa 4 pinos extras só para seleção. Em projetos com muitos periféricos, isso limita a escalabilidade.

Sinal SPIDireçãoPino Arduino UnoFunção
MOSIMaster → SlavePino 11Dados do master para o slave
MISOSlave → MasterPino 12Dados do slave para o master
SCKMaster → SlavePino 13Clock gerado pelo master
CS/SSMaster → SlavePino 10 (padrão)Seleciona qual slave está ativo (LOW = ativo)
SCK = pino 13 = LED_BUILTIN no Arduino Uno. Se você usar SPI enquanto tem um LED no pino 13, ele vai piscar na frequência do clock. Na prática, mova o LED para outro pino em projetos SPI.
UART — o protocolo que você já usa sem saber

Toda vez que você abre o Serial Monitor e faz Serial.println(), está usando UART. É o protocolo mais simples: dois fios (TX e RX), sem clock compartilhado — cada lado precisa concordar previamente com a mesma velocidade (baud rate).

UART é assíncrono: não há sinal de clock externo. A sincronização é feita por bits de start e stop embutidos em cada byte transmitido. Por isso o baud rate precisa ser idêntico nos dois lados — divergências causam dados corrompidos.

Display OLED SSD1306 — o painel do seu projeto

O display OLED SSD1306 de 0.96" tem resolução de 128×64 pixels, interface I2C (4 pinos: VCC, GND, SDA, SCL) e consome apenas ~20mA quando ativo. É o display padrão do ecossistema Arduino por custo, tamanho e facilidade de uso.

A biblioteca Adafruit SSD1306 (com dependência Adafruit GFX) abstrai toda a comunicação I2C e oferece funções de alto nível: texto, linhas, retângulos, bitmaps. Você não precisa entender o protocolo do chip para usar.

Arduino C — OLED SSD1306 básico
#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>

#define LARGURA   128
#define ALTURA     64
#define OLED_RESET  -1   // -1 = sem pino de reset
#define ADDR_OLED 0x3C   // endereço padrão do SSD1306

Adafruit_SSD1306 display(LARGURA, ALTURA, &Wire, OLED_RESET);

void setup() {
  Serial.begin(9600);

  if (!display.begin(SSD1306_SWITCHCAPVCC, ADDR_OLED)) {
    Serial.println("OLED não encontrado! Verifique endereço e ligação.");
    while (true);   // trava — não tem como continuar sem display
  }

  display.clearDisplay();                  // limpa buffer
  display.setTextSize(1);                  // tamanho 1 = 6×8 px por char
  display.setTextColor(SSD1306_WHITE);
  display.setCursor(0, 0);               // coluna=0, linha=0
  display.println("Ola, OLED!");
  display.display();                       // ENVIA o buffer para o hardware
}

void loop() {}
O padrão buffer/display: a biblioteca SSD1306 trabalha com um buffer em RAM. Você desenha no buffer com clearDisplay(), print(), drawLine() etc. Nada aparece no display até você chamar display.display(). Esquecer esse chamada é o erro mais comum de iniciantes com OLED.
🙋 Qual é a diferença fundamental entre I2C e SPI que define quando usar cada um?
A
I2C é mais rápido, SPI é mais lento — use I2C sempre que possível
B
I2C usa menos pinos e endereça múltiplos dispositivos; SPI é mais rápido mas precisa de um pino CS por dispositivo
C
SPI é para comunicação longa distância, I2C é para curta distância
D
I2C é síncrono, SPI é assíncrono — a diferença é apenas no clock

LAB — Estação meteorológica com OLED + DHT22
Objetivo: exibir temperatura, umidade e um indicador de conforto no display OLED, atualizando a cada 3 segundos com millis(). Sem delay() em nenhuma linha.

Experimente mudar o tamanho do texto (setTextSize(2) para o valor principal), adicionar linhas divisórias (drawFastHLine), ou exibir um ícone simples desenhado com setPixel(). O display é sua tela — explore.
Arduino C — Estação meteorológica OLED + DHT22 completa
#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>
#include <DHT.h>

#define OLED_W     128
#define OLED_H      64
#define OLED_RESET  -1
#define OLED_ADDR  0x3C
#define PINO_DHT    8
#define TIPO_DHT    DHT22

Adafruit_SSD1306 display(OLED_W, OLED_H, &Wire, OLED_RESET);
DHT dht(PINO_DHT, TIPO_DHT);

unsigned long ultimaAtualizacao = 0;
float temp = 0, umid = 0;

void desenharTela(float t, float u) {
  display.clearDisplay();

  // ── Cabeçalho ──
  display.setTextSize(1);
  display.setTextColor(SSD1306_WHITE);
  display.setCursor(20, 0);
  display.println("ESTACAO METEO");
  display.drawFastHLine(0, 10, OLED_W, SSD1306_WHITE);

  // ── Temperatura grande ──
  display.setTextSize(2);
  display.setCursor(10, 16);
  display.print(t, 1);
  display.print(" C");

  // ── Umidade pequena ──
  display.setTextSize(1);
  display.setCursor(0, 40);
  display.print("Umidade: ");
  display.print(u, 1);
  display.println("%");

  // ── Indicador de conforto ──
  display.drawFastHLine(0, 52, OLED_W, SSD1306_WHITE);
  display.setCursor(0, 55);
  if      (t < 18) display.print("Frio");
  else if (t > 28) display.print("Quente!");
  else if (u > 70) display.print("Umido");
  else             display.print("Confortavel");

  display.display();   // envia buffer → hardware
}

void setup() {
  Serial.begin(9600);
  dht.begin();

  if (!display.begin(SSD1306_SWITCHCAPVCC, OLED_ADDR)) {
    Serial.println("OLED nao encontrado!");
    while (true);
  }

  // Tela de boas-vindas
  display.clearDisplay();
  display.setTextSize(1);
  display.setTextColor(SSD1306_WHITE);
  display.setCursor(25, 28);
  display.println("Inicializando...");
  display.display();
  delay(2000);   // aguarda DHT22 estabilizar
}

void loop() {
  if (millis() - ultimaAtualizacao >= 3000) {
    ultimaAtualizacao = millis();

    float t = dht.readTemperature();
    float u = dht.readHumidity();

    if (isnan(t) || isnan(u)) {
      display.clearDisplay();
      display.setCursor(5, 28);
      display.println("ERRO: sensor DHT");
      display.display();
      return;
    }

    temp = t; umid = u;
    desenharTela(temp, umid);

    Serial.print("T:"); Serial.print(t,1);
    Serial.print(" U:"); Serial.println(u,1);
  }
}

Exercício 01 — Scanner e identificação

Execute o código Scanner I2C com o display OLED e o DHT22 conectados. Anote o endereço encontrado e responda: (a) o endereço retornado é em decimal ou hexadecimal? (b) Se você conectar um segundo OLED idêntico ao mesmo barramento, o scanner encontraria dois dispositivos no mesmo endereço? O que aconteceria na prática?

Conceito + LAB
  • (a) O scanner imprime em hexadecimal — Serial.println(addr, HEX). O OLED SSD1306 aparecerá como 0x3C.
  • (b) Não — dois dispositivos com o mesmo endereço no I2C causam conflito de barramento. Quando o master enviar dados para 0x3C, ambos responderão simultaneamente, corrompendo os dados. Para usar dois OLEDs iguais, um deles precisa ter o endereço alterado para 0x3D via pino SA0. O scanner encontraria apenas um endereço respondendo de forma imprevisível.
Exercício 02 — Barra de progresso no OLED

Adicione ao código da estação meteorológica uma barra de progresso horizontal que representa a umidade: 0% = barra vazia, 100% = barra cheia. A barra deve ocupar a linha inferior do display (y=56, largura 128px, altura 6px). Use display.drawRect() para o contorno e display.fillRect() para o preenchimento proporcional.

LAB
Trecho — adicionar na função desenharTela()
// Substituir o bloco de indicador de conforto por:
display.drawFastHLine(0, 52, OLED_W, SSD1306_WHITE);

// Contorno da barra
display.drawRect(0, 56, 128, 8, SSD1306_WHITE);

// Preenchimento proporcional à umidade
int larguraBarra = (int)(126 * (u / 100.0));
larguraBarra = constrain(larguraBarra, 0, 126);
display.fillRect(1, 57, larguraBarra, 6, SSD1306_WHITE);

// Label da umidade ao lado
display.setCursor(0, 44);
display.print("UR: "); display.print(u,0); display.print("%");
Exercício 03 — Menu de telas

Implemente um sistema de duas telas alternáveis por botão: Tela 1 mostra temperatura e umidade (como no LAB). Tela 2 mostra a distância do HC-SR04 com a barra de proximidade. Cada pressionamento do botão alterna entre as telas. Use debounce e millis() — sem delay().

Integração
Estrutura sugerida
  • Variável global int telaAtual = 0
  • Duas funções: desenharTelaDHT() e desenharTelaSonar()
  • No loop(): botão com debounce alterna telaAtual entre 0 e 1
  • Timer de 3s para DHT22 e timer de 300ms para HC-SR04 — independentes com millis()
  • A função de desenho chamada depende do telaAtual: if (telaAtual == 0) desenharTelaDHT(); else desenharTelaSonar();
Exercício 04 — Desafio: animação de boot

Crie uma tela de boot animada que exibe uma barra de progresso de 0 a 100% durante os 2 segundos de inicialização do DHT22. Use fillRect() crescendo de 0 a 128px. Ao terminar, limpa a tela e mostra "Sistema pronto!" por 1 segundo antes de iniciar a tela principal. Tudo usando millis() — sem delay() no loop() (o delay(2000) inicial ainda é permitido no setup()).

Desafio
Arduino C — animação de boot no setup()
// No setup(), substitua o delay(2000) por:
unsigned long inicio = millis();
unsigned long duracao = 2000;

while (millis() - inicio < duracao) {
  int progresso = (int)(128 * (float)(millis() - inicio) / duracao);

  display.clearDisplay();
  display.setTextSize(1);
  display.setCursor(30, 20);
  display.println("Inicializando...");

  // Barra de progresso
  display.drawRect(0, 40, 128, 10, SSD1306_WHITE);
  display.fillRect(1, 41, progresso, 8, SSD1306_WHITE);

  display.setCursor(55, 55);
  display.print((int)(progresso / 1.28)); display.print("%");
  display.display();
}

display.clearDisplay();
display.setCursor(25, 28);
display.println("Sistema pronto!");
display.display();

ETL, ELT e Arquitetura de Zonas
O que é um Pipeline de Dados?

Um pipeline é uma sequência automatizada de etapas que move dados de uma origem até um destino, aplicando transformações no caminho. É a infraestrutura que conecta o banco de dados ao modelo de ML.

Fonte
BD relacional, API, arquivo, stream
Ingestão
Extrair dados brutos para o lake
Transformação
Limpar, tipificar, enriquecer
Features
Calcular variáveis para ML
Modelo
Treino, inferência, resultado
ETL vs ELT — qual a diferença?

ETL — Extract, Transform, Load

  • Transforma antes de carregar no destino
  • Dados chegam já limpos e prontos
  • Tradicional — Data Warehouses clássicos
  • Mais lento para pipelines grandes

ELT — Extract, Load, Transform

  • Carrega bruto primeiro, transforma depois
  • Aproveitam o poder de processamento do BD
  • Moderno — Data Lakes, BigQuery, Snowflake
  • Mais flexível — reprocessar sem re-extrair
Na prática com IA: ELT é mais comum. Você carrega os dados brutos no lake, depois usa SQL (dbt, Spark SQL) ou Python para criar as features dentro da plataforma.
Arquitetura de Zonas — Raw, Silver, Gold

O padrão mais adotado em Data Lakes organiza os dados em três camadas, cada uma com nível crescente de qualidade e transformação.

🥉 Raw / Bronze
Dados brutos

Exatamente como vieram da fonte — sem modificação. Nunca deletar. Servem como histórico e ponto de reprocessamento.

🥈 Silver
Dados limpos

Nulos tratados, tipos corretos, duplicatas removidas, esquema validado. Pode ser consultado com confiança.

🥇 Gold
Dados prontos

Agregados, features calculadas, tabelas wide para ML ou relatórios. Consome direto sem transformação adicional.

Limpeza, Transformação e Feature Engineering
Limpeza de dados — o que verificar sempre

Antes de qualquer transformação, audite o dado bruto. As maiores surpresas acontecem aqui.

SQL — Auditoria de qualidade no banco
-- 1. Contar nulos por coluna
SELECT
  COUNT(*) AS total_linhas,
  COUNT(*) - COUNT(nome)   AS nulos_nome,
  COUNT(*) - COUNT(email)  AS nulos_email,
  COUNT(*) - COUNT(cidade) AS nulos_cidade
FROM clientes;

-- 2. Detectar duplicatas
SELECT email, COUNT(*) AS ocorrencias
FROM clientes
WHERE email IS NOT NULL
GROUP BY email
HAVING COUNT(*) > 1;

-- 3. Verificar valores fora do esperado
SELECT status, COUNT(*) AS qtd
FROM pedidos
GROUP BY status
ORDER BY qtd DESC;
-- esperado: entregue, pendente, processando, cancelado
-- se aparecer 'Entregue' ou 'PENDENTE': problema de padronização

-- 4. Checar ranges inválidos
SELECT COUNT(*) AS precos_invalidos
FROM produtos
WHERE preco <= 0 OR preco IS NULL;
Transformações comuns — raw → silver
SQL — Criar tabela silver de clientes
-- Tabela silver: dados limpos e padronizados
CREATE TABLE silver_clientes AS
SELECT
  id,
  -- padronizar texto: trim + capitalizar
  INITCAP(TRIM(nome))                          AS nome,
  LOWER(TRIM(email))                           AS email,
  -- tratar nulos com valor padrão
  COALESCE(TRIM(cidade), 'Desconhecida')        AS cidade,
  criado_em,
  -- flag de qualidade
  CASE WHEN email IS NOT NULL THEN TRUE ELSE FALSE END AS tem_email,
  NOW()                                          AS processado_em
FROM clientes;

-- Tabela silver: pedidos com status padronizado
CREATE TABLE silver_pedidos AS
SELECT
  id,
  cliente_id,
  LOWER(TRIM(status))                          AS status,
  COALESCE(total, 0)                             AS total,
  criado_em::date                                AS data_pedido
FROM pedidos
WHERE total > 0 OR total IS NULL; -- excluir pedidos com total negativo
Feature Engineering no SQL — silver → gold

Feature engineering é calcular variáveis que o modelo não consegue derivar sozinho a partir dos dados brutos. SQL é uma ferramenta poderosa para isso — especialmente para features baseadas em agregações e janelas temporais.

Exemplos de features para um modelo de churn prediction:

total_pedidos ticket_medio total_gasto dias_desde_ultimo_pedido tem_pedido_cancelado qtd_categorias_distintas pct_pedidos_entregues
SQL — Tabela gold: features de cliente para ML
CREATE TABLE gold_features_cliente AS
WITH

-- Métricas gerais de pedidos por cliente
metricas_pedidos AS (
  SELECT
    cliente_id,
    COUNT(*)                                          AS total_pedidos,
    ROUND(AVG(total)::numeric, 2)                     AS ticket_medio,
    SUM(total)                                        AS total_gasto,
    MAX(data_pedido)                                  AS ultimo_pedido,
    MIN(data_pedido)                                  AS primeiro_pedido,
    COUNT(*) FILTER (WHERE status = 'cancelado')    AS qtd_cancelados,
    COUNT(*) FILTER (WHERE status = 'entregue')     AS qtd_entregues
  FROM silver_pedidos
  GROUP BY cliente_id
),

-- Diversidade de categorias compradas
diversidade AS (
  SELECT
    p.cliente_id,
    COUNT(DISTINCT pr.categoria) AS qtd_categorias
  FROM silver_pedidos p
  INNER JOIN itens_pedido i  ON i.pedido_id  = p.id
  INNER JOIN produtos pr     ON pr.id        = i.produto_id
  GROUP BY p.cliente_id
)

-- Tabela gold final
SELECT
  c.id                                                        AS cliente_id,
  c.cidade,
  c.tem_email,
  mp.total_pedidos,
  mp.ticket_medio,
  mp.total_gasto,
  CURRENT_DATE - mp.ultimo_pedido                          AS dias_inativo,
  mp.ultimo_pedido,
  mp.qtd_cancelados,
  mp.qtd_entregues,
  ROUND(
    mp.qtd_cancelados::numeric
    / NULLIF(mp.total_pedidos, 0) * 100, 1
  )                                                           AS pct_cancelados,
  COALESCE(d.qtd_categorias, 0)                             AS qtd_categorias,
  NOW()                                                       AS gerado_em
FROM silver_clientes c
LEFT JOIN metricas_pedidos mp ON mp.cliente_id = c.id
LEFT JOIN diversidade d       ON d.cliente_id  = c.id;
FILTER (WHERE ...) é uma sintaxe PostgreSQL que aplica uma condição dentro de uma função de agregação — equivalente a um SUM(CASE WHEN ...), mas muito mais legível.

Integração Python ↔ PostgreSQL
Conectar Python ao banco — SQLAlchemy

SQLAlchemy é a forma padrão de conectar Python a bancos relacionais. Junto com pandas, permite ler e escrever tabelas diretamente como DataFrames.

Python — Setup e conexão
import os
import pandas as pd
from sqlalchemy import create_engine

# Nunca hardcode credenciais — use variáveis de ambiente
DATABASE_URL = os.getenv("DATABASE_URL")
# ex: "postgresql://usuario:senha@localhost:5432/loja_exemplo"

engine = create_engine(DATABASE_URL)

# Ler tabela inteira para DataFrame
df_clientes = pd.read_sql("SELECT * FROM clientes", engine)

# Query com filtro
df_pedidos = pd.read_sql("""
    SELECT p.id, c.nome, p.status, p.total, p.criado_em
    FROM pedidos p
    INNER JOIN clientes c ON c.id = p.cliente_id
    WHERE p.status = 'entregue'
    ORDER BY p.criado_em DESC
""", engine)

print(df_pedidos.head())
Pipeline completo — banco → features → arquivo

O padrão de um pipeline de preparação de dados para ML em Python:

Python — Pipeline BD → features → Parquet
import pandas as pd
import numpy as np
from sqlalchemy import create_engine
from sklearn.preprocessing import StandardScaler, LabelEncoder

engine = create_engine(os.getenv("DATABASE_URL"))

# ── ETAPA 1: Extrair features do banco (gold) ────────────────────
df = pd.read_sql("SELECT * FROM gold_features_cliente", engine)
print(f"Extraídos {len(df)} clientes")

# ── ETAPA 2: Verificar qualidade ─────────────────────────────────
print(df.isnull().sum())     # contar nulos
print(df.dtypes)             # verificar tipos

# ── ETAPA 3: Tratar nulos ────────────────────────────────────────
df["ticket_medio"].fillna(0, inplace=True)
df["total_gasto"].fillna(0, inplace=True)
df["qtd_categorias"].fillna(0, inplace=True)
df["dias_inativo"].fillna(999, inplace=True)  # 999 = nunca comprou

# ── ETAPA 4: Encoding de categorias ─────────────────────────────
le = LabelEncoder()
df["cidade_enc"] = le.fit_transform(df["cidade"].fillna("Desconhecida"))
df["tem_email_enc"] = df["tem_email"].astype(int)

# ── ETAPA 5: Normalização (apenas no treino!) ────────────────────
colunas_numericas = ["ticket_medio", "total_gasto", "total_pedidos"]
scaler = StandardScaler()
df[[c + "_norm" for c in colunas_numericas]] = scaler.fit_transform(df[colunas_numericas])

# ── ETAPA 6: Salvar como Parquet (gold) ─────────────────────────
df.to_parquet("gold/features_cliente_v1.parquet", index=False, compression="snappy")
print(f"Features salvas: {df.shape[1]} colunas, {len(df)} linhas")
Gravar resultados de volta no banco

Depois de treinar o modelo, as predições devem ir para o banco — onde outros sistemas (dashboard, API, CRM) podem consumi-las.

Python — Salvar predições no banco
import joblib

# Carregar modelo treinado e fazer predição
modelo = joblib.load("modelo_churn_v1.pkl")
cols_features = ["ticket_medio_norm", "total_pedidos", "dias_inativo",
                 "pct_cancelados", "qtd_categorias"]

df["prob_churn"] = modelo.predict_proba(df[cols_features])[:, 1]
df["churn_previsto"] = modelo.predict(df[cols_features])

# Salvar predições de volta no PostgreSQL
df[["cliente_id", "prob_churn", "churn_previsto"]].to_sql(
    "predicoes_churn",
    engine,
    if_exists="replace",  # ou "append" para acumular histórico
    index=False
)
print("Predições gravadas no banco.")
if_exists="replace" recria a tabela a cada execução. Use "append" se quiser acumular histórico de predições com timestamp. Nunca use replace em tabela com dados históricos importantes.
Consultando predições com SQL
SQL — Análise dos resultados do modelo
-- Top 10 clientes com maior risco de churn
SELECT
  c.nome,
  c.cidade,
  f.total_pedidos,
  f.dias_inativo,
  ROUND(p.prob_churn * 100, 1)    AS risco_pct,
  CASE
    WHEN p.prob_churn > 0.7 THEN '🔴 Alto'
    WHEN p.prob_churn > 0.4 THEN '🟡 Médio'
    ELSE '🟢 Baixo'
  END                              AS categoria_risco
FROM clientes c
INNER JOIN gold_features_cliente f  ON f.cliente_id = c.id
INNER JOIN predicoes_churn p        ON p.cliente_id = c.id
ORDER BY p.prob_churn DESC
LIMIT 10;

Exercício 01

Escreva uma query de auditoria de qualidade para a tabela pedidos: conte o total de linhas, quantos pedidos têm total IS NULL, quantos têm total <= 0, e quantos têm status fora dos valores esperados (entregue, pendente, processando, cancelado).

Básico — Auditoria SQL
SQL
SELECT
  COUNT(*)                                                            AS total_linhas,
  COUNT(*) FILTER (WHERE total IS NULL)                           AS total_nulo,
  COUNT(*) FILTER (WHERE total <= 0)                              AS total_invalido,
  COUNT(*) FILTER (
    WHERE status NOT IN ('entregue','pendente','processando','cancelado')
  )                                                                   AS status_invalido
FROM pedidos;
Exercício 02

Crie uma view silver chamada vw_pedidos_silver que normalize os dados da tabela pedidos: status em lowercase e sem espaços, total com COALESCE para 0 quando nulo, e apenas pedidos com total maior que zero.

Básico — Silver Layer
SQL
CREATE OR REPLACE VIEW vw_pedidos_silver AS
SELECT
  id,
  cliente_id,
  LOWER(TRIM(status))       AS status,
  COALESCE(total, 0)         AS total,
  criado_em::date            AS data_pedido
FROM pedidos
WHERE COALESCE(total, 0) > 0;
Exercício 03

Usando CTEs, crie uma tabela gold de features de produto com as seguintes colunas: produto_id, nome, categoria, preco, total_vendido_qtd, receita_total, qtd_pedidos_distintos e preco_relativo_categoria (quanto o preço está acima/abaixo da média da categoria, em %).

Médio — Feature Engineering SQL
SQL
WITH

vendas AS (
  SELECT
    i.produto_id,
    SUM(i.quantidade)                         AS total_vendido_qtd,
    SUM(i.quantidade * i.preco_unit)           AS receita_total,
    COUNT(DISTINCT i.pedido_id)                AS qtd_pedidos_distintos
  FROM itens_pedido i
  GROUP BY i.produto_id
),

media_categoria AS (
  SELECT
    categoria,
    AVG(preco) AS media_preco_cat
  FROM produtos
  GROUP BY categoria
)

SELECT
  p.id             AS produto_id,
  p.nome,
  p.categoria,
  p.preco,
  COALESCE(v.total_vendido_qtd, 0)           AS total_vendido_qtd,
  COALESCE(v.receita_total, 0)               AS receita_total,
  COALESCE(v.qtd_pedidos_distintos, 0)       AS qtd_pedidos_distintos,
  ROUND(
    (p.preco - mc.media_preco_cat) /
    NULLIF(mc.media_preco_cat, 0) * 100, 1
  )                                           AS preco_relativo_categoria
FROM produtos p
LEFT JOIN vendas v          ON v.produto_id = p.id
INNER JOIN media_categoria mc ON mc.categoria = p.categoria
ORDER BY receita_total DESC;
Exercício 04

Em Python, conecte ao banco, extraia a tabela gold_features_cliente (ou construa a query equivalente inline), trate os nulos nas colunas ticket_medio e dias_inativo, e salve o resultado em um arquivo Parquet chamado features_v1.parquet. Imprima o shape final do DataFrame.

Médio — Python + Parquet
Python
import os, pandas as pd
from sqlalchemy import create_engine

engine = create_engine(os.getenv("DATABASE_URL"))

# Extrair features do banco
df = pd.read_sql("""
    SELECT c.id AS cliente_id, c.cidade, c.tem_email,
           COUNT(p.id) AS total_pedidos,
           ROUND(AVG(p.total)::numeric, 2) AS ticket_medio,
           SUM(p.total) AS total_gasto,
           MAX(CURRENT_DATE - p.criado_em::date) AS dias_inativo
    FROM silver_clientes c
    LEFT JOIN silver_pedidos p ON p.cliente_id = c.id
    GROUP BY c.id, c.cidade, c.tem_email
""", engine)

# Tratar nulos
df["ticket_medio"].fillna(0, inplace=True)
df["total_gasto"].fillna(0, inplace=True)
df["dias_inativo"].fillna(999, inplace=True)

# Salvar como Parquet
df.to_parquet("features_v1.parquet", index=False, compression="snappy")
print(f"Shape final: {df.shape}")
print(df.dtypes)
Exercício 05 — Desafio

Monte um pipeline completo em Python: 1) extraia do banco as features de produto (query do exercício 3); 2) normalize as colunas preco, total_vendido_qtd e receita_total com StandardScaler; 3) salve em Parquet; 4) grave uma tabela features_produto_v1 de volta no banco com to_sql.

Desafio — Pipeline completo BD → Python → BD
Python
import os, pandas as pd
from sqlalchemy import create_engine
from sklearn.preprocessing import StandardScaler

engine = create_engine(os.getenv("DATABASE_URL"))

# 1. Extrair features de produto do banco
query = """
    WITH vendas AS (
        SELECT produto_id,
               SUM(quantidade) AS total_vendido_qtd,
               SUM(quantidade * preco_unit) AS receita_total
        FROM itens_pedido GROUP BY produto_id
    ),
    media_cat AS (
        SELECT categoria, AVG(preco) AS media_preco_cat FROM produtos GROUP BY categoria
    )
    SELECT p.id AS produto_id, p.nome, p.categoria, p.preco,
           COALESCE(v.total_vendido_qtd, 0) AS total_vendido_qtd,
           COALESCE(v.receita_total, 0) AS receita_total,
           ROUND((p.preco - mc.media_preco_cat) /
                 NULLIF(mc.media_preco_cat,0)*100,1) AS preco_rel_cat
    FROM produtos p
    LEFT JOIN vendas v ON v.produto_id = p.id
    INNER JOIN media_cat mc ON mc.categoria = p.categoria
"""
df = pd.read_sql(query, engine)

# 2. Normalizar colunas numéricas
cols = ["preco", "total_vendido_qtd", "receita_total"]
scaler = StandardScaler()
df[[c + "_norm" for c in cols]] = scaler.fit_transform(df[cols])

# 3. Salvar como Parquet
df.to_parquet("features_produto_v1.parquet", index=False, compression="snappy")

# 4. Gravar de volta no banco
df.to_sql("features_produto_v1", engine, if_exists="replace", index=False)
print(f"Pipeline concluído: {df.shape[0]} produtos, {df.shape[1]} features")

DAGs, tarefas e dependências — como o Airflow pensa
O problema da orquestração

Nas semanas anteriores criamos scripts Python que extraem dados do banco, limpam, particionam em Parquet e validam qualidade. Funcionam — mas só se alguém rodar manualmente. Em produção, pipelines de dados precisam:

  • Executar automaticamente num horário definido
  • Respeitar dependências — não rodar a limpeza antes de terminar a extração
  • Retentar automaticamente em caso de falha transitória
  • Notificar a equipe quando algo quebra
  • Guardar histórico de cada execução com logs
  • Permitir reprocessar datas passadas (backfill)

O Apache Airflow resolve todos esses pontos. É o orquestrador de pipelines open-source mais adotado no mercado de dados.

DAG — Directed Acyclic Graph

Uma DAG (Grafo Dirigido Acíclico) é a unidade central do Airflow. Ela descreve um conjunto de tarefas e as dependências entre elas — sem ciclos (não pode haver A → B → A).

Três conceitos fundamentais:
DAG — o "plano" completo do pipeline, com agendamento e configurações globais.
Task — uma unidade de trabalho dentro da DAG (extrair, limpar, validar…).
DAG Run — uma execução concreta da DAG para uma data específica (o logical date).

# Arquivo: dags/pipeline_pedidos.py
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.operators.bash import BashOperator
from datetime import datetime, timedelta

# ── Configuração global da DAG ────────────────────────────────────
default_args = {
    "owner": "engenharia",
    "retries": 3,                        # tentativas em caso de falha
    "retry_delay": timedelta(minutes=5), # intervalo entre tentativas
    "email_on_failure": True,
    "email": ["equipe@empresa.com"],
}

with DAG(
    dag_id="pipeline_pedidos",
    description="Extrai pedidos do banco e gera features gold diariamente",
    default_args=default_args,
    schedule="0 2 * * *",   # toda noite às 02h (cron expression)
    start_date=datetime(2025, 1, 1),
    catchup=False,            # não reprocessar datas passadas ao ativar
    tags=["banco-de-dados", "lakehouse"],
) as dag:

    # ── Tarefas ───────────────────────────────────────────────────
    t1 = PythonOperator(
        task_id="extrair_pedidos",
        python_callable=extrair_pedidos,   # função definida abaixo
    )

    t2 = PythonOperator(
        task_id="limpar_silver",
        python_callable=limpar_silver,
    )

    t3 = PythonOperator(
        task_id="validar_qualidade",
        python_callable=validar_qualidade,
    )

    t4 = PythonOperator(
        task_id="gerar_gold",
        python_callable=gerar_gold,
    )

    t5 = BashOperator(
        task_id="notificar_sucesso",
        bash_command='echo "Pipeline concluído em $(date)"',
    )

    # ── Dependências (ordem de execução) ─────────────────────────
    t1 >> t2 >> t3 >> t4 >> t5
    # Equivalente: t1.set_downstream(t2) etc.
O operador >> define a dependência: t1 >> t2 significa "t2 só começa quando t1 terminar com sucesso". Para paralelismo: t1 >> [t2, t3] >> t4 — t2 e t3 rodam em paralelo, t4 espera ambos.

Os principais operadores

Cada tarefa no Airflow é criada com um Operador — uma classe que define o que aquela tarefa faz. Os mais usados em pipelines de dados:

PythonOperator
Executa qualquer função Python. O mais versátil — cobre extração, transformação, validação e carga.
BashOperator
Executa comandos shell. Útil para scripts existentes, chamadas de CLI (dbt, spark-submit) e notificações simples.
SqlSensor / SqlOperator
Espera dados ficarem disponíveis no banco (Sensor) ou executa SQL direto (Operator). Ideal para esperar tabelas de origem serem populadas.
EmailOperator
Envia e-mail com assunto e corpo configuráveis. Usado em callbacks de sucesso ou falha.
S3KeySensor
Aguarda um arquivo aparecer no S3 / MinIO antes de continuar o pipeline — útil para ingestão de arquivos externos.
@task (TaskFlow API)
Decorator moderno que transforma uma função Python diretamente em task, sem instanciar PythonOperator manualmente.
Python — TaskFlow API (forma moderna)
from airflow.decorators import dag, task
from datetime import datetime, timedelta

@dag(
    dag_id="pipeline_taskflow",
    schedule="0 2 * * *",
    start_date=datetime(2025, 1, 1),
    catchup=False,
    default_args={"retries": 2, "retry_delay": timedelta(minutes=3)},
)
def pipeline_taskflow():

    @task
    def extrair():
        import pandas as pd
        from sqlalchemy import create_engine
        import os
        engine = create_engine(os.getenv("DATABASE_URL"))
        df = pd.read_sql("SELECT * FROM pedidos", engine)
        df.to_parquet("lake/raw/pedidos/", partition_cols=["status"], index=False)
        return len(df)   # retorno vai para o XCom automaticamente

    @task
    def limpar(n_raw: int):
        import pandas as pd
        df = pd.read_parquet("lake/raw/pedidos/")
        df["criado_em"] = pd.to_datetime(df["criado_em"])
        df["ano"] = df["criado_em"].dt.year
        df["mes"] = df["criado_em"].dt.month
        df_s = df[df["total"] > 0].copy()
        df_s.to_parquet("lake/silver/pedidos/", partition_cols=["ano","mes"],
                        index=False, compression="zstd")
        print(f"raw={n_raw} → silver={len(df_s)} ({n_raw - len(df_s)} removidos)")
        return len(df_s)

    @task
    def validar(n_silver: int):
        import pandas as pd
        df = pd.read_parquet("lake/silver/pedidos/")
        assert not df["id"].isnull().any(),   "id com nulos!"
        assert not df["id"].duplicated().any(), "id duplicado!"
        assert (df["total"] > 0).all(),          "total não-positivo encontrado!"
        print(f"✓ Qualidade validada — {n_silver} registros")

    # Encadeamento com passagem de valores via XCom
    n = extrair()
    n_s = limpar(n)
    validar(n_s)

Retry, timeout e XCom

Retry e timeout

  • retries=3 — tenta até 3 vezes antes de marcar como falha
  • retry_delay=timedelta(minutes=5) — espera entre tentativas
  • execution_timeout=timedelta(hours=1) — mata a task se demorar demais
  • on_failure_callback — função chamada ao falhar definitivamente

XCom — troca de dados entre tasks

  • XCom (Cross-Communication) permite que uma task passe um valor para outra
  • Na TaskFlow API: o return da função vira XCom automaticamente
  • Ideal para metadados: número de linhas, caminhos de arquivos, status
  • Não use XCom para DataFrames grandes — armazene em disco ou S3

Backfill — reprocessar datas passadas

Se o pipeline ficou offline por 3 dias ou se um bug foi corrigido e os dados precisam ser reprocessados, o Airflow permite rodar a DAG para datas passadas com um único comando.

Bash — backfill via CLI do Airflow
# Reprocessar os últimos 3 dias
airflow dags backfill \
  --dag-id pipeline_pedidos \
  --start-date 2025-01-10 \
  --end-date 2025-01-12

# Ver histórico de runs de uma DAG
airflow dags list-runs --dag-id pipeline_pedidos

# Testar uma task isolada sem afetar o banco de metadados
airflow tasks test pipeline_pedidos extrair_pedidos 2025-01-15
catchup=False evita que o Airflow execute automaticamente todas as datas passadas ao ativar uma DAG nova. Para pipelines novos em produção, sempre use catchup=False e faça backfill manualmente quando necessário.

DAG completa — pipeline Lakehouse com sensor e alerta
Python — dags/pipeline_lakehouse.py
# dags/pipeline_lakehouse.py
from airflow.decorators import dag, task
from airflow.operators.bash import BashOperator
from datetime import datetime, timedelta
import os

def _on_failure(context):
    task_id = context["task_instance"].task_id
    exec_dt = context["logical_date"].date()
    print(f"🚨 Falha em [{task_id}] para {exec_dt}")

@dag(
    dag_id="pipeline_lakehouse",
    description="BD → raw → silver → gold — execução diária",
    schedule="0 2 * * *",
    start_date=datetime(2025, 1, 1),
    catchup=False,
    tags=["lakehouse", "bd-tecnologo"],
    default_args={
        "retries": 2,
        "retry_delay": timedelta(minutes=5),
        "execution_timeout": timedelta(hours=1),
        "on_failure_callback": _on_failure,
    },
)
def pipeline_lakehouse():

    @task
    def extrair_raw(**context):
        """Etapa 1: Extrai pedidos do banco e salva na zona raw."""
        import pandas as pd
        from sqlalchemy import create_engine

        engine = create_engine(os.getenv("DATABASE_URL"))
        df = pd.read_sql("SELECT id, cliente_id, total, status, criado_em FROM pedidos", engine)
        df.to_parquet("lake/raw/pedidos/", partition_cols=["status"], index=False)
        print(f"raw: {len(df)} linhas extraídas")
        return len(df)

    @task
    def transformar_silver(n_raw: int):
        """Etapa 2: Limpa e tipifica — zona silver."""
        import pandas as pd

        df = pd.read_parquet("lake/raw/pedidos/")
        df["criado_em"] = pd.to_datetime(df["criado_em"])
        df["status"]    = df["status"].str.lower().str.strip()
        df["total"]     = df["total"].fillna(0)
        df["ano"]       = df["criado_em"].dt.year
        df["mes"]       = df["criado_em"].dt.month
        df_s = df[df["total"] > 0].copy()
        df_s.to_parquet("lake/silver/pedidos/", partition_cols=["ano","mes"],
                        index=False, compression="zstd")
        removidos = n_raw - len(df_s)
        print(f"silver: {len(df_s)} linhas ({removidos} removidas)")
        return len(df_s)

    @task
    def validar_silver(n_silver: int):
        """Etapa 3: Validação de qualidade — quebra o pipeline se falhar."""
        import pandas as pd

        df = pd.read_parquet("lake/silver/pedidos/")
        erros = []
        if df["id"].isnull().any():      erros.append("id com nulos")
        if df["id"].duplicated().any():   erros.append("id duplicado")
        if (df["total"] <= 0).any():     erros.append("total não-positivo")
        if erros:
            raise ValueError(f"Qualidade reprovada: {erros}")
        print(f"✓ Qualidade OK — {n_silver} registros validados")

    @task
    def gerar_gold():
        """Etapa 4: Agrega features de clientes e salva em Delta Lake."""
        import pandas as pd
        from deltalake import write_deltalake

        df = pd.read_parquet("lake/silver/pedidos/")
        gold = df.groupby("cliente_id").agg(
            total_pedidos  =("id",      "count"),
            ticket_medio   =("total",   "mean"),
            total_gasto    =("total",   "sum"),
            ultimo_pedido  =("criado_em","max"),
        ).reset_index()
        gold["dias_inativo"] = (pd.Timestamp("today") - gold["ultimo_pedido"]).dt.days
        gold["ticket_medio"] = gold["ticket_medio"].round(2)
        gold["gerado_em"]    = pd.Timestamp("now")
        write_deltalake("lake/gold/features_cliente", gold, mode="overwrite")
        print(f"gold Delta: {len(gold)} clientes")

    notificar = BashOperator(
        task_id="notificar_sucesso",
        bash_command='echo "✅ pipeline_lakehouse concluído em $(date --iso-8601=seconds)"',
    )

    # ── Encadeamento com passagem de contagens via XCom ──────────────
    n_raw    = extrair_raw()
    n_silver = transformar_silver(n_raw)
    validar_silver(n_silver) >> gerar_gold() >> notificar

pipeline_lakehouse()

from airflow import DAG
from airflow.operators.python import PythonOperator
from datetime import datetime, timedelta
import os

CONTADOR_PATH = "/tmp/airflow_retry_count.txt"

def tarefa_instavel(**context):
    # Ler tentativa atual
    try:
        with open(CONTADOR_PATH) as f:
            tentativa = int(f.read().strip()) + 1
    except FileNotFoundError:
        tentativa = 1

    with open(CONTADOR_PATH, "w") as f:
        f.write(str(tentativa))

    print(f"Tentativa {tentativa}/3")
    if tentativa < 3:
        raise RuntimeError(f"Falha simulada na tentativa {tentativa}")
    print("✓ Sucesso na terceira tentativa!")
    os.remove(CONTADOR_PATH)

Exercício 04

Crie uma DAG dag_paralela com o seguinte fluxo: a task inicio extrai os dados da tabela pedidos. Em seguida, em paralelo, as tasks contar_por_status (conta pedidos por status) e calcular_ticket_medio (calcula o ticket médio geral) executam ao mesmo tempo. Por fim, a task relatorio aguarda ambas e imprime um resumo. Use a sintaxe inicio >> [t2, t3] >> relatorio.

Médio — Paralelismo de tasks
Python — dags/dag_paralela.py
from airflow.decorators import dag, task
from datetime import datetime
import os

@dag(
    dag_id="dag_paralela",
    schedule=None,
    start_date=datetime(2025, 1, 1),
    catchup=False,
    tags=["exercicio"],
)
def dag_paralela():

    @task
    def inicio():
        import pandas as pd
        from sqlalchemy import create_engine
        engine = create_engine(os.getenv("DATABASE_URL"))
        df = pd.read_sql("SELECT id, status, total FROM pedidos", engine)
        df.to_parquet("/tmp/pedidos_dag.parquet", index=False)
        return len(df)

    @task
    def contar_por_status(n: int):
        import pandas as pd
        df = pd.read_parquet("/tmp/pedidos_dag.parquet")
        contagem = df.groupby("status")["id"].count().to_dict()
        print(f"Contagem por status: {contagem}")
        return str(contagem)

    @task
    def calcular_ticket_medio(n: int):
        import pandas as pd
        df = pd.read_parquet("/tmp/pedidos_dag.parquet")
        ticket = round(df["total"].mean(), 2)
        print(f"Ticket médio: R$ {ticket}")
        return ticket

    @task
    def relatorio(contagem: str, ticket: float):
        print("=== RELATÓRIO ===")
        print(f"Distribuição: {contagem}")
        print(f"Ticket médio: R$ {ticket}")

    n = inicio()
    c = contar_por_status(n)
    t = calcular_ticket_medio(n)
    relatorio(c, t)

        df = pd.read_parquet("lake/raw/pedidos/")
        if df["id"].isnull().any():
            raise ValueError("id com nulos na zona raw")
        if df["id"].duplicated().any():
            raise ValueError("id duplicado na zona raw")
        print(f"✓ raw validado: {n} registros")
        return n

    @task
    def transformar_silver(n: int):
        import pandas as pd, hashlib
        df = pd.read_parquet("lake/raw/pedidos/")
        df["criado_em"] = pd.to_datetime(df["criado_em"])
        df["status"]    = df["status"].str.lower().str.strip()
        df["total"]     = df["total"].fillna(0)
        df["ano"]       = df["criado_em"].dt.year
        df["mes"]       = df["criado_em"].dt.month
        SALT = os.getenv("HASH_SALT", "salt_dev")
        if "email" in df.columns:
            df["email_hash"] = df["email"].apply(
                lambda v: hashlib.sha256(f"{SALT}{v}".encode()).hexdigest()[:16]
            )
            df = df.drop(columns=["email"])
        df_s = df[df["total"] > 0].copy()
        df_s.to_parquet("lake/silver/pedidos/", partition_cols=["ano","mes"],
                        index=False, compression="zstd")
        print(f"silver: {len(df_s)} linhas")
        return len(df_s)

    @task
    def gerar_gold_delta(n: int):
        import pandas as pd
        from deltalake import write_deltalake
        df = pd.read_parquet("lake/silver/pedidos/")
        gold = df.groupby("cliente_id").agg(
            total_pedidos =("id", "count"),
            ticket_medio  =("total", "mean"),
            total_gasto   =("total", "sum"),
            ultimo_pedido =("criado_em", "max"),
        ).reset_index()
        gold["dias_inativo"] = (pd.Timestamp("today") - gold["ultimo_pedido"]).dt.days
        gold["ticket_medio"] = gold["ticket_medio"].round(2)
        gold["gerado_em"] = pd.Timestamp("now")
        write_deltalake("lake/gold/features_cliente", gold, mode="overwrite")
        print(f"gold: {len(gold)} clientes")
        return len(gold)

    @task
    def registrar_linhagem(n_raw: int, n_silver: int, n_gold: int):
        import json
        from datetime import datetime as dt
        log = {
            "ts": dt.now().isoformat(),
            "pipeline": "dag_lakehouse_completa",
            "etapas": [
                {"nome": "raw",    "registros": n_raw},
                {"nome": "silver", "registros": n_silver,
                 "ops": ["limpeza", "hash email", "filtro total>0"]},
                {"nome": "gold",   "registros": n_gold,
                 "ops": ["groupby cliente_id", "agg features", "delta lake"]},
            ]
        }
        os.makedirs("lake/logs", exist_ok=True)
        fname = f"lake/logs/linhagem_{dt.now().strftime('%Y%m%d_%H%M%S')}.json"
        with open(fname, "w") as f:
            json.dump(log, f, indent=2, ensure_ascii=False)
        print(f"Linhagem: raw={n_raw} → silver={n_silver} → gold={n_gold}")
        print(f"Gravado em {fname}")

    n_raw    = extrair_raw()
    n_raw_v  = validar_raw(n_raw)
    n_silver = transformar_silver(n_raw_v)
    n_gold   = gerar_gold_delta(n_silver)
    registrar_linhagem(n_raw, n_silver, n_gold)
